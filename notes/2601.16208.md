## 1. 理论推导

### RepRAE 核心设计

Representation Autoencoder (RAE) 的核心思想是：**用冻结的预训练表征编码器替代 VAE 编码器，只训练解码器来从高维语义特征重建像素**。这与标准 VAE tokenizer 有本质区别：

**标准 VAE tokenizer：**
- 编码器和解码器联合训练，目标是最小化重建损失 + KL 正则项
- 压缩到低维隐空间（通道维度通常 <64，如 FLUX VAE 为 16 通道）
- 编码器产生的隐变量缺乏语义结构，仅服务于像素重建
- 理解和生成需要两套不同的编码器（双塔设计）

**RAE (RepRAE) tokenizer：**
- 编码器完全冻结（本文使用 SigLIP-2 So400M，patch size 14）
- 只训练 ViT-XL 解码器，从冻结编码器的高维语义特征重建像素
- 隐空间维度远高于 VAE：输入 224x224 图像，编码器产生 N=16x16=256 个 token，每个 token 维度 d=1152，有效数据维度 m = N x d = 294,912
- 编码器产生的特征天然具有语义结构，可用于视觉理解任务

### 解码器训练目标

给定输入图像 x，RAE 解码器训练目标为：

L(x, x̂) = ℓ₁(x, x̂) + ω_L * LPIPS(x, x̂) + ω_G * Gram(x, x̂) + ω_A * Adv(x, x̂)

其中 x̂ = RAE(x)。各项含义：
- ℓ₁: 像素级 L1 重建损失
- LPIPS: 感知损失（学习到的图像相似度度量）
- Gram: Gram 矩阵风格损失，帮助保持重建的纹理细节
- Adv: 对抗损失，判别器使用冻结的 DINO-S/16
- 损失系数: ω_G = 100.0, ω_L = 1.0, ω_A = 10.0

### 维度感知噪声调度（Dimension-Dependent Noise Schedule Shift）

这是 RepRAE 中最关键的设计。高维隐空间中，标准噪声调度失效，需要根据有效数据维度 m 调整时间步：

给定基准维度 n 的基础调度 t_n ∈ [0, 1]，偏移后的时间步为：

t_m = (α * t_n) / (1 + (α - 1) * t_n)

其中 α = m / n，m = N × d（token 数 × token 维度），本文以 n = 4096 为基准。

实验表明，不加 shift 时 GenEval 仅 23.6、DPG-Bench 仅 54.8；加 shift 后分别提升到 49.6 和 76.8。

### Flow Matching 目标

采用线性插值的 flow matching：

x_t = (1 - t)x + tε,  其中 x ~ p(x), ε ~ N(0, I)

模型训练预测速度场 v(x_t, t)，推理使用 50 步 Euler 采样器。

### 规模化后的简化发现

论文系统性地消融了原始 RAE 在 ImageNet 上的设计选择，发现在大规模 T2I 场景下：
1. **维度感知噪声调度**：必不可少，直接解决高维隐空间的数学性质
2. **宽扩散头 (DiTDH)**：在小模型（0.5B）上有 +11.2 GenEval 提升，但 DiT >= 2.4B 时优势消失——因为大模型隐藏维度 (d >= 2048) 已自然超过隐变量维度 (d=1152)
3. **噪声增强解码 (Noise-augmented decoding)**：训练早期有收益，但 ~15k 步后收益可忽略——本质上是一种正则化，只在模型尚未学到鲁棒隐流形时有用

结论：规模化简化了 RAE 框架，只需标准 DiT 架构 + 正确的噪声调度即可。

## 2. 最小实验复现资源

### 数据集
- **解码器训练**：73M 图像，来自三个数据源：
  - Web 图像（FuseDiT 数据，~39.3M）
  - 合成图像（FLUX.1-schnell 生成，~24.7M）
  - 文本渲染图像（RenderedText）
- **T2I 预训练**：
  - Web 数据：CC12M, SA-1B, JourneyDB 等，约 39.3M
  - 合成数据：FLUX.1-schnell 生成，24.7M
  - 视觉理解：Cambrian-7M
  - 最优配置为 Web + Synthetic 混合（49.5 GenEval），优于单独加倍任一数据源
- **微调**：BLIP-3o 60k 数据集

### 模型
- **表征编码器**：SigLIP-2 So400M（patch size 14），冻结不训练
- **替代编码器**：WebSSL ViT-L（实验验证框架对编码器选择的鲁棒性）
- **解码器**：ViT-XL（28 blocks, hidden=1152, 16 heads）
- **语言模型**：Qwen-2.5（1.5B / 7B）
- **DiT**：0.5B / 2.4B / 3.3B / 5.5B / 9.8B（基于 LightningDiT 设计，宽度优先扩展）
- **VAE 基线**：FLUX VAE（SOTA）

### 硬件
- TPU v4, v5p, v6e
- 预训练主要用 TPU-v5p-128 和 TPU-v6e-64
- 微调用 TPU-v4-128 和 TPU-v5p-64
- 使用 TorchXLA + SPMD sharding 进行分布式训练

### 训练配置
- **预训练**：batch size 2048，30k iterations（1.5B LLM + 2.4B DiT）
- **微调**：batch size 1024，最多 256 epochs（BLIP-3o 60k）
- LLM 和 DiT 使用独立的优化器配置以提升稳定性

### 评估指标
- GenEval（文本-图像对齐）
- DPG-Bench（DPG 分数）
- rFID-50k（重建质量，在 ImageNet / YFCC / RenderedText 三个域评估）

## 3. 重要idea

### 3.1 表征自编码器作为扩散模型的更好 tokenizer

核心洞察：VAE 的编码器训练目标是像素重建 + KL 正则，产生的隐空间缺乏语义结构；而预训练表征编码器（如 SigLIP-2）经过大规模对比学习/自监督训练，其隐空间天然具有丰富的语义信息。RAE 的做法是**直接在这个语义空间上做扩散**，而不是压缩到低维 VAE 空间。实验证明，即使 RAE 的重建 rFID 不如 FLUX VAE（例如 ImageNet rFID 0.435 vs 0.288），基于 RAE 的扩散模型训练仍然显著优于 VAE——说明**tokenizer 对下游扩散训练的价值不只取决于重建保真度，更取决于隐空间的语义质量**。

### 3.2 隐空间表征对齐如何改善扩散训练

RAE 隐空间的高维语义结构带来两个关键优势：

1. **更快的收敛速度**：在 T2I 预训练中，RAE 相比 VAE 在 GenEval 上快 4.0x，DPG-Bench 上快 4.6x 达到同等性能。语义丰富的隐空间让扩散模型更容易学习数据分布。

2. **更好的抗过拟合能力**：微调阶段，VAE 模型在 64 epochs 后灾难性过拟合（训练 loss 骤降至接近零），而 RAE 模型在 256 甚至 512 epochs 后仍然稳定。论文假设高维语义隐空间（1152 维 vs VAE 的 <100 维）提供了隐式正则化效果，阻止模型记忆训练样本。

### 3.3 表征质量与生成质量的关联

论文揭示了一个重要的深层联系：**用于理解的表征质量可以直接转化为更好的生成质量**。这打破了"理解需要高维语义空间、生成需要低维压缩空间"的传统观念。具体体现：

- RAE 隐空间同时服务理解和生成，模型生成的隐变量就是它用于理解的同一组特征
- 统一隐空间使得 LLM 可以直接在隐空间内评估自己生成结果的质量（Latent Test-Time Scaling），无需 decode-re-encode 的往返
- 添加生成训练不损害理解性能（Tab. 6），且 RAE vs VAE 在生成路径的选择对理解性能影响甚微——因为两者共享同一冻结理解编码器

### 3.4 数据组成比数据规模更重要

解码器训练中，仅在 ImageNet 上训练的解码器无法重建文字场景（rFID 2.640）；加入针对性文字渲染数据后大幅改善（1.621）。在 T2I 预训练中，Web + Synthetic 混合数据（49.5 GenEval）优于单独加倍合成数据（48.0），说明互补数据源的协同效应大于单纯增加数据量。

### 3.5 Latent Test-Time Scaling

利用共享隐空间，LLM 可以直接作为验证器评估扩散模型生成的隐变量，无需渲染成像素再重新编码。两种验证指标：（1）Prompt Confidence——重新注入生成隐变量和原始文本，测量 prompt token 的置信度；（2）Answer Logits——直接询问 LLM "生成图像是否与 prompt 对齐"。Best-of-N 选择策略在 GenEval 上带来显著提升（如 4/32 选择从 53.2 提升到 60.0），验证了隐空间内 TTS 的可行性。
