# DeepSeek-R1: Incentivizing Reasoning Capability in LLMs via Reinforcement Learning (DeepSeek-AI, 2025)

**arxiv**: 2501.12948 | **GRPO / 纯 RL 推理开山之作** | 2025

---

## 一、核心贡献

DeepSeek-R1 证明了**纯强化学习（不依赖监督微调中的推理链数据）可以激发大语言模型的推理能力**。具体贡献：

1. **GRPO（Group Relative Policy Optimization）**：新的 RL 算法，无需独立的 critic 模型，通过组内相对奖励做基线，大幅降低训练成本
2. **DeepSeek-R1-Zero**：直接在基础模型上做 RL 训练，不使用任何监督推理数据，模型自发涌现出 Chain-of-Thought 推理行为（包括自我反思、尝试多种策略）
3. **DeepSeek-R1**：冷启动 + 多阶段训练流程，最终性能媲美 OpenAI-o1
4. **推理蒸馏**：将 R1 的推理能力蒸馏到小模型（1.5B-70B），小模型也获得强推理能力

## 二、关键技术点

### GRPO 算法

传统 RLHF 用 PPO，需要一个与策略模型同等规模的 value function（critic）来估计优势函数。GRPO 的核心创新：**不需要 critic**。

给定一个问题 $q$，从旧策略 $\pi_{\theta_{\text{old}}}$ 采样一组输出 $\{o_1, o_2, \ldots, o_G\}$（$G=16$），对每个输出计算奖励 $r_i$：

**优势函数估计**——组内归一化：

$$\hat{A}_i = \frac{r_i - \text{mean}(r_1, \ldots, r_G)}{\text{std}(r_1, \ldots, r_G)}$$

**GRPO 目标函数**：

$$\mathcal{J}_{\text{GRPO}}(\theta) = \mathbb{E}\left[\frac{1}{G}\sum_{i=1}^G \min\left(\frac{\pi_\theta(o_i|q)}{\pi_{\theta_{\text{old}}}(o_i|q)} \hat{A}_i, \text{clip}\left(\frac{\pi_\theta(o_i|q)}{\pi_{\theta_{\text{old}}}(o_i|q)}, 1 \pm \epsilon\right) \hat{A}_i\right) - \beta \mathbb{D}_{\text{KL}}(\pi_\theta \| \pi_{\text{ref}})\right]$$

与 PPO 的区别：
- **无需 value function**：用组内奖励的统计量替代 critic 的估计
- **组内相对比较**：奖励的含义是"相对于组内其他输出好不好"，而非绝对值
- **KL 惩罚**：限制策略不要偏离参考策略太远（$\beta = 0.001$）

**采样组大小 G** 的选择是关键超参数——G 越大，优势估计越准确，但计算成本越高。

### DeepSeek-R1-Zero：纯 RL 训练

R1-Zero 的训练流程极其简洁：

1. 基础模型：DeepSeek-V3-Base（671B MoE）
2. 直接在数学/编程/推理任务上做 GRPO 训练
3. **没有任何监督推理数据**（无 SFT 阶段）

涌现行为：
- 模型自发学会生成推理链（Chain-of-Thought）
- 出现"自我反思"：在推理过程中发现错误，回退并尝试新策略
- 输出中自然出现 "Wait, let me reconsider..." 之类的思考标记——"Aha moment"
- 推理长度随训练进行而增长（从 ~2000 增长到 ~8000 token）

**问题**：R1-Zero 的输出可读性差、语言混杂（中英文混用），这促使了 R1 的多阶段训练。

### DeepSeek-R1：多阶段训练

**阶段一：冷启动（Cold Start）**
- 用少量高质量长 CoT 数据做 SFT
- 目的：给模型一个推理格式的"样板"，避免 R1-Zero 的格式混乱
- 数据来源：用 R1-Zero 生成 + 人工过滤，或用 few-shot prompting 生成

**阶段二：推理导向的 RL 训练**
- 在数学、编程、科学推理任务上做 GRPO
- 加入语言一致性奖励：鼓励输出使用统一语言，减少语言混杂

**阶段三：拒绝采样 + 全域 SFT**
- 用阶段二的模型生成候选输出，用奖励模型筛选高质量数据
- 与非推理数据（写作、对话等）混合，做全参数 SFT
- 目的：恢复通用能力（纯 RL 训练会损害非推理任务的表现）

**阶段四：全域 RL**
- 在所有任务上做第二轮 RL
- 推理任务用规则奖励（数学答案是否正确、代码是否通过测试）
- 通用任务用神经网络奖励模型

### 奖励设计

DeepSeek-R1 的奖励系统有两个特点：

1. **规则奖励优先**：数学用精确匹配，编程用测试用例，不依赖神经网络奖励模型（避免 reward hacking）
2. **不用过程奖励模型（PRM）**：只用结果奖励。作者发现 PRM 容易被 hack，且推理过程的每步打分很难定义
3. **Format reward**：推理过程是否放在 `<think>...</think>` 标签里

### 蒸馏：将推理能力传递给小模型

DeepSeek-R1 可以作为教师模型，将推理能力蒸馏到更小的模型：
1. 用 R1 生成推理链数据
2. 在小模型（Qwen-1.5B/7B/14B/32B, Llama-8B/70B）上做 SFT
3. 蒸馏后的小模型推理能力显著超过原始模型

关键发现：**蒸馏比小模型直接做 RL 效果更好**。小模型的 RL 训练不稳定，可能因为基础推理能力不足。

## 三、实验结果

### 推理基准

| 模型 | MATH-500 | AIME 2024 | Codeforces | LiveCodeBench |
|------|----------|-----------|------------|--------------|
| DeepSeek-V3 | 90.2 | 39.6 | 51.6% | 32.0 |
| OpenAI o1-mini | 90.0 | 63.6 | 72.0% | 53.8 |
| DeepSeek-R1-Zero | 95.0 | 71.6 | - | - |
| OpenAI o1-1217 | 96.8 | 79.2 | 96.6% | - |
| **DeepSeek-R1** | **97.3** | **79.8** | **96.3%** | **71.8** |

DeepSeek-R1 在 MATH-500 上超越 o1-1217，在 AIME 上接近，在代码竞赛上达到 96.3 百分位。

### 通用能力

在 MMLU、MMLU-Pro、IFEval 等通用基准上，R1 也保持在 V3 水平附近，没有因推理训练而退化。

### 蒸馏效果

| 模型 | MATH-500 | AIME 2024 |
|------|----------|-----------|
| Qwen-1.5B (原始) | 39.0 | 10.0 |
| Qwen-1.5B + R1 蒸馏 | **83.9** | 28.9 |
| Qwen-7B + R1 蒸馏 | **92.8** | 46.7 |
| Qwen-32B + R1 蒸馏 | **94.4** | 55.3 |
| Llama-70B + R1 蒸馏 | **95.0** | 56.7 |

蒸馏使小模型推理能力大幅跃升。

### R1-Zero 的训练动态

- AIME 2024 准确率从 15.6% -> 77.9%（pass@1），cons@16 达 86.7%
- 响应长度自然增长：模型自发地用更多 token 来思考
- "Aha moment"：模型学会了自我反思——"Wait, wait. That's an aha moment I can flag here."

### 关键消融

1. **冷启动 vs 零启动**：冷启动在格式、可读性上显著更好，推理精度也略高
2. **RL 轮数**：更多 RL 轮数持续提升推理能力，没有饱和
3. **语言一致性奖励**：显著减少语言混杂，对推理精度影响不大
4. **过程奖励 vs 结果奖励**：结果奖励更稳定，过程奖励容易被 hack

## 四、个人思考与影响

### 纯 RL 可以激发推理能力

R1-Zero 最令人震撼的结果：**不需要任何人类推理示范，纯 RL 就能让模型自发学会推理**。这暗示：

1. 大语言模型的推理能力可能是"潜在"的——预训练已经赋予了推理能力，RL 只是"释放"了它
2. 类比：人类学生做数学题，没人教推理过程，只是做对了得奖励——这和 R1-Zero 的训练方式惊人相似
3. 这对"推理是否需要特殊训练数据"的争论给出了强有力的一侧证据

### GRPO 的意义

GRPO 的核心价值是**去掉了 critic 模型**：
- PPO 需要策略模型 + critic 模型，训练成本翻倍
- GRPO 用组内统计量替代 critic，简单有效
- 代价：优势估计不如 critic 精确（尤其是 G 较小时），但实际效果很好

这降低了 RL 训练 LLM 的门槛——不需要训练和维护一个巨大的 value function。几乎所有后续的推理模型训练都采用了 GRPO 或其变体。

### 蒸馏 > 小模型 RL

蒸馏效果优于小模型直接 RL 这一发现有重要含义：
- 推理能力可能存在"规模门槛"——太小的模型缺乏推理的"基础设施"
- 大模型的推理链是宝贵资源，应该充分利用（蒸馏）
- 对工程实践：训练一个超级推理模型 + 蒸馏到各尺寸，比每个尺寸都做 RL 更高效

### 开放问题

1. **推理的涌现机制**：为什么 RL 能"解锁"推理？模型内部发生了什么？这需要更多可解释性研究
2. **过度思考**：R1 有时会在简单问题上过度推理，浪费计算
3. **推理泛化**：RL 训练的推理能力能迁移到训练分布外的问题吗？
4. **安全风险**：更强的推理能力意味着更强的"越狱"能力，需要配套安全措施
5. **GRPO 的理论性质**：组内归一化的优势估计是否无偏？收敛性如何？
6. **SFT 可能限制推理**：过早 SFT 会让模型模仿人类思维模式，错过更好的非人类推理路径

### 对行业的冲击

DeepSeek-R1 以开源方式发布了与 o1 媲美的推理模型，且训练方法透明。这对行业的影响：
- 证明了高性能推理模型不一定需要海量推理数据，RL 是可行的替代路径
- 蒸馏方案使得推理能力可以低成本地扩散到各种规模
- 开源 + 透明的训练流程，对闭源 API 构成竞争

DeepSeek-R1 是 2025 年初最重要的工作之一，它不仅是一个强力模型，更验证了"纯 RL 激发推理"这一范式的可行性，为后续推理模型的研究奠定了基础。
