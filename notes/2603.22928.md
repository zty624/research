# SoK: The Attack Surface of Agentic AI — Tools, and Autonomy (Dehghantanha & Homayoun, 2026)

**arxiv**: 2603.22928 | **Agentic AI 安全 SoK** | 2026

---

## 一、核心动机

随着 LLM 从聊天机器人进化为具备工具调用、RAG 检索、自主规划能力的 Agentic 系统，攻击面发生了质的变化。传统 LLM 安全研究关注单轮输入输出的安全对齐，而 Agentic AI 引入了**系统组合风险**——攻击可以通过检索内容、工具接口、多 Agent 交互等间接路径实现端到端利用。本文作为 SoK（Systematization of Knowledge），系统梳理 Agentic LLM 系统的信任边界、威胁模型和攻击面。

## 二、关键概念与框架

### 参考架构与信任边界

论文提出了 Agentic LLM 的参考管线（Figure 1），包含用户接口、规划器、LLM 核心、工具代理、检索器、记忆存储、审计等组件，并标注了 10 个攻击面（AS1-AS10）：

- **AS1**: 用户输入
- **AS2**: 检索内容入口
- **AS3**: 工具调用序列化
- **AS4**: 沙箱边界
- **AS5**: 文件 I/O
- **AS6**: API Token 范围
- **AS7**: 索引器
- **AS8**: 检索器/ANN
- **AS9**: 长期记忆
- **AS10**: 审计/遥测

可信计算基（TCB）包括：模型权重和系统提示、工具执行基座（沙箱/broker/schema 验证器）、密钥存储、专有 RAG 语料库的索引管线和存储。非 TCB 元素（用户输入、外部内容、插件输出）必须在整个管线中防御性处理。

### 威胁模型与对手分类

四类对手：
1. **外部攻击者**（无特权）：通过直接提示注入或恶意 URL 进行攻击
2. **恶意内容提供者**：控制 Agent 可能消费的数据（网页中嵌入隐藏指令、投毒 PDF、恶意 API 响应）
3. **供应链攻击者**：发布木马插件/包、投毒模型检查点或向量库
4. **内部人员/开发者**：可直接访问配置、提示、密钥或语料库

### 因果威胁图

论文创新性地使用有向无环图建模攻击传播路径，节点表示管线阶段或系统状态，边表示因果影响。例如：恶意网页 → [间接提示注入] → LLM 输出恶意工具命令 → [工具执行] → 系统文件被修改。这种图有助于识别最脆弱或最关键的防御断裂点。

### 攻击分类体系

七大攻击目标（G1-G7）：信息泄露、策略绕过、代码执行、拒绝服务、财务欺诈、持久化、供应链妥协。

五大攻击路径（P1-P5）：
- **P1**: 直接提示 → 工具滥用
- **P2**: 间接提示注入（通过检索内容）
- **P3**: 工具/API 漏洞利用
- **P4**: RAG 索引投毒
- **P5**: 多 Agent 交叉操纵

### 攻击向量分类

| 向量 | 典型前置条件 | 持久性 | 影响 |
|------|-------------|--------|------|
| Prompt/Content | 用户输入或可达内容 | Session | 泄露、策略绕过 |
| RAG (Indexing) | 语料库修改权限 | Index | 静默腐败、延迟触发 |
| RAG (Retrieval) | 未审查来源、弱 ACL | N/A | 针对性污染 |
| Tool | 工具存在、弱 Schema | Workspace | RCE、成本滥用 |
| API/Cloud | 过宽 Token | Token | 横向移动 |
| State/Memory | 可写记忆 | Memory | 目标劫持、后门 |
| Multi-agent | 开放总线、弱认证 | Agent mesh | 级联越狱 |
| Supply chain | 依赖/插件信任 | Update channel | 持久妥协 |

## 三、证据综合

论文综述了 2023-2025 年间 20+ 篇同行评审和归档研究、行业报告及标准。关键发现：

1. **间接提示注入**是最严重的 Agentic 威胁：恶意指令嵌入检索内容中可端到端劫持 Agent 行为
2. **RAG 投毒**：少量植入文档即可操纵回答方向
3. **工具桥暴露经典漏洞**：一项研究发现 11 个框架中存在 19 个 RCE 缺陷
4. **多 Agent 系统**的交叉操纵风险：一个被劫持的 Agent 可通过消息总线影响其他 Agent

## 四、评估指标

论文提出了一套攻击者感知的评估指标：

- **UAR**（Unsafe Action Rate）：不安全动作比例
- **PAR**（Policy Adherence Rate）：策略遵从率
- **PED**（Privilege Escalation Distance）：提权距离
- **RRS**（Retrieval Risk Score）：检索风险分数
- **TTC**（Time-to-Contain）：遏制时间
- **OORAR**（Out-of-Role Action Rate）：越角色行为率
- **CES**（Cost-Exploit Susceptibility）：成本利用敏感性

## 五、防御与实务

防御深度框架跨越四层：数据层、推理层、规划/执行层、环境层。关键防御措施包括：

- 输入清洗和内容过滤
- 检索过滤器（ACL 感知检索）
- 沙箱隔离和最小权限
- Schema 验证和工具调用约束
- AI 护栏（guardrails）
- 审计和遥测

论文还提供了分阶段安全检查清单（设计时加固、运行时监控、事件响应），映射到 OWASP GenAI 和 MITRE ATLAS 框架。

## 六、个人评价

这篇 SoK 是 Agentic AI 安全领域最早期的系统化工作之一，其核心贡献在于：

1. **攻击面的系统化建模**：将散落在各论文中的攻击实例整理为统一的分类体系和因果威胁图，使安全工程师能够系统地评估风险
2. **TCB 和信任边界的清晰定义**：为 Agentic 系统的安全架构设计提供了理论基础
3. **指标体系**：提出的 UAR/PED 等指标为安全评估提供了可操作的量化方法

不足之处：
- 防御措施部分主要依赖传统安全思路（沙箱、ACL、过滤），对 LLM 内生安全能力的讨论较浅
- 缺乏实验性的攻击/防御验证，更多是文献综述和框架设计
- 多 Agent 安全威胁的讨论仍偏概念性，实证数据有限

这篇论文对理解 Agentic AI 的安全风险非常有价值，随着 Agent 系统的大规模部署，其提出的威胁模型和防御框架将越来越重要。
