# Improved Baselines with Visual Instruction Tuning (Liu et al., 2023)

**arxiv**: 2310.03744 | **关键词**: 视觉指令微调, 多模态大模型, LLaVA, MLP连接器 | University of Wisconsin-Madison, Microsoft Research | CVPR 2024

---

## 一、核心问题

如何系统性地提升大型多模态模型（LMM）的性能？LLaVA 展示了视觉指令微调的潜力，但在需要简短答案的学术基准（如 VQA）上表现不佳；InstructBLIP 虽然在 VQA 上更强，却难以平衡自然对话和学术任务。

关键矛盾：**多任务平衡问题**——如何在保持自然对话能力的同时，提升学术任务上的短答案精度？现有方法要么偏向长回答（LLaVA），要么偏向短回答（InstructBLIP），无法兼顾。

## 二、方法

### 2.1 响应格式提示（Response Format Prompting）

核心发现：InstructBLIP 无法平衡长/短答案的根本原因是**提示歧义**和**未微调 LLM**：
- 歧义提示 `Q: {Question} A: {Answer}` 不清楚指示输出格式，导致 LLM 行为偏向短答案
- 仅微调 QFormer 而非 LLM，QFormer 容量有限，无法控制输出长度

解决方案：在 VQA 问题时附加格式提示——"Answer the question using a single word or phrase." 当 LLM 被微调时，它能根据提示正确调整输出格式，无需额外处理 VQA 答案。

仅通过添加 VQAv2 + 格式提示，LLaVA 在 MME 上的分数从 809.6 提升到 1323.8（+514），超过 InstructBLIP 111 分。

### 2.2 MLP 视觉-语言连接器

将 LLaVA 原始的单层线性投影替换为**两层 MLP**，受自监督学习中线性投影到 MLP 的改进启发。MLP 连接器提供更强的表征能力，同时保持数据高效。

### 2.3 学术任务导向数据

在 LLaVA 原有的视觉指令数据基础上，逐步增加：
- 开放知识 VQA：OKVQA、A-OKVQA
- OCR：OCRVQA、TextCaps
- 区域级 VQA：Visual Genome、RefCOCO/RefCOCO+/RefCOCOg
- 额外视觉知识：GQA

### 2.4 扩展策略

- **分辨率提升**：从 CLIP-ViT-L-14@224px 升级到 CLIP-ViT-L-14@336px
- **LLM 扩展**：从 7B 升级到 13B（Vicuna-13B）
- **语言数据**：加入 ShareGPT 数据

### 2.5 LLaVA-1.5-HD：高分辨率扩展

为突破 CLIP 视觉编码器的分辨率限制，提出**分块编码策略**：
1. 将高分辨率图像分割为编码器原生分辨率的小块
2. 独立编码每个小块的特征
3. 合并为大特征图
4. 拼接一个下采样图像的特征作为全局上下文

优势：无需位置编码插值，可扩展到任意分辨率，保持数据高效。

## 三、实验结果

### 渐进消融（7B 模型）
| 配置 | GQA | MME | MM-Vet |
|------|-----|-----|--------|
| LLaVA 基线 | - | 809.6 | - |
| +VQAv2 | 47.0 | 1197.0 | - |
| +格式提示 | 46.8 | 1323.8 | - |
| +MLP 连接器 | 47.3 | 1355.2 | - |
| +OKVQA/OCR | 50.0 | 1377.6 | - |

### 最终结果（LLaVA-1.5-13B）
- **VQAv2**: 80.0%（超过 Qwen-VL-Chat 78.2%）
- **GQA**: 63.3%（超过 Qwen-VL-Chat 57.5%）
- **TextVQA**: 58.2%
- **MME**: 1531.3（超过 InstructBLIP 1212.8）
- **MM-Vet**: 36.1（超过 Qwen-VL-Chat 25.6%）
- **POPE**: 85.9 F1（抗幻觉）

仅使用 1.2M 公开数据，单 8-A100 节点 ~1 天训练完成。对比 Qwen-VL-Chat 使用 1.4B 预训练数据 + 50M 微调数据（含非公开数据），InstructBLIP 使用 129M 预训练数据。

### LLaVA-1.5-HD
- 分辨率提升到 448×448 进一步改善，尤其 OCR 和细节感知
- 全局上下文消融：+0.9 GQA, +71 MME, +3.2 MM-Vet

### 涌现能力
- 格式指令泛化：仅用少量格式提示训练，可泛化到新的输出格式（如 JSON）
- 减少幻觉：数据组合 + 格式提示有效降低幻觉率

## 四、个人评价

### 影响
- **简洁即力量**：证明最简单的架构（MLP 连接器 + 全量微调 LLM）配合正确的数据策略，可以超越复杂架构（Q-Former + 大规模预训练），为后续 LMM 研究提供了清晰的基线
- **格式提示的重要性**：揭示了多任务平衡的关键在于清晰的输出格式指示，这一发现在后续工作中被广泛采用
- **开源标杆**：LLaVA-1.5 成为开源 LMM 的标准基线，推动了多模态研究的民主化
- **数据效率**：1.2M 公开数据 + 1 天训练即可达到 SoTA，对资源受限的研究者意义重大

### 局限
- **分辨率受限于 CLIP 编码器**：HD 版本的分块编码虽能扩展分辨率，但存在分块边界伪影
- **视频理解缺失**：仅支持图像输入，未涉及视频理解
- **文本提示能力有限**：仍主要依赖几何提示（点、框），文本到 mask 的能力弱
- **学术任务数据泄漏**：GQA 等训练集在评测中被使用（标记 *）
- **缺乏深度理解**：对复杂场景图、空间推理等能力有限
