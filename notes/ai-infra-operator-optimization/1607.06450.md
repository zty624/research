# Layer Normalization (Ba et al., 2016)

**arxiv**: 1607.06450 | **Transformer 归一化基础** | 2016

---

## 一、核心问题

Batch Normalization (Ioffe & Szegedy, 2015) 在训练深度网络时非常有效，但有一个根本缺陷：**归一化统计量沿 batch 维度计算，严重依赖 batch size**。

这在以下场景成为致命问题：

1. **RNN / LSTM**：每个时间步的隐状态长度不同，无法在 batch 维度上对齐做归一化
2. **小 batch 训练**：当 batch size 很小时（如在线学习、大模型受显存限制），batch 统计量噪声极大，BN 效果骤降
3. **训练/测试不一致**：BN 训练时用 batch 统计量，测试时用 running statistics（指数移动平均），两者分布可能有差异
4. **序列模型**：不同样本的序列长度不同，padding 位置不应参与统计

本文提出 Layer Normalization：沿特征维度归一化，彻底消除对 batch 的依赖。

## 二、方法

### LayerNorm 计算

对每个样本独立计算归一化统计量，沿特征维度（而非 batch 维度）：

$$\mu = \frac{1}{H} \sum_{i=1}^{H} x_i, \quad \sigma = \sqrt{\frac{1}{H} \sum_{i=1}^{H} (x_i - \mu)^2 + \epsilon}$$

$$\hat{x}_i = \frac{x_i - \mu}{\sigma}, \quad y_i = \gamma_i \cdot \hat{x}_i + \beta_i$$

其中 $H$ 是特征维度大小，$\gamma$ 和 $\beta$ 是可学习的缩放和偏移参数（与 BN 相同），$\epsilon$ 是防止除零的小常数。

### 与 BatchNorm 的核心区别

| 属性 | BatchNorm | LayerNorm |
|------|-----------|-----------|
| **归一化维度** | 沿 batch 维度（跨样本） | 沿 feature 维度（跨特征） |
| **统计量** | 每个 feature channel 一个 $\mu, \sigma$ | 每个样本一个 $\mu, \sigma$ |
| **对 batch size 的依赖** | 严重依赖 | 完全独立 |
| **训练/测试行为** | 不同（测试用 running stats） | 完全相同 |
| **适用场景** | CNN，大 batch | RNN/LSTM，任意 batch size |
| **参数 $\gamma, \beta$ 的维度** | (C,) 每个 channel | (H,) 每个 feature |

直观理解：BN 是"在同一个 feature 位置上，跨样本看分布"，LN 是"在同一个样本内，跨特征看分布"。

### 为什么 LN 适合 RNN

在 RNN 中，每个时间步的隐状态 $h_t$ 是一个特征向量。LN 对每个 $h_t$ 独立归一化：

- 不同时间步不需要共享统计量，天然适配变长序列
- 不需要维护 running statistics
- batch 中不同样本的序列长度可以不同
- 在每个时间步都能获得稳定的梯度

而 BN 在 RNN 中需要为每个时间步维护独立的统计量（因为各时间步的分布不同），这既不优雅也不实用。

### LN 在 CNN 中的局限

论文承认 LN 对 CNN 帮助不大。原因是 CNN 的同一 feature map 内不同空间位置之间有强相关性（都在检测类似的模式），沿 feature 维度归一化会破坏这种结构。而 BN 沿 batch 维度归一化，每个 channel 独立处理，保留了空间结构。

这也解释了为什么后来的 Vision Transformer 仍使用 LayerNorm（因为 ViT 的 patch embedding 已经打破了局部空间结构），而 CNN 架构偏好 GroupNorm 或 BatchNorm。

## 三、实验结果

### RNN 实验

- 在多个 RNN 任务上，LN 显著优于 BN 和无归一化基线
- LN 使训练更稳定、收敛更快，尤其在长序列上优势明显
- BN 在 RNN 上有时甚至比无归一化更差（小 batch 导致统计量不稳定）

### 注意力模型

论文还将 LN 应用于一个简单的注意力模型，同样观察到训练稳定性的提升。

### 其他任务

在各种序列任务上，LN 一致地加速收敛并提升最终性能。

## 四、影响

LayerNorm 的影响远超论文最初设想的 RNN 场景：

1. **Transformer 的标配**：原始 Transformer (Vaswani et al., 2017) 就使用 LayerNorm，这是 LN 走向主流的转折点
2. **所有现代 LLM 都用 LN 或其变体**：GPT 系列、BERT、LLaMA、PaLM 等
3. **Pre-LN vs Post-LN**：原始 Transformer 用 Post-LN（子层输出后归一化），后来发现 Pre-LN（归一化后输入子层）训练更稳定，成为主流
4. **Vision Transformer (ViT)** 将 LN 的适用范围从 NLP 扩展到 CV

## 五、后续发展：RMSNorm

2019 年，Zhang & Sennrich 提出 Root Mean Square Layer Normalization (RMSNorm)，对 LN 做了两个简化：

1. **去掉 mean subtraction**：不计算均值，直接用 RMS 归一化

$$\text{RMS}(x) = \sqrt{\frac{1}{H} \sum_{i=1}^{H} x_i^2 + \epsilon}, \quad \hat{x}_i = \frac{x_i}{\text{RMS}(x)}$$

2. **去掉偏移参数 $\beta$**：只保留缩放参数 $\gamma$

$$y_i = \gamma_i \cdot \hat{x}_i$$

RMSNorm 的动机：LN 的 re-centering（减均值 + 加 $\beta$）对表达力的影响不大，真正起作用的是 re-scaling（除标准差 + 乘 $\gamma$）。去掉均值减法节省约 7-64% 的计算开销，效果几乎无损。

LLaMA、Mistral、DeepSeek-V2 等现代 LLM 已全面转向 RMSNorm。

## 六、个人评价

LayerNorm 的核心贡献不是技术复杂度——公式极其简单——而是**选择了一个正确的归一化方向**。沿特征维度归一化这个看似微小的改动，解决了 BN 在序列模型和小 batch 场景的根本缺陷，并最终成为 Transformer 的标配。

几个有意思的观察：

1. **论文最初是为 RNN 设计的**，但 LN 真正大放异彩是在 Transformer 中。这说明好的设计往往超越其原始应用场景
2. **LN 在 CNN 上效果不好**这个发现，暗示了归一化方法与数据结构（空间局部性 vs 序列/集合结构）的深层关系
3. **从 LN → RMSNorm 的简化**是一个有趣的趋势：从复杂到简单，去掉不必要的组件。这与 SwiGLU 取代 GELU、RoPE 取代学习位置编码的趋势类似——现代 LLM 偏好更简洁但同样有效的设计
4. **Pre-LN 的胜利**说明归一化的位置（before vs after sublayer）可能比归一化方法本身更重要。Pre-LN 使得残差路径上始终有一个干净的恒等映射，梯度可以直接回传，这对深层网络的训练至关重要
