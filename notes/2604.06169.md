# In-Place Test-Time Training

2604.06169 | Guhao Feng, Shengjie Luo 等 | ByteDance Seed / PKU | ICLR 2026 Oral

---

TTT 这个方向我一直觉得挺有想象力的——推理的时候不是死板地用固定权重，而是让一小部分参数跟着输入流动态更新，相当于给模型一个"在线记忆"。但之前的做法一直没法用到真实的大模型上，这篇算是把这条路给打通了。

## 做了什么

核心就一句话：把 Transformer 里 MLP 的 down-projection 矩阵在推理时原地更新，当成 fast weight 用。不换 attention，不加新模块，拿过来就能用。

这为什么说得通？MLP 本来就是 key-value memory，存的是预训练学到的通用知识（慢权重）。让同一个 W_down 在推理时也吸收当前上下文的信息（快权重），角色上不冲突。W_gate 和 W_up 冻住不动，只动 W_down。

更新公式特别简单，用负内积做 loss，一步梯度下降就是：

W_down += η * V^T * Z

就是一个 outer product 累加，所以天然能做 parallel scan，也兼容 context parallelism。

因为 TTT 不再充当 token mixer（那是 attention 的事），chunk 可以开很大，消融显示 512-1024 最优。之前的 TTT 方法被迫用小 chunk 因为它要替代 attention 做细粒度混合，chunk 一大信息粒度就粗了，但现在不需要扛这个活。

## 目标函数的事

这是论文我觉得最有意思的地方。之前 TTT 的 self-supervised 目标是 reconstruction：让 fast weight 记住当前 token 的表示。但这对语言模型来说不太对——你存了 "the cat sat"，查询的时候你想知道的是下一个词是什么，而不是把 "the cat sat" 再复述一遍。

所以他们把 target 换成了下一个 token 的 embedding，用 Conv1D + 可学习投影来实现。Conv1D 引入未来信息，W_target 做投影变换，两者缺一不可。

理论部分在 induction head 的设定下给了个干净的证明：用 NTP target，正确 token 的 logit 会显著提升，其他 token 基本不动；用 reconstruction target，正确 token 的 logit 变化可以忽略。假设也合理——embedding 近似正交，key-query 有对齐——跟 Bietti 等人分析 induction head 的框架是一路的。

## 实验数字

Drop-in 增强 Qwen3-4B，继续训练 35B tokens 后在 RULER 上测：

4k 和 8k 差不多，甚至 4k 还略降了一点。但从 64k 开始拉开——74.3 到 78.7，128k 从 74.8 到 77.0。256k 外推也有提升。趋势很清楚：上下文越长，In-Place TTT 的优势越大。

LLaMA-3.1-8B 和 Qwen3-14B 也试了，64k 上分别 +2.1 和 +2.7，说明不是 Qwen 系的特例。

从头训的实验也做了，500M 到 4B 都有，ppl 一直比 GLA、DeltaNet、LaCT 低。4B 模型在 common sense reasoning 上基本不掉分，RULER 提升很大（Full Attn 的 RULER-16k 从 6.58 到 19.99）。

消融里几个发现：TTT 层越多效果越好，单调递增；chunk size 512 和 1024 最好；Conv1D 对长上下文关键，W_target 对短上下文关键；throughput 和 memory 开销很小。

## 我的看法

这篇文章的品味不错。In-Place 这个想法本身就很漂亮——不引入新模块，复用已有的 MLP，这在工程上的意义太大了。你有一个训好的 70B 模型，不用从头训，加上 In-Place TTT 做少量继续训练就能增强长上下文能力，这个卖点很实在。理论部分也不凑数，Theorem 1 确实解释了为什么 NTP target 比 reconstruction 好，不是那种"为了有理论而写理论"的感觉。

不足的地方也有。35B tokens 的继续训练成本不算低，有没有更轻量的路？W_down 推理时不断更新，偏离原始值多远？会不会跑飞？这些论文没讨论。还有，和纯 LoRA 对比缺失——如果只对 W_down 做 LoRA fine-tune 能拿到多少提升？这能帮我们区分"动态在线更新"和"单纯加可训练参数"各自的贡献。RAG 的对比也没有，长上下文场景下把信息压缩到参数里 vs 检索外部知识，各有各的 trade-off，值得聊。

跨模态的迁移也没验证。TTT 在视觉、视频上本来也有应用，MLP 复用的思路能不能搬过去？

和同期工作比：TTT-Linear、Titans、LaCT 都要改架构从头训，In-Place TTT 是第一个真正做到 drop-in 的。和 linear attention / SSM 是互补关系，可以叠加。M²RNN 也用 matrix state 做长上下文，思路有交叉，但 In-Place TTT 的"复用已有 MLP"在工程上更实际。

总的来说，这篇把 TTT 从研究玩具往实用工具推了一大步。drop-in + NTP 目标 + 大 chunk，三个设计互相成就，逻辑自洽。ICLR 给 Oral 合理。
