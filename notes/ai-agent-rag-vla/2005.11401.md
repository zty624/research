# Retrieval-Augmented Generation for Knowledge-Intensive NLP Tasks (Lewis et al., 2020)

**arxiv**: 2005.11401 | **RAG 开山之作** | NeurIPS 2020

---

## 一、核心动机

预训练语言模型（如 BERT、T5、BART）将知识存储在参数中，在下游任务上微调后效果很好，但存在根本问题：

1. **知识访问不精确**：无法精确地检索和操作存储在参数中的事实知识
2. **幻觉 (Hallucination)**：生成看似合理但事实上错误的内容
3. **知识更新困难**：世界知识变化时，需要重新训练才能更新
4. **缺乏可解释性**：无法提供决策的来源/出处 (provenance)

之前的混合模型（REALM、ORQA）将检索器与 MLM 结合，但只探索了抽取式问答。本文的核心洞察：**将检索增强推广到通用的 seq2seq 生成框架**，让模型既利用参数记忆（BART）又利用非参数记忆（Wikipedia 索引），两者端到端联合训练。

## 二、方法

### 2.1 整体架构

RAG = **检索器 (Retriever)** + **生成器 (Generator)**

- **参数记忆 (Parametric Memory)**：BART-large（400M 参数的 seq2seq Transformer）
- **非参数记忆 (Non-parametric Memory)**：Wikipedia 的密集向量索引（21M 个 100 词的文档块）

对输入 $x$，检索 top-K 文档 $z$，然后生成器基于 $(x, z)$ 生成输出 $y$。将检索到的文档视为**潜变量**，通过边际化得到最终预测。

### 2.2 两种 RAG 变体

**RAG-Sequence**：整个序列使用同一篇文档

$$p_{\text{RAG-Sequence}}(y|x) \approx \sum_{z \in \text{top-k}} p_\eta(z|x) \prod_{i=1}^{N} p_\theta(y_i | x, z, y_{1:i-1})$$

- 对每篇文档独立做 beam search，然后边际化各 beam 的概率
- 解码需要 "Thorough Decoding"（对所有文档重算未出现在某 beam 中的假设）或 "Fast Decoding"（近似忽略）

**RAG-Token**：每个 token 可以基于不同文档

$$p_{\text{RAG-Token}}(y|x) \approx \prod_{i=1}^{N} \sum_{z \in \text{top-k}} p_\eta(z|x) p_\theta(y_i | x, z, y_{1:i-1})$$

- 先边际化得到每个 token 的分布，再做标准自回归 beam search
- 更灵活，可从多篇文章中整合信息

### 2.3 检索器：DPR

$$p_\eta(z|x) \propto \exp(d(z)^\top q(x))$$

- $d(z) = \text{BERT}_d(z)$：文档编码器（BERT_BASE）
- $q(x) = \text{BERT}_q(x)$：查询编码器（BERT_BASE）
- top-K 检索 = MIPS（最大内积搜索），用 FAISS 的 HNSW 近似索引，亚线性时间

### 2.4 生成器：BART

- BART-large，400M 参数
- 输入：查询 $x$ 与检索文档 $z$ 拼接
- 输出：目标序列 $y$

### 2.5 训练

- 联合训练检索器（query encoder）和生成器，**不使用文档级别的检索监督**
- 冻结文档编码器 $BERT_d$（避免频繁重建索引），只微调 $BERT_q$ 和 BART
- 优化目标：负边际对数似然，Adam 优化器
- 训练时 $k \in \{5, 10\}$，测试时 $k$ 通过验证集确定

## 三、实验结果

### 3.1 开放域问答

| 模型 | NQ | TQA | WQ | CT |
|------|-----|-----|-----|-----|
| T5-11B (Closed-Book) | 34.5 | - | 37.4 | - |
| T5-11B+SSM | 36.6 | - | 44.7 | - |
| REALM | 40.4 | - | - | - |
| DPR | - | 41.5 | 57.9 | - |
| **RAG-Token** | **44.5** | 55.2 | 66.1 | 50.0 |
| **RAG-Sequence** | 44.5 | **56.8** | **68.0** | **52.2** |

- RAG 在所有四个开放域 QA 任务上达到 SOTA
- 不需要 REALM/T5+SSM 那样昂贵的 salient span masking 预训练
- 即使正确答案不在任何检索文档中，RAG 仍能达到 11.8% 准确率（抽取式模型为 0%）

### 3.2 抽象式问答（MS-MARCO NLG）

- RAG-Sequence 比 BART 高 2.6 BLEU + 2.6 Rouge-L
- 接近 SOTA（SOTA 使用了 gold passages，RAG 没有）

### 3.3 Jeopardy 问题生成

人工评估（452 对比较）：
- RAG 更事实准确：42.7% vs BART 仅 7.1%
- RAG 更具体：37.4% vs BART 仅 16.8%
- 两者都正确：17%
- 生成多样性（distinct trigram ratio）：RAG-Sequence 83.5% > RAG-Token 77.8% > BART 70.7%

关键案例：生成 Hemingway 相关 Jeopardy 问题。RAG-Token 在生成 "A Farewell to Arms" 时后验集中于文档 1，生成 "The Sun Also Rises" 时集中于文档 2。这展示了**参数记忆和非参数记忆的协作**——非参数检索引导生成方向，参数记忆补全具体内容。

### 3.4 事实验证（FEVER）

- 3 分类：RAG 达到 89.5% 准确率，距 SOTA 仅 4.3%（SOTA 使用检索监督的流水线系统）
- 2 分类：90.6%，距使用 gold evidence 的 RoBERTa 仅 2.7%
- RAG 检索到的 top-1 文档 71% 来自 gold 文章，top-10 中 90% 包含 gold 文章

### 3.5 消融实验

1. **检索学习很重要**：冻结检索器（Frozen）性能下降显著（NQ 上 44.0 → 41.2）
2. **密集检索 vs BM25**：
   - FEVER 上 BM25 略好（实体密集型查询适合词重叠匹配）
   - QA 任务上密集检索显著优于 BM25
3. **索引热替换 (Hot-Swapping)**：
   - 2016 年索引回答 2016 年世界领导人：70% 正确
   - 2018 年索引回答 2018 年世界领导人：68% 正确
   - 时间错配：12% 和 4% 正确
   - **无需重新训练即可更新知识**——这是 RAG 相对纯参数模型的核心优势

4. **检索文档数**：RAG-Sequence 性能随 K 单调提升；RAG-Token 在 K=10 附近达峰

## 四、个人评价

**贡献**：
1. **RAG 范式的确立**：将检索增强从特定任务（抽取式 QA）推广到通用 seq2seq 框架，定义了 RAG 这一全新范式
2. **参数记忆 + 非参数记忆的优雅结合**：BART 的语言能力 + Wikipedia 的事实知识，1+1 > 2
3. **端到端可微**：检索器和生成器联合训练，不需要文档级检索监督
4. **知识可更新**：索引热替换实验证明，换索引即可更新世界知识
5. **生成多样性 + 事实性**：相比纯参数模型，RAG 生成的文本更具体、更多样、更少幻觉

**局限**：
1. **检索粒度**：固定 100 词的文档块可能太短或太长，无法自适应
2. **检索是离散的**：top-K 检索是不可微操作，梯度无法反传到文档编码器（冻结了 $BERT_d$）
3. **单次检索**：只能检索一次，无法根据已生成内容动态追加检索（后续 ReAct/IRCoT 解决）
4. **延迟**：检索+生成的两阶段流程比纯参数模型慢
5. **Wikipedia 局限性**：只用了 Wikipedia，无法处理需要专业领域知识或实时信息的场景

**后续影响**：
RAG 已成为大语言模型的标准增强方式。2023 年后，几乎所有 LLM 应用都采用了某种形式的 RAG。关键发展包括：
- **主动检索**（FLARE、Self-RAG）：模型自己决定何时检索
- **迭代检索**（IRCoT、ReAct）：多步推理 + 多次检索
- **混合检索**（BM25 + dense + reranking）
- **长上下文 vs RAG**：随着 LLM 上下文窗口增长（128K+），RAG 的必要性在部分场景被重新审视，但成本效率和知识更新优势仍在

RAG 的核心洞见——**将知识的存储和推理分离**——比具体技术方案更持久，即使具体实现方式可能随技术演进而变化。
