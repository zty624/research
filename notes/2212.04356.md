# Whisper: Robust Speech Recognition via Large-Scale Weak Supervision (Radford et al., 2022)

**arxiv**: 2212.04356 | **OpenAI** | arxiv 2022 | 开山

---

## 一、核心问题

语音识别领域存在一个深刻的矛盾：在标准基准（如 LibriSpeech）上，模型性能已超过"人类水平"（WER 1.4% vs 人类 5.8%），但在真实场景中表现远逊于人类。根因在于：模型在训练分布上的"超人"表现是过拟合数据集特定模式的结果，而非真正的泛化能力。

之前的方法路线（Wav2Vec 2.0 等）虽然用无监督预训练大幅提升了音频编码器质量，但缺乏对应的高质量预训练解码器，仍需针对每个数据集微调——这既复杂又限制了鲁棒性。已有工作（SpeechStew）尝试混合监督数据，但仅有 5,140 小时，远小于无监督方法使用的 1,000,000 小时。

Whisper 的目标：通过**大规模弱监督预训练**，构建一个**开箱即用**的语音处理系统，在零样本设置下跨领域、跨任务、跨语言可靠工作。

## 二、方法

### 数据集构建

从互联网上收集音频-转录本配对数据，经过多层过滤得到 **680,000 小时**标注音频：
- 438,000 小时英语语音识别
- 117,000 小时覆盖 96 种语言的非英语语音
- 125,000 小时 X→en 语音翻译

关键过滤措施：
1. **去除机器生成转录**：检测全大写/全小写、缺乏标点等 ASR 输出特征
2. **语言一致性检查**：用音频语言检测器（VoxLingua107 微调）与 CLD2 文本语言检测比对
3. **模糊去重**：减少重复和自动生成内容
4. **迭代过滤**：训练初始模型后，按错误率和数据源大小排序手动审查，去除部分转录、对齐错误的数据

### 模型架构

采用标准的 **Encoder-Decoder Transformer**，避免架构创新混淆对规模化效果的评估：
- 输入：80 通道 log-Mel 频谱图（25ms 窗口，10ms 步长）
- Encoder stem：2 层卷积（第 2 层 stride=2）+ 正弦位置编码
- Decoder：学习位置编码 + 输入输出权重共享
- 模型家族：Tiny(39M) → Base(74M) → Small(244M) → Medium(769M) → Large(1550M)

### 多任务格式

用特殊 token 序列统一指定任务，使单一模型替代整个语音处理管线：

```
<|startoftranscript|> <language_tag> <transcribe|translate> <notimestamps> ... <|endoftranscript|>
```

支持的任务：
- 语音识别（多语言）
- X→en 语音翻译
- 语言识别
- 语音活动检测（VAD）
- 时间戳预测（量化到 20ms）

训练时以一定概率将前文文本加入 decoder 上下文，鼓励模型利用更长文本上下文消歧。

### 训练细节

- AdamW + 梯度裁剪 + 线性学习率衰减
- Batch size 256，训练 2^20 步（约 2-3 个 epoch）
- FP16 + 动态 loss scaling + 激活检查点
- 不使用数据增强或正则化（数据多样性足够）
- Large V2 额外训练 2.5x epoch，加入 SpecAugment、Stochastic Depth、BPE Dropout

### 长音频转录策略

模型只接受 30 秒输入，长音频通过滑动窗口处理。关键技术：
1. Beam search（5 beams）减少重复循环
2. 温度调度：平均 log 概率 < -1 或 gzip 压缩率 > 2.4 时提高温度 0.2（最高 1.0）
3. 低温度时使用前文文本作为上下文
4. 时间戳约束避免幻觉

## 三、实验结果

### 英文语音识别与鲁棒性

| 数据集 | wav2vec 2.0 Large | Whisper Large V2 | 相对错误降低 |
|--------|-------------------|------------------|-------------|
| LibriSpeech clean | 2.7 | 2.7 | 0% |
| Common Voice | 29.9 | 9.0 | 69.9% |
| CHiME-6 | 65.8 | 25.5 | 61.2% |
| Switchboard | 28.3 | 13.8 | 51.2% |
| **平均** | **29.3** | **12.8** | **55.2%** |

核心发现：在 LibriSpeech 上两者性能相同，但 Whisper 在其他数据集上错误率降低 55.2%。这说明**零样本鲁棒性**与**同分布精度**是两种不同能力。

### 接近人类水平

在 25 段 Kincaid46 录音上：
- 人类转录 WER：约 5-6%
- 计算机辅助人类转录 WER：比 Whisper 低 1.15 个百分点
- Whisper 与纯人类转录差距仅零点几个百分点

### 多语言语音识别

- 在 MLS 上 zero-shot 超越 XLS-R(1B) 和 mSLAM-CTC(2B)（7.3 vs 9.7 WER）
- 在 VoxPopuli 上落后于监督微调方法（13.6 vs 8.1）
- 训练数据量与 zero-shot 性能高度相关（$r^2 = 0.83$），WER 每增加 16 倍数据减半

### 语音翻译（X→en, CoVoST2）

| 分组 | mSLAM-CTC(2B) | Whisper |
|------|---------------|---------|
| 高资源 | 37.8 | 36.2 |
| 中资源 | 29.6 | 32.6 |
| 低资源 | 18.5 | 25.2 |
| **总体** | **24.8** | **29.1** |

Whisper 在低资源语言上优势显著（6.7 BLEU 提升），得益于 68,000 小时翻译预训练数据。

### 噪声鲁棒性

在加性白噪声和 Pub 噪声（餐厅环境）测试中，LibriSpeech 训练的模型在 SNR < 10 dB 时性能急剧下降，而 Whisper 在高噪声下仍保持显著优势。

### 长音频转录

在 7 个长音频数据集上，Whisper 优于所有开源模型（NVIDIA STT），并在大多数数据集上超过 4 个商业 ASR 服务。

### 规模化分析

- **模型规模**：除英文 ASR 外（接近人类水平饱和），所有任务随模型增大持续改善
- **数据规模**：英文 ASR 在 3K→13K 小时快速提升后减缓；多语言 ASR 和翻译则持续受益于更大数据集
- **多任务多语言迁移**：小模型存在负迁移，但大模型中多任务多语言训练超越单任务训练——正迁移随规模涌现

## 四、个人评价

Whisper 是"简单方法 + 大规模数据 = 鲁棒系统"的经典范例。它没有架构创新，用的是标准 Transformer；没有训练技巧，甚至不做数据增强。它的全部贡献在于：证明了弱监督规模化在语音领域可以替代精标注 + 微调范式。

影响：
1. **定义了语音基础模型范式**：Whisper 是语音领域的 GPT-3 时刻——一个预训练模型零样本覆盖多种任务，无需微调即可部署
2. **零样本评估的重要性**：论文最有洞见的发现是同分布性能与鲁棒性的脱节，这对整个 ML 评估方法论有启示
3. **广泛的应用生态**：Whisper 开源后成为语音转写的默认工具，催生了 WhisperX、faster-whisper、whisper.cpp 等生态
4. **多任务格式设计**：用特殊 token 统一多任务的思想影响了后续语音大模型（如 SpeechGPT、AudioPaLM）

局限：
- 非英语语言性能仍有较大差距，低资源语言 WER 很高
- 语言识别能力不如专门的监督模型
- 30 秒窗口限制和长音频幻觉问题需要额外工程处理
- 弱监督数据中仍有机器生成转录残留（如 Welsh 翻译数据的语言误分类问题）
- 论文未充分讨论数据偏见和公平性问题
