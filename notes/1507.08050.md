**arxiv**: 1507.08050 | **关键词**: 概率编程, 贝叶斯推断, MCMC, NUTS, PyMC3, Theano | University of Oxford, D.E. Shaw Group, Vanderbilt University | JSS 2015

## 一、核心动机

贝叶斯统计方法提供了建模不确定性的理论框架，但实际使用面临三大障碍：

1. **推断困难**：后验分布通常没有解析解，需要数值方法（MCMC、变分推断），而传统采样器（如 Metropolis-Hastings）在高维空间中效率极低——随机游走行为导致收敛慢、自相关高
2. **编程门槛高**：定义贝叶斯模型需要大量手工推导和底层代码，统计学家更希望用接近数学符号的方式表达模型
3. **缺乏统一工具**：2015 年前，WinBUGS/JAGS 使用自己的 DSL（不易扩展），Stan 刚起步（C++ 底层、R/Python 绑定有限），Python 生态中缺乏成熟的概率编程框架

PyMC3 的目标：让贝叶斯建模像写数学公式一样自然，同时利用现代采样算法（HMC/NUTS）和自动微分实现高效推断。

## 二、方法

### 2.1 模型定义语法

采用 Python 上下文管理器 `with Model() as model:` 语法，随机变量作为一等对象直接创建：

```python
with Model() as basic_model:
    alpha = Normal('alpha', mu=0, sd=10)
    beta = Normal('beta', mu=0, sd=10, shape=2)
    sigma = HalfNormal('sigma', sd=1)
    mu = alpha + beta[0]*X1 + beta[1]*X2
    Y_obs = Normal('Y_obs', mu=mu, sd=sigma, observed=Y)
```

设计要点：
- **上下文管理器**自动将随机变量加入模型，无需手动管理图结构
- **observed 参数**区分观测变量与潜变量，同一框架统一处理
- 代码几乎逐行对应统计模型定义，极大降低认知负担

### 2.2 Theano 后端与自动微分

PyMC3 的核心工程决策是选择 Theano 作为计算后端：

- **自动微分**：HMC/NUTS 需要对数后验密度对参数的梯度，Theano 自动计算，用户无需手写梯度
- **计算图优化**：Theano 将模型透明转译为 C 代码并编译为机器码，支持 GPU 加速
- **NumPy 兼容**：Theano 的 tensor 与 NumPy ndarray 紧密集成，支持广播和高级索引

这是实现 NUTS 的关键前提——没有自动微分，HMC 类采样器在实际模型中几乎不可用。

### 2.3 采样算法

#### NUTS（No-U-Turn Sampler）

PyMC3 的核心采样器，是 HMC 的自调优变体：

- **HMC 原理**：引入辅助动量变量 $p$，通过模拟哈密顿动力学 $\frac{\partial z}{\partial t} = \nabla_p H, \frac{\partial p}{\partial t} = -\nabla_z H$ 在参数空间中高效移动，避免随机游走行为
- **NUTS 的改进**：HMC 需要手动设置轨迹长度（步数 × 步长），NUTS 通过检测轨迹的"U 形回转"自动停止，无需调参
- **自调优**：NUTS 在 warmup 阶段自动调整步长（dual averaging）和质量矩阵（对角或全矩阵），使接受率接近最优

论文指出 PyMC3、Stan 和 R 的 LaplacesDemon 是当时仅有的提供 HMC/NUTS 的概率编程包。

#### 其他采样器

- **Metropolis-Hastings**：通用但低效，适用于离散变量或梯度不可得的场景
- **Slice 采样**：另一种无需梯度的自适应方法

### 2.4 辅助功能

- **MAP 估计**：`find_MAP()` 返回最大后验估计，可作为 NUTS 的初始点和缩放矩阵参考
- **自定义分布**：`DensityDist` 接受任意 log-probability 函数；也可继承 `Continuous`/`Discrete` 类
- **自定义变换**：`@as_op` 装饰器将 Python 函数包装为 Theano 算子（但会失去自动微分）
- **GLM 子包**：R-like 公式语法（`y ~ x1 + x2`），通过 patsy 解析，一行代码定义广义线性模型
- **多种后端**：内存（默认）、文本文件、SQLite，适合大规模采样场景

## 三、实验结果

论文通过两个案例研究展示 PyMC3 的能力：

### 3.1 随机波动率模型

对 S&P 500 过去 400 天的日收益率建模：

$$\log(\sigma_t^2) \sim \text{Normal}(\log(\sigma_{t-1}^2), \tau^{-2})$$
$$r_t \sim \text{StudentT}(\nu, 0, \sigma_t^2)$$

- 400+ 参数的模型（每天一个隐波动率），NUTS 相比 Metropolis-Hastings 显著更高效
- NUTS 自动调参，无需手动设置步长和轨迹长度
- 展示了混合采样：NUTS 处理连续参数，Metropolis 处理离散参数（自由度 $\nu$）

### 3.2 煤矿灾害变点检测

$$\text{rate} = \text{switch}(t \geq \text{switchpoint}, \text{early\_rate}, \text{late\_rate})$$
$$\text{disasters}_t \sim \text{Poisson}(\text{rate})$$

- `switch` 函数实现变点：在某个年份前后灾害率不同
- 成功估计安全法规变化的年份（约 1891 年），与历史记录一致
- 混合使用 NUTS（连续参数）和 Metropolis（离散变点参数）

## 四、个人评价

### 核心贡献

1. **NUTS 的普及**：将当时最先进的 MCMC 算法带给 Python 用户，大幅降低了贝叶斯推断的实践门槛。NUTS 在高维问题上的效率远超 Metropolis-Hastings，是 PyMC3 相比 PyMC2 的决定性优势
2. **模型即代码的设计哲学**：上下文管理器语法让模型定义代码接近统计符号，这种设计影响了后来的概率编程框架（如 NumPyro）
3. **自动微分 + 概率编程的结合**：Theano 后端使 HMC/NUTS 可用于任意用户定义的模型，无需手写梯度

### 局限

1. **Theano 依赖**：Theano 已于 2017 年停止维护，PyMC v4+ 迁移至 PyTensor/JAX 后端，此论文描述的架构已过时
2. **静态图限制**：Theano 的静态计算图模型不如 PyTorch 的动态图灵活，调试也更困难
3. **变分推断缺失**：论文仅讨论 MCMC 方法，未提及变分推断（ADVI 后来在 PyMC3 中加入）
4. **可扩展性**：对于超大规模数据（如百万级观测），MCMC 的每个 step 需要遍历全数据，不如变分推断高效

### 影响

作为工具论文，理论贡献有限，但工程价值很高。PyMC3 让 Python 生态中的贝叶斯建模从"需要自己写采样器"变为"几行代码即可"，对概率编程的普及起了重要作用。其设计理念（上下文管理器语法 + 自动微分后端 + NUTS）成为后来框架的标配。
