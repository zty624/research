## 一、理论推导

### 1. VQGAN 架构

VQGAN 在 VQ-VAE 基础上增加两个关键改进：**感知损失**和**对抗训练**。

编码器 $E$ 将图像 $x \in \mathbb{R}^{H \times W \times 3}$ 映射到隐空间 $\hat{z} = E(x) \in \mathbb{R}^{h \times w \times n_z}$，然后量化：

$$z_q = q(\hat{z}) := \arg\min_{z_k \in \mathcal{Z}} \|\hat{z}_{ij} - z_k\|$$

解码器重建：$\hat{x} = G(z_q) = G(q(E(x)))$

### 2. VQGAN 损失函数

原始 VQ-VAE 用 L2 重建损失，VQGAN 改为**感知损失 + GAN 损失**：

$$\mathcal{L}_{GAN}(\{E,G,\mathcal{Z}\}, D) = \log D(x) + \log(1 - D(\hat{x}))$$

总目标：

$$Q^* = \arg\min_{E,G,\mathcal{Z}} \max_D \mathbb{E}_{x \sim p(x)} \left[\mathcal{L}_{VQ}(E,G,\mathcal{Z}) + \lambda \mathcal{L}_{GAN}(\{E,G,\mathcal{Z}\}, D)\right]$$

其中 $\mathcal{L}_{VQ}$ 与 VQ-VAE 相同（重建 + VQ损失 + 承诺损失），但 $\mathcal{L}_{rec}$ 改为感知损失（LPIPS）。

自适应权重 $\lambda$：

$$\lambda = \frac{\|\nabla_{G^L}[\mathcal{L}_{rec}]\|}{\|\nabla_{G^L}[\mathcal{L}_{GAN}]\| + \delta}$$

其中 $G^L$ 是解码器最后一层，$\delta = 10^{-6}$。这个设计平衡了重建梯度和对抗梯度的量级。

### 3. Transformer 先验

图像编码为离散序列 $s \in \{0, \ldots, |\mathcal{Z}|-1\}^{h \times w}$，用自回归 Transformer 学习：

$$p(s) = \prod_i p(s_i | s_{<i})$$

最大化 log-likelihood：$\mathcal{L}_{Transformer} = \mathbb{E}_{x \sim p(x)}[-\log p(s)]$

条件生成：$p(s|c) = \prod_i p(s_i | s_{<i}, c)$，将条件信息 $c$ 的编码序列 prepend 到 $s$ 前面。

### 4. 高分辨率生成：滑动窗口

对于百万像素图像，序列长度超过 Transformer 可行范围。使用滑动窗口策略：
- 训练时裁剪小 patch
- 采样时逐步滑动生成

VQGAN 确保局部上下文足以重建图像。

### 5. Patch Discriminator

使用 patch-based discriminator 而非全图判别器，关注局部真实性。这对高压缩率下的感知质量至关重要。

## 二、最小实验复现资源

| 项目 | 要求 |
|------|------|
| **数据集** | ImageNet 256×256, FFHQ 256×256, LSUN Churches, COCO-Stuff |
| **VQGAN** | 编码器/解码器: m=4 下采样块, codebook |Z|=1024, patch discriminator |
| **Transformer** | GPT-2 medium (~307M), 序列 16×16=256 |
| **训练** | 两阶段: 先训练 VQGAN, 再训练 Transformer |
| **硬件** | 单 GPU 12GB VRAM (Transformer), 更多 GPU 更快 |
| **时间** | VQGAN ~1-2 天, Transformer ~2-5 天 (单 GPU) |
| **总估算** | **1-2 GPU, ~1 周** |

## 三、重要 Idea

1. **VQGAN = VQ-VAE + 感知损失 + 对抗训练**：核心改进。VQ-VAE 的 L2 重建导致模糊，VQGAN 用感知损失 (LPIPS) + Patch discriminator 确保高压缩率下仍有感知质量。这使得 Transformer 可以专注于全局语义而非低级纹理。

2. **CNN 归纳偏置 + Transformer 表达力的分工**：CNN (VQGAN) 负责学习局部视觉组成的 codebook，Transformer 负责建模全局组合关系。这种分工让两者各司其职 —— CNN 处理局部性，Transformer 处理长程依赖。

3. **自适应权重 λ**：平衡重建损失和 GAN 损失的梯度量级。这个简单技巧避免了手动调权重的麻烦。

4. **Context-rich codebook 的重要性**：实验证明下采样因子 f 越大（codebook 编码越多上下文），生成质量越好。f=16 的 VQGAN 远优于 f=1 的 k-means 聚类（iGPT 方案）。这反驳了"浅编码以保持空间不变性"的思路。

5. **Transformer 在隐空间优于 PixelSNAIL**：在相同 codebook 上，Transformer 的 NLL 一致优于卷积自回归模型 PixelSNAIL，验证了 Transformer 在隐空间建模的优势。

## 评价

VQGAN 是从 VQ-VAE 到现代视觉生成模型的关键桥梁。它直接启发了 DALL-E、Stable Diffusion 的 VAE 设计。核心贡献是证明了：高质量的离散 tokenizer 需要感知级别的损失，而不仅仅是像素级 L2。这让 Transformer 在视觉生成中变得可行。局限：两阶段训练仍非最优；百万像素生成需要滑动窗口，不够优雅；FID 不如当时的 StyleGAN2。但作为通用框架，其影响远超其定量结果。
