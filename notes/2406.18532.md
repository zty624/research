# Symbolic Learning Enables Self-Evolving Agents (Zhou et al., 2024)

**arxiv**: 2406.18532 | **关键词**: 语言智能体, 符号学习, 自进化, 自动优化, 提示优化 | AIWaves Inc., Zhejiang University | 2024

---

## 一、核心问题

当前语言智能体（language agents）的研究是**以模型为中心或以工程为中心**的——智能体的提示词（prompts）、工具（tools）和流水线（pipeline）需要大量人工工程来设计和优化，而非从数据中自动学习。

核心矛盾：
1. 现有优化方法（如 DSPy、GPTSwarm）**仅优化单独模块**（如单个提示词），容易陷入局部最优
2. 搜索方法需要**数值可编码的评估指标**，无法处理开放性任务（如创意写作、软件开发）
3. 缺乏一种能**联合优化**智能体所有符号组件（提示词、工具、流水线）的系统性框架

本文提出：将语言智能体视为**符号网络**，借鉴连接主义学习（神经网络训练）的反向传播和梯度下降思想，在自然语言空间中实现端到端优化。

## 二、方法

### 2.1 类比建立

| 神经网络 | 语言智能体 |
|----------|-----------|
| 计算图 | 智能体流水线（Agent Pipeline $\mathcal{A}$） |
| 层 | 节点（Node $N$） |
| 可学习权重 | 提示词（$P_n$）和工具（$T_n$） |
| 计算图/激活值 | 轨迹（Trajectory $\tau$） |
| 数值损失 | 语言损失（Language Loss $\mathcal{L}_\text{lang}$） |
| 数值梯度 | 语言梯度（Language Gradient $\nabla_\text{lang}$） |
| 梯度下降优化器 | 符号优化器（Symbolic Optimizer） |

### 2.2 Agent Symbolic Learning 流程

**Step 1: Forward Pass（前向传播）**
$$O_n = N(I_n, P_n, T_n)$$
智能体按流水线执行，存储每个节点的输入、输出、提示词、工具使用到轨迹 $\tau$ 中。

**Step 2: Language Loss Computation（语言损失计算）**
$$\mathcal{L}_\text{lang} = \text{LLM}(P_\text{loss}(\tau))$$
将轨迹输入 LLM，使用精心设计的提示模板 $P_\text{loss}$ 评估智能体表现，输出包含自然语言评语和数值分数。

$P_\text{loss}$ 组成：任务描述 + 输入 + 轨迹 + few-shot 示例 + 原则 + 输出格式控制

支持两种模式：
- **有监督智能体学习**：提供 ground truth 标签
- **无监督智能体学习**：仅根据任务描述评估，无需标准答案，实现自进化

**Step 3: Back-propagation of Language Gradients（语言梯度反向传播）**
$$\nabla_n^\text{lang} = \text{LLM}(P_\text{gradient}(\nabla_{n+1}^\text{lang}, I_n, O_n, P_n, T_n, \mathcal{L}_\text{lang}))$$
从最后一个节点到第一个节点逐层反向传播。每个节点不仅考虑自身子目标，还要考虑对后续节点的影响。通过链式传递，所有节点的语言梯度都与整体成功相关，有效避免局部最优。

**Step 4: Language Gradient-based Update（符号优化器更新）**

三类符号优化器：
- **PromptOptimizer**：将提示词拆分为任务描述、few-shot 示例、原则、输出格式控制等组件，分别优化
- **ToolOptimizer**：决定工具操作类型——编辑（修改工具描述）、删除、创建新工具
- **PipelineOptimizer**：使用原子操作（添加、删除、移动节点）更新流水线结构

优化器引入**"学习率"**概念：控制 LLM 修改提示词/工具/流水线时的激进程度。

**批处理训练**：类似 mini-batch SGD，对多个样本分别计算语言梯度后，批量提供给 LLM 进行综合考虑后更新。

**安全机制**：
- 非法更新重试（最多 3 次）
- 性能回退策略：优化后重新运行，若语言损失下降则回滚

## 三、实验结果

### 标准 LLM 基准
| 方法 | HotPotQA (F1/EM) | MATH (Acc) | HumanEval (Pass@1) |
|------|-------------------|------------|---------------------|
| GPTs (GPT-3.5/GPT-4) | 24/38.8, 33/44.3 | 23.2, 53.1 | 59.2, 71.7 |
| Agents | 27/37.5, 39/49.8 | 23.8, 56.0 | 59.5, 85.0 |
| Agents w/ AutoPE | 29/39.8, 38/50.3 | 22.5, 57.2 | 63.5, 82.3 |
| DSPy | 35/43.9, 40/50.5 | 17.3, 48.4 | 66.7, 77.3 |
| **Ours** | **35/44.8, 41/54.0** | **38.8, 60.7** | **64.5, 85.8** |

在 MATH 上提升最为显著（GPT-4 backbone: 60.7% vs 56.0% baseline）。DSPy 和 AutoPE 不稳定，有时反而降低性能。

### 复杂任务
**软件开发**（1-4分评分）：
| 任务 | GPTs | Agents | Ours |
|------|------|--------|------|
| 平均分 | 1.6 | 2.4 | **3.8** |

**创意写作**（GPT-4 评分）：
| 方法 | GPT-3.5 | GPT-4 |
|------|---------|-------|
| GPTs | 4.0 | 6.0 |
| Agents | 4.2 | 6.0 |
| ToT | 3.8 | 6.8 |
| **Ours** | **6.9** | **7.4** |

在创意写作上甚至超过了精心设计的 Tree-of-Thought 方法。框架自动发现了"计划-写作-修订"流水线，与 MetaGPT 的标准操作流程类似。

### 分析发现
- **初始化影响**：简单初始化优于过度工程化的初始化，类比神经网络初始化的重要性
- **复杂任务更有效**：在开放性任务上的改进比标准基准更显著和稳定
- **Case Study**：框架成功将单节点写作智能体优化为多角色（writer + editor）协作流水线

## 四、个人评价

### 影响
- **范式转变**：从"工程驱动"到"数据驱动"的智能体开发，是语言智能体研究的重要方向
- **系统化优化**：首次提出联合优化智能体所有符号组件（提示词 + 工具 + 流水线）的框架，而非局部优化
- **自进化智能体**：无监督模式下，智能体可在部署后从经验中学习并自我更新，打开"自进化 AI"的研究方向
- **与神经网络训练的类比**：将成熟的深度学习概念（反向传播、梯度下降、学习率、mini-batch）迁移到符号优化空间，直觉清晰

### 局限
- **依赖 LLM 作为优化器**：整个优化过程依赖 GPT-4 等强 LLM，成本高昂且不可控
- **语言梯度的质量**：自然语言"梯度"不如数值梯度精确，优化方向可能不稳定
- **缺乏理论保证**：与神经网络不同，符号优化没有收敛性保证
- **评估不充分**：实验规模较小，每种任务仅 5 个软件开发测试用例，统计显著性有限
- **回滚策略的局限**：基于当前样本的回滚可能过拟合单一样本
- **与 fine-tuning 的关系**：仅优化符号组件，未涉及 LLM backbone 的微调，两者结合的效果未探索
