## 1. 理论推导

### VQ-VAE 的核心瓶颈

标准 VQ-VAE 将编码器输出 z ∈ R^{D×D×C_c} 通过通道降维到 C_c 维，再在码本 E = {e_i}_{i=1}^K ⊂ R^{K×C_c} 中做最近邻查找：

```
z_q(x) = e_k,  where k = argmin_{e_i ∈ E} ||z(x) - e_i||^2
```

VQ-VAE 面临两个矛盾：(1) 增大 C_c 可以保留更多信息，但高维码本容易导致码本坍塌(codebook collapse)，大量码向量成为死点(dead points)；(2) 增大码本大小 K 可以提升表示容量，但在高维空间中同样加剧码本坍塌。现有方法（如 LlamaGen 将 C_c 降到 8、Open-MAGVIT2 用 LFQ 将码嵌入维度降到 0）本质上是通过牺牲信息来换取码本利用率，造成严重信息损失。

### 多组量化 (Multi-group Quantization) 设计

MGVQ 的核心思路是**保留较大的潜空间维度 C_l**，同时将高维 token 拆分为多个低维 sub-token，各自在独立的 sub-codebook 上量化，从而同时实现：

1. 大的表示容量
2. 码本易于优化（避免坍塌）
3. 最小化信息损失

具体数学形式化如下：

编码器输出 z ∈ R^{D×D×C_l}，其中 C_l 较大（如 32）。对每个位置的特征 z(x) ∈ R^{C_l}，均匀拆分为 G 个 sub-token：

```
{z(x_i)}_{i=1}^G,  每个 sub-token 维度为 C_l / G
```

对应地，设置 G 个不共享的 sub-codebook，每个大小为 K：

```
E_i = {e_{j}^{i}}_{j=1}^K ⊂ R^{K × (C_l/G)}
```

每个 sub-token 在各自的 sub-codebook 中独立做最近邻查找：

```
k_i = argmin ||z(x_i) - e_{k_i}^i||^2,   e_{k_i}^i ∈ E_i
```

最终将所有量化后的 sub-token 拼接得到重建输入：

```
z_q(x) = concat(e_{k_1}^1, ..., e_{k_G}^G)
```

### 表示容量的指数级提升

标准 VQ-VAE 的表示容量为 K（码本大小）。MGVQ 的表示容量为：

```
Capacity = K^G
```

例如 G=4、K=8192 时，容量为 8192^4 = 2^52；G=8、K=2048 时，容量为 2048^8 = 2^88。相比标准方法的 2^14 ~ 2^18，提升了超过十亿倍。

### 为什么分组优于单一大码本

关键洞察在于：低维 sub-codebook 的优化远比高维大码本容易。论文通过可视化码本点分布（Fig. 3）表明：

- 高维 + 小码本 → 各向异性分布，利用率低
- 高维 + 大码本 → 某些方向被偏好，产生大量死点
- 低维 + 小码本 → 可被充分利用，无死点
- 低维 + 大码本 → 均匀分布，但可能有冗余

MGVQ 通过分组将"高维大码本"问题转化为多个"低维小码本"问题，每个 sub-codebook 维度低（如 8 或 4）、大小适中（如 8192 或 2048），优化难度大幅降低，同时组合起来的容量远超单一大码本。

### Nested Masking 训练策略

为防止各 sub-codebook 收敛到相同的表示，论文提出嵌套掩码策略：训练时随机采样保留的 sub-token 数量 M_keep ∈ {1, ..., G}，将后 G - M_keep 个 sub-token 掩码掉。这强制编码器以有序方式压缩信息（先粗后细），解码器也按粗到细的方式重建。M_keep=1 时只能恢复粗略纹理，M_keep=G 时恢复全部细节。

### 训练损失函数

```
L_total = λ_1 L_l2 + λ_2 L_Char + λ_3 L_Commit + λ_4 L_vq + λ_5 L_GAN + λ_6 L_Percept
```

其中 L_Char 为 Charbonnier loss（近似 L1），权重设置：λ_1=2, λ_2=1, λ_3=0.25, λ_4=1, λ_5=0.5, λ_6=1。采用 straight-through 梯度估计器。

## 2. 最小实验复现资源

- **数据集**: ImageNet 训练（256×256）；零样本评估额外使用 FFHQ、OpenImages、Segment Anything、Argoverse2、Mapillary Vistas、LAION-HQ
- **模型规模**: 三个版本 — 308M (32× 下采样), 210M (16× 下采样), 122M (8× 下采样)
- **训练设置**: 学习率 10^{-5}（常数）, weight decay 5×10^{-2}, AdamW 优化器 (β_1=0.9, β_2=0.95)
- **Nested masking 概率** (G=4): {0.1, 0.1, 0.1, 0.7}
- **评估基准**: ImageNet 50k val (256×256), 自建 UHDBench (2K 分辨率, 2293 张图), 以及 CelebA/DAVIS/TextOCR/VFHQ/Spring/ENeRF (512×512)
- **硬件/训练时间**: 论文未明确报告 GPU 型号和训练时长

## 3. 重要idea

1. **多组量化 (Multi-group Quantization)**: 核心创新。保留高维潜空间（避免信息损失），将 token 拆分为多组低维 sub-token，每组独立量化。这是"分治"思想在 VQ 中的应用——将一个困难的高维量化问题分解为多个简单的低维量化问题，每个 sub-codebook 易于优化且利用率高，而组合起来的表示容量 K^G 指数级增长。

2. **表示容量 vs 码本大小的解耦**: 论文明确区分了"码本大小 K"和"表示容量 K^G"，指出 VQ-VAE 性能瓶颈在于表示容量不足而非码本大小不足。增大码本大小在单码本设置下会加剧坍塌，而 MGVQ 通过分组实现了在不增大单个码本的前提下指数级扩充表示容量。

3. **Nested Masking 实现粗到细的层次化表示**: 借鉴 Matryoshka 表示学习的思想，训练时随机丢弃部分 sub-token，强制模型按重要性排序编码信息。这既防止了 sub-codebook 间的冗余，又赋予了模型渐进式解码的能力。

4. **VQ-VAE 首次在重建质量上超越 VAE**: MGVQ 在 ImageNet 256×256 上 rFID=0.49，显著优于 SD-VAE 的 0.91；PSNR=24.70 也远超 SD-VAE 的 22.65。这是 VQ-VAE 类方法首次在重建指标上全面超越连续 VAE tokenizer，证明离散化不一定意味着信息损失。

5. **零样本泛化性**: 构建了包含 512p 和 2K 分辨率的 8 个零样本基准测试，MGVQ 在所有数据集上 PSNR 均排名第一，包括超越 SD-VAE，验证了高维潜空间保留信息对泛化的重要性。
