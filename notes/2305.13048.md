# RWKV: Reinventing RNNs for the Transformer Era (Peng et al., 2023)

**arxiv**: 2305.13048 | **关键词**: 线性注意力, RNN, O(N) 复杂度, WKV 注意力, 时间衰减, 并行训练 | Bo Peng | EMNLP 2023

## 一、核心问题

Transformer 的自注意力机制具有 $O(N^2)$ 的计算复杂度和 $O(N)$ 的推理内存（KV cache），在长序列场景下成为严重瓶颈。经典 RNN 虽具有 $O(1)$ 的推理内存，但训练难以并行化且长期依赖建模能力不足。本文提出核心问题：

**能否设计一种架构，兼具 Transformer 的并行训练能力和 RNN 的高效推理特性？**

RWKV 给出了肯定答案——通过线性注意力变体实现 $O(N)$ 训练复杂度，同时以 RNN 模式实现 $O(1)$ 推理内存。

## 二、方法

### 2.1 整体架构

RWKV 由交替堆叠的时间混合（time-mixing）和通道混合（channel-mixing）块组成，类似 Transformer 的注意力层 + FFN 层。关键区别在于：

- 注意力被替换为 WKV 线性注意力
- 所有计算均为线性递推形式，可并行训练、递推推理

### 2.2 时间混合：WKV 注意力

核心创新是 WKV（Weighted Key-Value）注意力机制。传统自注意力：

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d}}\right)V$$

复杂度为 $O(N^2 d)$。RWKV 将其改造为线性递推形式：

$$wkv_t = \frac{\sum_{i=1}^{t-1} e^{-(t-1-i) \cdot w + k_i} \cdot v_i + e^{u + k_t} \cdot v_t}{\sum_{i=1}^{t-1} e^{-(t-1-i) \cdot w + k_i} + e^{u + k_t}}$$

其中：
- $w$ 是通道级可学习的时间衰减参数，控制历史信息的衰减速率
- $u$ 是补偿项，弥补当前 token 与自身交互时的位置偏差
- $k_i, v_i$ 分别为 key 和 value 向量

**关键数学变换**：衰减因子 $\mu = e^{-e^{w}}$，采用双指数变换确保衰减率始终在 $(0, 1)$ 之间。

### 2.3 通道级可学习衰减

RWKV 的衰减参数 $w$ 是通道级（channel-wise）可学习的：

$$w \in \mathbb{R}^{d}, \quad \mu_j = e^{-e^{w_j}}, \quad j = 1, \ldots, d$$

这意味着每个维度有独立的衰减速率：
- 某些维度衰减快（类似短期记忆，只关注最近 token）
- 某些维度衰减慢（类似长期记忆，保留远距离信息）

双指数变换 $e^{-e^{w}}$ 的妙处：
- 无论 $w$ 取何值，$\mu \in (0, 1)$，天然满足衰减率的有界性
- 梯度流动良好，无需额外约束

### 2.4 时间偏移（Time-shift）

RWKV 不使用位置编码，而是通过**时间偏移**注入位置信息：

- 当前 token 的输入 = 当前 token 的嵌入 + 前一个 token 的嵌入（线性插值）
- 对 Q, K, V 分别使用不同的混合比例 $\mu_r, \mu_k, \mu_v$

$$x'_t = \mu \cdot x_t + (1 - \mu) \cdot x_{t-1}$$

这种简单的偏移操作使得模型能区分相邻 token 的顺序，是一种隐式的相对位置编码。

### 2.5 通道混合

通道混合块使用平方 ReLU 激活：

$$\text{ChannelMix}(x_t) = \max(0, W_1 \cdot (\mu_w \cdot x_t + (1 - \mu_w) \cdot x_{t-1}))^2 \cdot W_2$$

时间偏移同样应用于通道混合，确保位置信息的一致性。

### 2.6 两种运行模式

**训练模式（并行）**：
- WKV 注意力可改写为前缀和形式，利用关联扫描（associative scan）实现 $O(N)$ 并行计算
- 整个序列一次性处理，无需逐步递推
- 类似 Transformer 的训练效率

**推理模式（RNN）**：
- 将 WKV 递推展开，每步只需维护一个固定大小的状态向量
- 内存消耗 $O(1)$（与序列长度无关），而非 Transformer 的 $O(N)$ KV cache
- 适合长序列生成和流式推理

RNN 递推形式：

$$a_t = e^{-w} \odot a_{t-1} + e^{k_t} \odot v_t$$

$$b_t = e^{-w} \odot b_{t-1} + e^{k_t}$$

$$\text{wkv}_t = \frac{a_{t-1} + e^{u+k_t} \odot v_t}{b_{t-1} + e^{u+k_t}}$$

隐藏状态仅 $(a, b)$ 两个向量，推理时每步计算和内存复杂度为 $O(d)$。

## 三、实验结果

### 3.1 语言建模

在 Pile 数据集上训练 RWKV-4 系列：

| 模型 | 参数量 | Pile 验证 loss |
|------|--------|---------------|
| RWKV-4-1.5B | 1.5B | 2.105 |
| RWKV-4-7B | 7B | 1.876 |
| RWKV-4-14B | 14B | 1.771 |

与同规模 Transformer 相比，RWKV 在语言建模上具有竞争力。

### 3.2 下游任务

在多个 NLP 基准上，RWKV-4-14B 与 GPT-NeoX-20B、LLaMA-7B 等模型相比，多数任务上差距在合理范围内，部分任务（如 LAMBADA）甚至优于更大的模型。

### 3.3 推理效率

| 指标 | Transformer | RWKV |
|------|-------------|------|
| 推理内存（每步） | $O(N)$ (KV cache) | $O(1)$ |
| 训练并行度 | $O(1)$ 序列级 | $O(1)$ 序列级 |
| 训练复杂度 | $O(N^2 d)$ | $O(N d)$ |

RWKV 遵循与 Transformer 相同的 scaling law 形式：loss 与 compute 的 log-log 线性关系，$r^2 = 0.994$。

## 四、关键洞察

1. **并行训练 + RNN 推理 = 两全其美**：RWKV 的核心贡献不是某一单一技术，而是将线性注意力的并行训练特性与 RNN 的增量推理特性统一起来。训练时像 Transformer，推理时像 RNN。

2. **通道级衰减 = 自适应记忆长度**：不同维度以不同速率衰减，模型自动学习哪些维度需要长期记忆、哪些只需短期上下文。这比固定注意力窗口更灵活。

3. **时间偏移的优雅性**：用最简单的偏移操作替代位置编码，说明位置信息不需要显式编码——模型可以通过 token 间的相对关系推断位置。

4. **线性注意力的实用化**：之前线性注意力工作（如 Linear Transformer）因性能差距未能广泛采用，RWKV 通过通道级衰减、时间偏移、补偿项 $u$ 等设计弥补了性能差距。

5. **O(1) 推理内存的实际意义**：在长上下文生成场景（如书籍续写、长对话），Transformer 的 KV cache 可达数十 GB，RWKV 的固定内存开销使部署成本大幅降低。

## 五、实验复现 (53_rwkv.py)

模拟 RWKV 和 Transformer 在周期序列建模上的对比：

- 在周期性序列任务上，RWKV 与 Transformer 达到竞争性表现
- RWKV 推理时内存为 $O(1)$（固定状态向量），Transformer 为 $O(N)$（KV cache）
- 验证了线性注意力在特定序列建模任务上的可行性
- 通道级衰减允许模型自适应地捕捉不同频率的周期性模式

## 六、评价

**影响**：RWKV 是**首个成功扩展到 14B 参数的线性注意力/RNN 架构**，打破了"RNN 无法扩展到大规模"的固有认知。它直接催生了 RWKV-5/6/7 等后续版本，并影响了 Mamba、RetNet、TransNorm 等线性 RNN 方向的工作。

**优点**：
- 架构设计精巧：WKV 注意力兼具理论优雅性和工程实用性
- 训练效率高：$O(N)$ 复杂度使得超长序列训练成为可能
- 推理效率极高：$O(1)$ 内存消除了 KV cache 瓶颈
- 工程完成度高：从 169M 到 14B 的完整训练和评估

**局限**：
- 回溯能力（recall）不如 Transformer：线性注意力的信息压缩不可避免地丢失细节，在需要精确检索上下文的任务上弱于全注意力
- 时间偏移的位置建模能力有限：仅偏移一个位置，对远距离相对位置关系的建模不如 RoPE
- 长程依赖仍有衰减：虽然衰减可学习，但递推形式决定了信息必然随距离衰减
- 对 prompt 顺序敏感：RNN 无法回看和重新加权先前信息，调整信息顺序后性能波动显著
- 社区生态不如 Transformer：预训练数据、训练 recipe 等积累不足

**后续发展**：
- **RWKV-5 (Eagle)**：引入数据依赖的衰减率，而非静态通道级衰减
- **RWKV-6 (Finch)**：进一步改进时间混合，增强回溯能力
- **Mamba (Gu & Dao, 2023)**：选择性状态空间模型，与 RWKV 思路相似但用 SSM 框架
- **RetNet**：微软的线性 RNN，同样实现训练并行 + 推理 O(1)
- 线性 RNN 家族已成为 Transformer 之外最具竞争力的架构方向
