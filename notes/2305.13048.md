## 一、核心问题

Transformer 统治了几乎所有的序列建模任务，但其自注意力机制的 $O(T^2)$ 计算和内存复杂度在长序列和资源受限场景下成为严重瓶颈。RNN 虽然推理时复杂度为 $O(T)$，但受限于梯度消失和无法在时间维度并行化训练，难以扩展到大规模。

核心问题：**能否设计一种架构，同时拥有 Transformer 的高效并行训练和 RNN 的高效推理？**

RWKV 的回答：可以。通过线性注意力机制 + 时间衰减设计，使模型在训练时可并行（如 Transformer），推理时保持 $O(d)$ 常数复杂度（如 RNN）。

## 二、方法

### 核心要素：R, W, K, V

- **R（Receptance）**：接收过去信息的门控向量
- **W（Weight）**：可训练的通道级时间衰减向量，$w \in (\mathbb{R}_{\geq 0})^d$
- **K（Key）**：类似注意力中的 Key
- **V（Value）**：类似注意力中的 Value

### Token Shift

所有线性投影的输入不是当前 token 本身，而是当前与上一个 token 的线性插值：

$$r_t = W_r \cdot (\mu_r \odot x_t + (1 - \mu_r) \odot x_{t-1})$$
$$k_t = W_k \cdot (\mu_k \odot x_t + (1 - \mu_k) \odot x_{t-1})$$
$$v_t = W_v \cdot (\mu_v \odot x_t + (1 - \mu_v) \odot x_{t-1})$$

$\mu$ 是可训练的插值系数，在 PyTorch 中通过 `nn.ZeroPad2d((0,0,1,-1))` 高效实现。

### WKV 算子

RWKV 注意力的核心，受 AFT（Attention Free Transformer）启发，但将 $W$ 从配对矩阵简化为通道级向量乘以相对位置：

$$w_{t,i} = -(t-i)w$$

WKV 操作定义为：

$$\text{wkv}_t = \frac{\sum_{i=1}^{t-1} e^{-(t-1-i)w + k_i} \odot v_i + e^{u+k_t} \odot v_t}{\sum_{i=1}^{t-1} e^{-(t-1-i)w + k_i} + e^{u+k_t}}$$

其中 $u$ 是对当前 token 的额外注意力 bonus（避免 $W$ 衰减导致的退化），$w$ 是每个通道独立的时间衰减率。

### 输出门控

Time-mixing 输出：
$$o_t = W_o \cdot (\sigma(r_t) \odot \text{wkv}_t)$$

Channel-mixing 输出：
$$o'_t = \sigma(r'_t) \odot (W_{v'} \cdot \max(k'_t, 0)^2)$$

Channel-mixing 使用 squared ReLU 激活函数。

### RNN 递推形式

WKV 可以改写为递推形式（推理时使用）：

$$a_t = e^{-w} \odot a_{t-1} + e^{k_t} \odot v_t$$
$$b_t = e^{-w} \odot b_{t-1} + e^{k_t}$$
$$\text{wkv}_t = \frac{a_{t-1} + e^{u+k_t} \odot v_t}{b_{t-1} + e^{u+k_t}}$$

隐藏状态仅 $(a, b)$ 两个向量，每层状态大小为 $5DL$（含辅助状态），推理时计算和内存复杂度为 $O(d)$。

### 训练优化

- **Custom CUDA Kernel**：加速 WKV 的串行扫描计算
- **Small Init Embedding**：嵌入矩阵初始化为极小值 + 额外 LayerNorm，加速训练初期收敛
- **Custom Initialization**：大部分权重初始化为零，建立清晰的信息流

## 三、实验结果

### 模型规模

训练了 6 个模型（169M → 14B），均在 Pile 上训练 1 epoch（330B tokens）。

### NLP 基准

在 FLOP 匹配的基础上与 Pythia、OPT、BLOOM 比较。12 个 NLP 任务平均性能：

RWKV 与同等计算量的 Transformer 性能相当，部分任务（如 LAMBADA）甚至优于 Transformer。

### Scaling Laws

RWKV 遵循与 Transformer 相同的 scaling law 形式：loss 与 compute 的 log-log 线性关系，$r^2 = 0.994$。外推一个数量级后 $r^2 = 0.875$。

### 推理效率

| 模型 | 推理时间复杂度 | 推理空间复杂度 |
|------|---------------|---------------|
| Transformer | $O(Td)$ | $O(T+Td)$ |
| RWKV | $O(Td)$ | $O(d)$ |

RWKV 推理时内存不随序列长度增长，文本生成时间随序列线性增长（Transformer 为二次增长）。

### 长上下文

通过渐进式微调（1024→2048→4096→8192），测试损失随上下文长度增加持续下降，证明 RWKV 能有效利用长上下文。

### LRA 基准

在 Long Range Arena 上，RWKV 在 5 个数据集上仅次于 S4 模型，在自然语言相关任务上接近 S4。

## 四、个人评价

### 影响

RWKV 是**首个成功扩展到 14B 参数的线性注意力/RNN 架构**，打破了"RNN 无法扩展到大规模"的固有认知：

1. **开辟了 Transformer 替代架构的实用路线**：此前 MEGA、S4 等虽理论优秀，但未展示大模型可行性
2. **推理效率优势显著**：$O(d)$ 空间复杂度使其适合端侧部署，对资源受限场景意义重大
3. **催生了后续系列工作**：RWKV-5/6/7 持续改进，数据门控线性注意力（Data-dependent linear attention）成为 RWKV-5 的核心创新
4. **开源社区驱动**：与 EleutherAI 合作，模型权重开放，推动了 RNN 复兴的研究热潮

### 局限

1. **线性注意力的表达能力限制**：信息通过单一向量在各时间步间传递，无法像 Transformer 那样"回看"所有先前 token，在需要精确回忆长距离细节的任务上表现较差
2. **对 prompt 工程更敏感**：RNN 无法回看和重新加权先前信息，信息顺序至关重要。调整 prompt 中信息顺序后，RTE 任务 F1 从 44.2% 提升到 74.8%，说明 RNN 架构对输入顺序的鲁棒性较差
3. **原始版本（RWKV-4）的 time-mixing 仅用 token shift**：缺乏像 Mamba 那样的数据依赖选择机制，后续版本（RWKV-5/6）已改进
4. **训练上下文长度有限**：默认仅 1024 tokens，需要额外微调才能支持更长上下文
