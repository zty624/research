# DiT: Scalable Diffusion Models with Transformers (Peebles & Xie, 2022)

**arxiv**: 2212.09748 | **DiT 开山** | UC Berkeley / NYU / Meta FAIR | ICCV 2023

---

## 一、核心问题

2022年之前，扩散模型统一使用 U-Net 作为 backbone。虽然 Transformer 已经统一了 NLP 和 Vision，但扩散模型还是个"例外"——U-Net 的归纳偏置（卷积、下采样-上采样、skip connection）被认为对扩散去噪很重要。

本文问：**U-Net 的归纳偏置真的是必需的吗？能否用纯 Transformer 替代？**

答案是：可以，而且更好。DiT-XL/2 在 ImageNet 256×256 上达到 FID 2.27，超越所有之前的扩散模型。

## 二、架构

### 整体流程

1. 用预训练 VAE（来自 Stable Diffusion）把 256×256×3 图像压缩到 32×32×4 潜空间
2. 把潜表示 "patchify" 成 token 序列（类似 ViT）
3. 过 Transformer blocks 做去噪
4. VAE decoder 解码回像素

### Patchify

把 32×32×4 的潜表示按 patch size $p$ 切成 $T = (I/p)^2$ 个 token，每个 token 维度 $d$。

- $p=2$：$T=256$ tokens（Gflops 最高）
- $p=4$：$T=64$ tokens
- $p=8$：$T=16$ tokens

减小 $p$（更细的 patch）→ 更多 token → 更多 Gflops → 更好质量，但参数量不变。

### DiT Block

比较了三种条件注入方式：
1. **In-context conditioning**：把 $t$ 和 $c$ 的 embedding 拼到输入 token 序列里
2. **Cross-attention**：$t$ 和 $c$ 作为 cross-attention 的输入
3. **adaLN-Zero**（最佳）：用 $t$ 和 $c$ 通过 MLP 预测 LayerNorm 的 scale 和 shift 参数（6 个输出：$\gamma_1, \beta_1, \alpha_1, \gamma_2, \beta_2, \alpha_2$），其中 $\alpha$ 是残差缩放因子，初始化为零

**adaLN-Zero 为什么好？**
- 比 cross-attention 更计算高效（没有额外的 attention 层）
- 初始化时 $\alpha=0$，整个 block 相当于恒等映射——训练初始阶段更稳定
- FID 几乎是 in-context 方式的一半

### 模型规模

| 配置 | 深度 | 隐层维度 | 头数 | 参数量 |
|------|------|---------|------|--------|
| DiT-S | 12 | 384 | 6 | 33M |
| DiT-B | 12 | 768 | 12 | 130M |
| DiT-L | 24 | 1024 | 16 | 458M |
| DiT-XL | 28 | 1152 | 16 | 675M |

## 三、核心发现：Gflops 与 FID 的强相关性

**Transformer Gflops（而非参数量）是预测 FID 的最佳指标**。相关性达 -0.93。

增加 Gflops 的两种方式：
1. 增大模型（S → B → L → XL）：更多层/更宽
2. 减小 patch size（8 → 4 → 2）：更多 token

两种方式都有效，但 **减小 patch size 在同等参数量下提升更大**（因为它同时增加了 token 数和 attention 的空间分辨率）。

### Compute-efficient scaling

更大的模型在同样总计算量下 FID 更低——即大模型 + 少步 > 小模型 + 多步。这和 LLM 的 scaling law 一致。

## 四、实验结果

### ImageNet 256×256

| 模型 | FID-50K | IS |
|------|---------|-----|
| ADM | 10.94 | 101 |
| LDM-4 | 10.56 | 103.5 |
| ADM-U | 7.49 | 127.5 |
| LDM-4 (cfg=1.5) | 3.60 | 247.7 |
| **DiT-XL/2 (cfg=1.5)** | **2.27** | **278.8** |

在 512×512 上 DiT-XL/2 也达到 3.04 FID，超越之前所有方法。

### 训练细节

- 使用 ADM 的超参数几乎不做修改（lr=1e-4, batch=256, no warmup, no weight decay）
- EMA decay=0.9999
- 没有观察到 loss spike——训练非常稳定
- 最快的 DiT-XL/2 在 TPU v3-256 上约 5.7 iter/s

## 五、个人评价

DiT 的意义远不止于"把 U-Net 换成 Transformer"——它证明了：

1. **U-Net 的归纳偏置不是必需的**：卷积、skip connection、下采样-上采样这些结构对扩散去噪不是本质的。纯 Transformer + adaLN 就够了
2. **扩散模型也能享受 scaling law**：Gflops 和 FID 的强相关性意味着可以通过简单 scale up 来提升质量，不需要精巧的架构设计
3. **架构统一的价值**：用 Transformer 做 diffusion backbone 让整个领域可以共享 ViT 的训练技巧、硬件优化（FlashAttention 等）

DiT 的影响极其深远：
- **Sora**（OpenAI 的视频生成模型）直接基于 DiT 架构
- **Stable Diffusion 3** 用了 DiT + Flow Matching
- **PixArt** 系列也基于 DiT
- 后续所有主流图像/视频生成模型基本都转向了 Transformer backbone

论文的一个遗憾是没有做更大的模型（XL 只有 675M 参数），也没有在 LAION 等大规模数据集上验证。但后来 Stable Diffusion 3 和 Sora 证明了 DiT 在更大规模上的有效性。

adaLN-Zero 这个设计细节也很重要——后来的实践表明，条件注入方式对生成质量影响巨大，adaLN-Zero 几乎成了 DiT 系列的标准配置。
