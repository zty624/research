# GLU Variants Improve Transformer (Shazeer, 2020)

**arxiv**: 2002.05202 | **现代 LLM FFN 激活函数标准** | 2020

---

## 一、核心问题

Transformer 的 FFN 层用什么激活函数最好？原始 Transformer 用 ReLU，后来 GPT-2/BERT 改用 GELU，但这些选择缺乏系统性的对比研究。本文将 GLU（Gated Linear Unit）的门控思想引入 Transformer FFN，并系统比较了各种 GLU 变体的效果。

## 二、背景：GLU (Gated Linear Unit)

GLU 最早由 Dauphin et al. (1705.03122) 提出，用于语言模型。核心思想是用一个门控信号对线性变换的结果做逐元素调制：

$$\text{GLU}(x) = \sigma(Wx + b) \otimes (Vx + c)$$

其中 $\sigma$ 是 sigmoid 函数，$\otimes$ 是逐元素乘法。左半部分 $\sigma(Wx+b)$ 充当"门"，控制右半部分 $Vx+c$ 信息的通过量。sigmoid 输出在 (0,1) 之间，天然适合做软门控。

这种门控机制与 LSTM 中的遗忘门/输入门思想一脉相承：让网络学会选择性传递信息。

## 三、关键创新：GLU 变体作为 Transformer FFN 激活

本文的核心观察：GLU 中的 sigmoid 门可以用其他激活函数替换，产生一系列变体。每种变体对应一个不同的门控函数：

| 变体 | 门控函数 | FFN 公式 |
|------|---------|----------|
| GLU | $\sigma(x)$ (sigmoid) | $W_2(\sigma(W_1 x) \otimes W_3 x)$ |
| Bilinear | 无（恒等） | $W_2((W_1 x) \otimes W_3 x)$ |
| ReGLU | $\text{ReLU}(x)$ | $W_2(\text{ReLU}(W_1 x) \otimes W_3 x)$ |
| GEGLU | $\text{GELU}(x)$ | $W_2(\text{GELU}(W_1 x) \otimes W_3 x)$ |
| SwiGLU | $\text{Swish}(x) = x \cdot \sigma(x)$ | $W_2(\text{Swish}(W_1 x) \otimes W_3 x)$ |

对比标准 Transformer FFN（只用一个权重矩阵做激活）：

- **标准 ReLU FFN**: $\text{FFN}(x) = W_2 \cdot \text{ReLU}(W_1 x)$ — 2 个权重矩阵
- **GLU 变体 FFN**: $\text{FFN}(x) = W_2 \cdot (\text{gate}(W_1 x) \otimes W_3 x)$ — 3 个权重矩阵

GLU 变体多了一个 $W_3$ 矩阵，将输入投影到另一个空间与门控信号逐元素相乘，再通过 $W_2$ 映射回原维度。

## 四、实验结果

在标准 Transformer 架构上做对比实验，控制参数量一致：

- **SwiGLU 和 GEGLU 表现最好**，在多个任务上显著优于标准 ReLU/GELU FFN
- ReGLU 也不错但略逊于 SwiGLU/GEGLU
- 原始 GLU（sigmoid 门控）和 Bilinear 效果较差
- Swish 门控（$x \cdot \sigma(x)$）比单纯 sigmoid 门控效果好，说明门控函数本身也需要非线性

关键结论：门控机制 + 好的门控激活函数 = 最佳 FFN 激活方案。

## 五、参数量考量

GLU 变体 FFN 有 3 个权重矩阵（$W_1, W_2, W_3$），而标准 FFN 只有 2 个。为了保持总参数量相近，需要调整中间维度 $d_{ff}$：

- **标准 FFN**: $d_{ff} = 4d$，参数量 $\approx 2 \times d \times 4d = 8d^2$
- **GLU 变体 FFN**: $d_{ff} = \frac{8d}{3}$，参数量 $\approx 3 \times d \times \frac{8d}{3} = 8d^2$

因此现代 LLM 采用 SwiGLU 时通常用 $d_{ff} = \frac{8d}{3}$（或近似为 $\lfloor \frac{8d}{3} \rfloor$ 的整数倍），而非传统的 $4d$。例如 LLaMA 系列就遵循这一约定。

## 六、影响

这篇短文（仅 4 页）对现代 LLM 架构的影响极其深远：

1. **LLaMA** 系列（Meta）使用 SwiGLU
2. **PaLM**（Google）使用 SwiGLU
3. **Mistral**、**Qwen**、**DeepSeek** 等几乎所有现代开源 LLM 均采用 SwiGLU
4. SwiGLU 已成为现代 LLM FFN 层的事实标准，取代了原始 Transformer 的 ReLU 和 BERT/GPT-2 的 GELU

SwiGLU 之所以胜出，可能因为 Swish 函数本身的性质：它在正值区间近似线性（梯度不消失），在负值区间有平滑的稀疏性（类似 ReLU 但处处可导），同时门控机制提供了更灵活的信息选择性。

## 七、个人评价

这篇论文的简洁程度和影响力形成了鲜明对比。没有复杂的理论推导，没有庞大的实验，只是系统性地比较了一个简单但关键的设计选择——FFN 用什么激活函数。但它改变了所有后续 LLM 的架构设计。

值得注意的几点：

1. **门控思想与 MoE 的联系**：GLU 的门控是逐元素的软选择，而 MoE (Mixture of Experts) 是 token 级别的硬选择（top-k routing）。两者本质都在增加 FFN 的表达能力，只是粒度不同
2. **为什么 SwiGLU > GEGLU**：Swish ($x\sigma(x)$) 在负值区间的行为比 GELU 更平滑，且 Swish 的非单调性可能在门控场景下更有优势。但论文本身没有深入分析这个差异
3. **论文的局限**：实验规模相对较小（与后来的 LLM 相比），但在 LLM 时代的实践已经充分验证了结论的正确性
