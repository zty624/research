# Mamba: Linear-Time Sequence Modeling with Selective State Spaces (Gu & Dao, 2023)

**arxiv**: 2312.00752 | **选择性 SSM 开山之作** | COLM 2024

---

## 一、核心贡献

Mamba 解决的核心问题是：**如何让状态空间模型（SSM）具备选择性（selectivity）——根据输入决定记住或遗忘信息**。具体贡献：

1. **选择性机制（Selective Mechanism）**：将 SSM 的参数（B, C, $\Delta$）从输入无关变为输入依赖，使模型能像注意力一样"关注"重要信息、"忽略"无关信息
2. **硬件感知并行扫描算法**：选择性 SSM 无法用卷积高效实现（因为参数随输入变化），设计了 GPU 友好的并行扫描算法，保持训练高效
3. **无需注意力机制的序列模型**：Mamba 在语言建模等任务上匹配甚至超越同规模 Transformer，推理速度随序列长度线性增长

## 二、关键技术点

### 背景：结构化状态空间模型（S4）

S4 基于连续线性时不变（LTI）系统：

$$\mathbf{x}'(t) = A\mathbf{x}(t) + B\mathbf{u}(t), \quad \mathbf{y}(t) = C\mathbf{x}(t) + D\mathbf{u}(t)$$

离散化后：

$$\mathbf{x}_t = \bar{A}\mathbf{x}_{t-1} + \bar{B}\mathbf{u}_t, \quad \mathbf{y}_t = C\mathbf{x}_t$$

其中 $\bar{A} = \exp(\Delta A)$，$\bar{B} = (\Delta A)^{-1}(\exp(\Delta A) - I) \cdot \Delta B$。

S4 的关键限制：**A, B, C, $\Delta$ 是固定的**，不随输入变化——这等价于线性时不变（LTI）系统，是卷积的一种推广。LTI 意味着模型对所有输入做同样的变换，无法选择性处理。

### 选择性机制：核心创新

Mamba 让 B, C, $\Delta$ 变为输入的函数：

$$B_t = \text{Linear}_B(\mathbf{u}_t), \quad C_t = \text{Linear}_C(\mathbf{u}_t), \quad \Delta_t = \text{softplus}(\text{Linear}_\Delta(\mathbf{u}_t))$$

而 A 仍通过 $\Delta$ 间接依赖于输入（$\bar{A} = \exp(\Delta_t A)$）。

**为什么选择性很重要？** 通过几个直觉性任务说明：

1. **选择性复制**：从序列中只复制特定标记（如"复制标记后的内容"）。LTI SSM 无法区分哪些位置该复制，选择性 SSM 可以在特定位置增大 $\Delta$（更快更新隐藏状态）、调整 B/C 来控制信息流
2. **过滤噪声**：在两个有用 token 之间插入噪声 token。选择性 SSM 可以让 B 在噪声位置接近 0，从而"忽略"噪声
3. **诱导头（Induction Heads）**：需要回忆之前见过的模式。选择性机制让模型只在相关信息出现时更新状态

**$\Delta$ 的门控角色**：
- $\Delta$ 大 -> 模型更关注当前输入（重置状态）
- $\Delta$ 小 -> 模型更保留之前的状态（延续信息）
- 这和 LSTM 的遗忘门/输入门本质上是同一件事

### 参数化的维度设计

- S4：A $\in \mathbb{R}^{N \times N}$，B, C $\in \mathbb{R}^{N}$，$\Delta \in \mathbb{R}$——参数对所有通道共享
- Mamba：A $\in \mathbb{R}^{D \times N}$（每个通道独立），B, C $\in \mathbb{R}^{D \times N}$，$\Delta \in \mathbb{R}^{D}$——每个通道独立参数化

维度扩展使参数量增加，但通过硬件感知实现保持高效。

### 硬件感知并行扫描

选择性 SSM 的挑战：B, C, $\Delta$ 随输入变化，**无法用卷积实现**（卷积要求 LTI），只能用递推：

$$\mathbf{x}_t = \bar{A}_t \mathbf{x}_{t-1} + \bar{B}_t \mathbf{u}_t$$

递推是串行的（$O(N)$ 步，无法并行）。但这是一个**结合性（associative）** 操作，可以用并行扫描（parallel scan）算法在 $O(\log N)$ 步完成：

1. 在 GPU SRAM 中扫描，避免物化中间状态到 HBM
2. 利用结合性做树形归约，步骤从 $O(N)$ 降到 $O(\log N)$
3. 输入扩展维度在 SRAM 中做矩阵乘法，减少 IO

**关键技巧**：不激活完整的 $N$ 维状态，而是在低维空间做扫描，最后再扩展。类似 FlashAttention 的 tiling 思想。

### Mamba 块架构

Mamba 将选择性 SSM 与线性投影融合为统一块，替代 Transformer 的注意力 + FFN：

```
输入 x -> Linear(扩展 2x) -> 分支1: SiLU -> SSM -> x -> Linear(压缩) -> 输出
                                    分支2: SiLU ->/
```

- 输入先投影到 2 倍宽度，分成两个分支
- 一个分支过 SSM，另一个分支做门控（类似 SwiGLU 的思路）
- 最终投影回原始宽度

**设计选择**：
- 激活函数用 SiLU（Swish）
- 初始化：$\Delta$ 初始化为 $\Delta_{\text{init}} = 1$（S4D-Real 初始化）
- 整个块只有一个线性投影 + SSM + 门控，比 Transformer 的注意力 + FFN 更简洁

## 三、实验结果

### 语言建模

| 模型 | 参数量 | Perplexity (Pile) |
|------|--------|------------------|
| Transformer++ | 1.3B | 6.67 |
| Mamba | 1.3B | **6.58** |
| Transformer++ | 2.7B | 6.27 |
| Mamba | 2.7B | **6.22** |

Mamba 在相同参数量下 perplexity 低于 Transformer++（指用 RoPE + SwiGLU + FlashAttention 的现代 Transformer）。

### 下游任务（Zero-shot）

在 6 个常识推理数据集上：
- Mamba-1.4B 匹配或超越 GPT-3 (1.3B) 的零样本表现
- Mamba-2.8B 匹配或超越 Pythia-2.8B
- Mamba-3B 甚至比 Pythia-7B 好

### 长序列建模

- Mamba 随着序列长度增加，perplexity 持续下降（能利用更长的上下文）
- 在音频和基因组学上，性能随序列长度提升到 **1M tokens** 仍不饱和
- 音频生成上 FID 降低超过 50%

### 推理速度

| 序列长度 | Transformer 吞吐量 | Mamba 吞吐量 |
|---------|-------------------|-------------|
| 2K | 1.0x | ~1.5x |
| 8K | 1.0x | ~3x |
| 32K | 1.0x | ~5x |

Mamba 推理吞吐量不随序列长度下降（因为状态大小固定），Transformer 因 KV cache 线性增长而变慢。

### 消融实验

1. **选择性 vs 非选择性**：去掉选择性（B, C, $\Delta$ 固定），性能显著下降，证明选择性是关键
2. **$\Delta$ 的选择性最关键**：只让 $\Delta$ 依赖输入，B/C 固定，效果接近全选择性的 80%
3. **A 的初始化**：S4D-Real 初始化 > 随机初始化
4. **块设计**：门控分支比纯 SSM 好很多

### DNA 建模

Mamba 在长 DNA 序列建模上也表现优异（序列长度可达 100 万），进一步验证了选择性 SSM 的通用性。

## 四、个人思考与影响

### 选择性是关键洞察

Mamba 最深刻的贡献不是"比 Transformer 快"，而是**将选择性的概念引入 SSM**。之前 S4 等工作虽然高效，但本质是 LTI 系统——对所有输入做同样的事。Mamba 通过输入依赖参数化实现了：

- **动态记忆**：$\Delta$ 大时快速更新状态（"记住"），$\Delta$ 小时保持状态（"保持"）
- **动态遗忘**：B 接近 0 时忽略输入，C 接近 0 时不输出
- **这些操作都是数据驱动的**——模型学会了何时记住/遗忘

这种机制在概念上与 LSTM 的门控异曲同工，但形式更简洁、并行性更好。

### 与 Transformer 的关系

| 特性 | Transformer | Mamba |
|------|------------|-------|
| 训练并行性 | $O(1)$ 序列操作 | $O(\log N)$ 并行扫描 |
| 推理状态 | KV cache $O(N)$ | 隐藏状态 $O(D)$ |
| 推理速度 | $O(N)$ 每步 | $O(1)$ 每步 |
| 长程依赖 | $O(1)$ 路径长度 | $O(N)$（递推），但选择性缓解 |
| 选择性 | 天然有（attention 权重） | 通过输入依赖参数实现 |

Mamba 推理更高效，但在"任意两位置直接交互"的能力上不如 Transformer。实际中，对大多数 NLP 任务，选择性 SSM 的交互模式已足够。

### 后续影响

- **Mamba-2**：将 SSM 与结构化注意力统一（SSD 框架），训练更快
- **Jamba (AI21)**：Mamba + Transformer 混合架构，52B 参数
- **Mamba 在视觉/多模态的应用**：VMamba, Mamba-based 视觉模型
- **Zyphra/Zamba**：将 Mamba 用于生产级语言模型

### 局限与开放问题

1. **长程回忆能力**：对"在极远位置查找一个精确 token"这种任务，SSM 的压缩状态可能不够——注意力可以直接检索
2. **训练稳定性**：并行扫描的数值稳定性需要仔细处理
3. **生态成熟度**：FlashAttention 有成熟的 Triton/CUDA 实现，Mamba 的生态还在发展中
4. **扩展定律**：Mamba 在大规模（>10B 参数）的扩展行为尚不完全清楚
5. **混合架构可能是最优**：纯 Mamba 和纯 Transformer 都非最优，Jamba 式的混合可能更实用

Mamba 的价值在于证明了：**选择性状态空间模型可以是 Transformer 的可行替代方案**。它打破了"序列建模必须用注意力"的垄断，为架构设计开辟了新方向。
