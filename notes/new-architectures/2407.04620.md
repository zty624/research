# Learning to (Learn at Test Time): RNNs with Expressive Hidden States (Sun et al., 2024)

**arxiv**: 2407.04620 | **关键词 序列建模 测试时训练 开山** | Stanford / UCSD / UC Berkeley / Meta AI | 2024

---

## 一、核心问题

自注意力在长上下文中表现优异但二次复杂度，现有 RNN 层线性复杂度但隐藏状态表达力有限。具体表现：Mamba 在 16k 上下文后困惑度不再下降——上下文越长，RNN 的压缩瓶颈越严重。

核心洞察：**自监督学习可以将海量训练集压缩进模型权重，使模型展现对数据深层结构和关系的理解**——这正是 RNN 压缩启发式所需要的。因此，让隐藏状态本身成为一个机器学习模型，更新规则成为一步自监督学习。

## 二、方法

### 2.1 TTT 层的基本框架

所有序列建模层都可以从"将历史上下文存储到隐藏状态"的视角理解：

| 层类型 | 初始状态 | 更新规则 | 输出规则 | 每步成本 |
|--------|---------|---------|---------|---------|
| Naive RNN | $s_0 = \text{vector}()$ | $s_t = \sigma(\theta_s s_{t-1} + \theta_x x_t)$ | $z_t = \theta_z s_t + \theta_x x_t$ | $O(1)$ |
| Self-Attention | $s_0 = \text{list}()$ | $s_t = s_{t-1}.\text{append}(k_t, v_t)$ | $z_t = V_t \text{softmax}(K_t^\top q_t)$ | $O(t)$ |
| **Naive TTT** | $W_0 = f.\text{params}()$ | $W_t = W_{t-1} - \eta \nabla \ell(W_{t-1}; x_t)$ | $z_t = f(x_t; W_t)$ | $O(1)$ |

TTT 层的核心：
- **隐藏状态** = 模型 $f$ 的权重 $W_t$
- **更新规则** = 对自监督损失的一步梯度下降
- **输出规则** = 用更新后的权重做预测

即使是测试时，TTT 层仍在训练新的权重序列 $W_1, \ldots, W_T$，因此称为**测试时训练（Test-Time Training）**。

### 2.2 可学习的自监督任务

原始重建任务 $\ell(W; x_t) = \|f(\tilde{x}_t; W) - x_t\|^2$（$\tilde{x}_t$ 为 $x_t$ 的损坏版本）不够灵活。本文提出**多视角重建**，引入外循环参数：

$$\ell(W; x_t) = \|f(\theta_K x_t; W) - \theta_V x_t\|^2$$

- **训练视角**（training view）：$\theta_K x_t$，低秩投影的损坏输入
- **标签视角**（label view）：$\theta_V x_t$，选择性地重建哪些信息
- **测试视角**（test view）：$\theta_Q x_t$，输出规则变为 $z_t = f(\theta_Q x_t; W_t)$

外循环优化 $\theta_K, \theta_Q, \theta_V$ 以及网络其余参数 $\theta_{\text{rest}}$；内循环仅优化 $W$。

**内外循环的区别**：
- 内循环：数据 = token $x_t$，训练集 = 序列 $x_1, \ldots, x_T$，目标 = 重建，参数 = $W$
- 外循环：数据 = 序列，训练集 = 序列数据集，目标 = 下一 token 预测，参数 = $\theta$ 系列

### 2.3 Mini-batch TTT：并行化

原始 TTT 的更新 $W_t = W_{t-1} - \eta \nabla \ell(W_{t-1}; x_t)$ 不可并行（$W_t$ 两处依赖 $W_{t-1}$）。

三种梯度下降变体：
- **Online GD**：$G_t = \nabla \ell(W_{t-1}; x_t)$，顺序但最有效
- **Batch GD**：$G_t = \nabla \ell(W_0; x_t)$，可并行但只离 $W_0$ 一步，搜索空间小
- **Mini-batch GD**：$G_t = \nabla \ell(W_{t'}; x_t)$，$t' = t - \text{mod}(t, b)$，折中方案

$$W_t = W_{t-1} - \eta G_t = W_0 - \eta \sum_{s=1}^t G_s$$

Mini-batch 大小 $b$ 控制速度-质量权衡。$b=16$ 为本文所有实验的选择。

### 2.4 对偶形式：硬件高效

朴素实现需要逐个计算外积 $G_t = 2(W_0 x_t - x_t) x_t^\top$，这是 $d \times d$ 矩阵，内存和 I/O 开销大。

**关键观察**：我们不需要显式构造 $G_1, \ldots, G_b$，只需计算 mini-batch 末尾的 $W_b$ 和所有输出 $z_1, \ldots, z_b$。

对 TTT-Linear（$f(x) = Wx$）的推导：

$$W_b = W_0 - 2\eta(W_0 X - X)X^\top$$

$$\Delta = (W_0 X - X) \cdot \text{mask} \cdot X^\top X, \quad Z = W_0 X - 2\eta \Delta$$

其中 $\text{mask}$ 是上三角 mask，$X = [x_1, \ldots, x_b]$。所有运算都是矩阵乘法，可高效利用 TensorCore。

对偶形式 vs 原始形式：
- 原始：$O(b \times d^2)$ 但几乎无 matmul
- 对偶：$O(b \times d^2) + O(b^2 \times d)$，但有大量 matmul，实际快 5× 以上

### 2.5 理论等价性

**Theorem 1（TTT-Linear = 线性注意力）**：当 $f(x) = Wx$，使用 batch GD，$\eta = 1/2$，$W_0 = 0$ 时：

$$z_t = \sum_{s=1}^t (\theta_V x_s)(\theta_K x_s)^\top (\theta_Q x_t)$$

这正是线性注意力的定义（无 softmax）。

**Theorem 2（TTT + Nadaraya-Watson = 自注意力）**：使用非参数学习器 Nadaraya-Watson 估计器：

$$f(x; x_1, \ldots, x_t) = \frac{\sum_{s=1}^t \kappa(x, x_s) y_s}{\sum_{s=1}^t \kappa(x, x_s)}$$

其中核函数 $\kappa(x, x'; \theta_K, \theta_Q) \propto e^{(\theta_K x)^\top (\theta_Q x')}$，则输出等价于自注意力。

**统一抽象——Learner**：参数学习器（线性模型、MLP）和非参数学习器（Nadaraya-Watson）都实现 `train` 和 `predict` 方法，统一为 TTT 层。

### 2.6 实现细节

- **TTT-Linear**：$f_{\text{lin}}(x) = Wx$（方阵）
- **TTT-MLP**：两层 MLP，隐藏维度 4×，GELU 激活
- $f$ 始终包含 LayerNorm + 残差：$f(x) = x + \text{LN}(f_{\text{res}}(x))$
- **可学习 $W_0$**：$\theta_{\text{init}} = W_0$，外循环参数，显著改善训练稳定性
- **可学习 $\eta$**：$\eta(x) = \eta_{\text{base}} \sigma(\theta_{\text{lr}} \cdot x)$，输入相关的学习率/门控
- **骨干**：使用 Mamba 骨干（含时间卷积）而非 Transformer 骨干，因前者的时间卷积帮助收集局部信息

## 三、实验结果

### 3.1 从线性注意力到 TTT-Linear 的消融

| 配置 | 困惑度 | 差值 |
|------|--------|------|
| 线性注意力 | 15.91 | — |
| 改进的线性注意力 | 15.23 | −0.68 |
| TTT 等价 | 15.23 | 0 |
| + 可学习 $W_0$ | 15.27 | +0.04 |
| + LN 和残差 | 14.05 | −1.22 |
| **+ mini-batch TTT** | **12.35** | **−1.70** |
| + 可学习 $\eta$ | 11.99 | −0.36 |
| + Mamba 骨干 | 11.09 | −0.90 |

**最大贡献来自 mini-batch TTT**（从 batch GD 改为 mini-batch GD，即从 $b = T = 2048$ 改为 $b = 16$）。

### 3.2 Pile 上的缩放趋势

- **2k 上下文**：TTT-Linear (M)、Mamba、Transformer 性能可比
- **8k 上下文**：TTT-Linear (M) 和 TTT-MLP (M) 显著优于 Mamba

### 3.3 Books 上的长上下文

- **2k**：Mamba 略优于 TTT-Linear
- **32k**：TTT-Linear (M) 和 TTT-MLP (M) 均优于 Mamba。关键观察：**随着上下文变长，TTT 相对 Mamba 的优势增大**

### 3.4 上下文利用

类似 Transformer，TTT-Linear 和 TTT-MLP 在条件化更多 token 时困惑度持续下降；而 Mamba 在 16k 后困惑度平台期。

### 3.5 推理延迟

- Forward（prefill）：Transformer 线性增长，其余常数
- Generate（decode）：所有方法常数延迟，TTT-MLP 略慢于 TTT-Linear

## 四、个人评价

### 影响

1. **新范式**：将序列建模重新表述为"学习如何（在测试时学习）"，将监督学习重述为双层优化。内循环 = TTT，外循环 = 常规训练。这种概念框架有深远的理论意义
2. **表达力与效率的统一**：TTT 层通过让隐藏状态成为模型本身，在常数每步成本下实现了高表达力的隐藏状态，提供了 RNN 和 Transformer 之间的第三条路
3. **理论统一**：Theorem 1 和 2 将线性注意力和自注意力统一在 TTT 框架下，揭示了它们分别是参数学习器和非参数学习器的特例
4. **长上下文优势**：TTT 层的核心优势在长上下文中体现——隐藏状态的表达力随训练（即使在测试时）而增长
5. **搜索空间巨大**：模型 $f$ 和优化器 $\times$ 的组合空间广阔，本文只探索了 TTT-Linear 和 TTT-MLP 两种实例化

### 局限

1. **TTT-MLP 的实际效率问题**：虽然 FLOPs 上 TTT-MLP 在长上下文中有优势，但 MLP 的额外复杂度使 wall-clock time 远高于 FLOPs 所暗示的。内存 I/O 仍是瓶颈
2. **仅验证到 1.3B**：所有实验最大到 1.3B 参数，未在更大规模上验证。缩放律是否在更大模型上保持不明
3. **未训练超长上下文**：受学术资源限制，未训练百万级上下文长度，而这正是 TTT 理论上最应该闪耀的场景
4. **骨干依赖**：TTT 层目前依赖 Mamba 骨干的时间卷积来收集局部信息。如果隐藏状态表达力足够，这种依赖应该不必要，但当前尚未实现
5. **缺乏混合架构探索**：未探索 TTT 层与自注意力的混合架构（如 Griffin 的做法），这可能进一步提升性能
6. **与同期工作的对比不足**：DeltaNet、Mamba 2、Gated DeltaNet 等有类似的矩阵隐藏状态设计，论文未进行详细对比
7. **mini-batch GD 的理论保证不足**：为什么 $b=16$ 是好的选择？不同 $b$ 对收敛的影响缺乏理论分析
8. **"测试时训练"与推理计算的权衡**：TTT 在推理时仍需做梯度下降，这对延迟敏感的应用可能不理想
