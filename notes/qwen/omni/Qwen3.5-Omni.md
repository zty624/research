- **作者**: Qwen Team, Alibaba Group
- **发表时间**: 2026年4月

- **项目主页**: https://qwenlm.github.io/blog/qwen3.5-omni/

## 2. 核心贡献

Qwen3.5-Omni 是一个端到端的全模态大模型，在单一框架内统一了感知（perception）、思考（thinking）和生成（generation）三种能力。其核心贡献可以概括为以下三点：

1. **统一的全模态架构设计**：提出了一种基于 Thinker-Talker 范式的架构，Thinker 负责深层推理，Talker 负责流式多模态生成，两者协同实现了"边想边说"的高效推理-生成流程。模型能够同时处理文本、图像、音频、视频作为输入，并以文本和语音双通道同步输出。

2. **多模态生成能力的突破**：在文本理解和生成方面达到或超过 Qwen3-235B-A22B 的水平，同时在语音生成质量上超越此前的 Qwen2.5-Omni，图像生成能力接近专用模型。这是目前首个在理解和生成两个维度都达到强水平的大一统模型。

3. **高效的训练策略**：通过分阶段渐进训练（从纯文本到多模态理解再到多模态生成），以及创新的思维链蒸馏和流式生成机制，使得模型在有限算力下实现了全模态能力的平衡发展，避免了"跷跷板效应"（即一个模态能力提升导致另一个模态退化）。

## 3. 模型架构详解

### 3.1 整体架构：Thinker-Talker 范式

Qwen3.5-Omni 的架构继承并扩展了 Qwen2.5-Omni 的 Thinker-Talker 设计，但在多个关键方面进行了升级。

**Thinker** 是整个模型的核心推理引擎，本质上是一个大语言模型（基于 Qwen3 的 dense 架构），参数量约 28B。Thinker 负责处理所有输入模态的深层语义理解、思维链推理以及决策。它接收经过各模态编码器处理后的 token 序列，通过自注意力机制进行跨模态融合和推理。

**Talker** 是多模态流式生成模块，接收 Thinker 的输出作为输入，以自回归方式逐 token 生成最终的多模态输出。Talker 的关键设计在于它支持双通道并行输出——文本和语音可以同步流式生成，用户不需要等待完整推理完成就能听到语音回应。此外，Talker 还集成了图像生成能力。

这种 Thinker-Talker 分离设计的核心优势在于：**推理和生成解耦**，Thinker 可以专注于高质量的思维链推理，而 Talker 可以专注于高效的多模态信号生成。两者之间通过一个紧凑的接口连接，避免了端到端训练时推理能力和生成能力互相干扰的问题。

### 3.2 模态编码器

模型为不同输入模态设计了专门的编码器：

- **文本编码器**：直接使用 LLM 自身的 tokenizer（基于 BPE），文本 token 进入 Thinker 后无需额外编码。
- **视觉编码器（ViT）**：采用 Vision Transformer 架构处理图像和视频帧。图像被分割为 patch 后经过 ViT 提取视觉 token，再通过视觉-语言适配器（adapter）映射到与文本 token 同维的嵌入空间。视频则按固定帧率采样后逐帧处理。
- **音频编码器**：采用专门的音频编码器处理语音输入。音频信号先转换为梅尔频谱图（mel-spectrogram），再通过卷积层和 Transformer 层提取音频 token，同样通过适配器映射到统一嵌入空间。

值得注意的是，各模态的适配器设计使得不同模态的 token 被投影到相同的语义空间，这是实现跨模态理解的关键。

### 3.3 图像生成模块

Qwen3.5-Omni 的图像生成能力是本次新增的重要特性。其设计思路是：将图像生成任务统一到语言建模框架中，通过将图像 token 化（利用 tokenizer），使得图像生成可以像文本生成一样通过自回归方式完成。

具体实现上，模型采用了一个图像 tokenizer 将图像编码为离散 token 序列，Talker 在生成时可以无缝切换文本 token 和图像 token。图像生成训练中，模型学习了从文本描述到图像 token 序列的映射，并结合了思维链推理来提升生成质量——即模型会先"想"清楚图像的构图、风格、细节，然后再逐 token 生成图像。

### 3.4 语音生成模块

语音生成延续了 Qwen2.5-Omni 的双通道设计，但进行了重要改进：

- **流式语音合成**：Talker 在生成文本 token 的同时，同步生成对应的音频 token，通过流式音频解码器实时播放，大幅降低了首字延迟。
- **语音风格控制**：支持通过文本指令控制生成语音的情感、语速、语调等风格特征。
- **语音一致性**：在多轮对话中保持语音风格的一致性。

### 3.5 关键设计选择分析

**为什么选择 Thinker-Talker 而非完全端到端？** 完全端到端的一个大模型同时负责推理和生成，在训练中容易出现严重的跷跷板效应——推理任务和生成任务的梯度方向冲突，导致优化困难。Thinker-Talker 的分离使得两个模块可以各自采用最适合的训练策略，同时通过接口连接保证信息流通。

**为什么采用渐进式模态扩展而非一次性多模态训练？** 从纯文本模型出发，逐步叠加理解和生成能力，每一步都建立在已充分训练的基础上。这种方式比一次性训练更稳定，也更节省算力。

## 4. 训练数据和方法

### 4.1 训练数据

Qwen3.5-Omni 的训练数据覆盖了多个模态和多种任务类型：

**文本数据**：继承了 Qwen3 的海量高质量文本语料，涵盖多语言、多领域，是模型基础语言能力的根基。数据量达到万亿 token 级别。

**视觉理解数据**：包括图像描述（image captioning）、视觉问答（VQA）、图表理解、文档理解、OCR 等任务。使用了公开数据集和内部高质量标注数据的混合。特别强调了高分辨率图像的理解能力。

**视频理解数据**：包含视频描述、视频问答、时序推理等任务。视频按固定帧率采样（通常 2-4 fps），覆盖短视频和长视频场景。

**音频理解数据**：涵盖语音识别（ASR）、语音翻译、音频事件检测等任务。数据覆盖多种语言和口音。

**图像生成数据**：高质量图文对，每条数据包含详细的图像描述和对应图像。描述由更强的语言模型生成或人工标注，确保文本指令到图像内容的精确映射。

**语音生成数据**：大量文本-语音对，用于训练语音合成能力。数据覆盖多种说话人、情感和语言。

### 4.2 训练方法

训练采用多阶段渐进策略：

**阶段一：基础语言模型预训练**。以 Qwen3 的预训练权重为起点，这为模型提供了强大的文本理解和推理基础。这一阶段已经在之前完成。

**阶段二：多模态理解训练**。冻结语言模型主体，先训练各模态编码器和适配器，使模型获得多模态感知能力。然后解冻部分参数进行联合微调，使模态融合更充分。这一阶段主要使用理解型任务数据（VQA、ASR 等）。

**阶段三：多模态生成训练**。在理解能力基础上，引入生成训练。这一阶段的关键挑战是如何在不损害理解能力的前提下培养生成能力。具体做法包括：
- 先在大量图文对上进行图像生成预训练
- 再在高质量数据上进行精细化训练
- 生成训练过程中保持一定比例的理解数据，防止理解能力退化

**阶段四：统一对齐**。通过指令微调和 RLHF（或其变体如 DPO），使模型的行为更符合人类期望，包括安全性、有用性和多模态输出格式的一致性。

**思维链蒸馏**：为了让 Thinker 具备更强的推理能力，论文采用了思维链蒸馏技术——使用更强的模型（如 Qwen3-235B）生成详细的思维链，然后将这些思维链作为训练目标。这使得 28B 参数的 Thinker 能够逼近更大模型的推理水平。

### 4.3 训练基础设施

训练使用了大规模 GPU 集群，支持完全分片数据并行（FSDP）和混合精度训练。考虑到多模态数据的异构性（文本短序列 vs 图像/视频长序列 vs 音频连续信号），训练框架对不同的数据类型进行了批处理优化和动态填充策略。

## 5. 主要实验结果

### 5.1 文本理解与推理

Qwen3.5-Omni 在文本基准上表现出色，关键指标：

- **MMLU**: 达到与 Qwen3-32B 相当的水平（约 85%+），远超同等参数规模的纯文本模型
- **MATH / GSM8K**: 数学推理能力接近 Qwen3-235B-A22B，这归功于思维链蒸馏
- **HumanEval / MBPP**: 代码生成能力同样接近更大模型
- **多语言基准**: 在多语言任务上保持了 Qwen3 系列的强势表现

这一结果非常重要——它证明了在加入多模态理解和生成能力后，模型的文本推理能力几乎无损，成功克服了"跷跷板效应"。

### 5.2 视觉理解

- **MMBench / MMMU**: 在综合视觉理解基准上达到同级别模型的最优水平
- **MathVista / AI2D**: 在视觉数学推理和科学图表理解上表现突出
- **DocVQA / ChartQA**: 文档和图表理解能力强
- **视频理解基准（Video-MME 等）**: 在长视频理解上表现出色

### 5.3 音频理解

- **语音识别（ASR）**: 在 LibriSpeech 等标准基准上表现优异
- **语音翻译**: 支持多种语言间的语音翻译
- **音频事件检测**: 能识别环境声音和事件

### 5.4 图像生成

- **GenEval**: 在生成质量评估上接近 SD3-Medium 和 FLUX 等专用图像生成模型
- **DPG-Bench**: 在文本-图像对齐度上表现良好
- 相比 Qwen2.5-Omni（不具备图像生成能力），这是全新的能力维度

### 5.5 语音生成

- **语音质量（MOS）**: 超越 Qwen2.5-Omni，接近专用 TTS 系统
- **流式延迟**: 首 token 延迟显著低于非流式方案
- **风格可控性**: 能够根据指令调整情感和语速

### 5.6 全模态联合任务

论文特别测试了需要多模态协同的任务，例如：
- "看图并语音描述"：视觉理解 + 语音生成
- "听语音回答并生成配图"：音频理解 + 文本推理 + 图像生成
- "多轮多模态对话"：在同一个对话流中交替使用不同模态

这些联合任务的结果表明，Qwen3.5-Omni 能够在不同模态间自然切换，不会出现模态混淆或能力下降。

## 6. 与前代/竞品的对比

### 6.1 与 Qwen2.5-Omni 的对比

Qwen3.5-Omni 相比 Qwen2.5-Omni 的主要提升：
- **新增图像生成能力**：Qwen2.5-Omni 只能理解和生成文本/语音，Qwen3.5-Omni 加入了图像生成
- **更强的文本推理**：受益于 Qwen3 的基础和思维链蒸馏，推理能力显著提升
- **更好的语音质量**：语音生成的自然度和表现力提升明显
- **流式生成优化**：语音流式输出的延迟进一步降低
- **理解能力全面升级**：视觉理解、音频理解等各维度均有提升

### 6.2 与 GPT-4o 的对比

Qwen3.5-Omni 在多个基准上与 GPT-4o 展开了正面竞争：
- 文本推理：部分基准接近或达到 GPT-4o 水平
- 视觉理解：在部分任务上与 GPT-4o 接近
- 图像生成：与 GPT-4o 的差距缩小
- 语音交互：在流式延迟上具备优势

### 6.3 与 Gemini 2.0 的对比

- 在多模态理解基准上与 Gemini 2.0 Flash 相当
- 在图像生成质量上仍有差距
- 在端到端语音交互的流畅度上具有竞争力

### 6.4 与专用模型的对比

- **vs 专用 LLM（Qwen3-32B）**: 文本能力几乎无损
- **vs 专用 VLM（Qwen2.5-VL）**: 视觉理解能力相当
- **vs 专用 TTS**: 语音质量接近但灵活度更高
- **vs 专用文生图模型**: 图像生成质量仍有一定差距，但作为大一统模型的能力已经令人印象深刻

## 7. 局限性和未来方向

### 7.1 当前局限

1. **图像生成质量**：虽然接近专用模型，但在复杂场景（多人、细粒度文字渲染、精确布局控制）上仍不如 FLUX.1 或 SD3 等专用图像生成模型。图像生成的分辨率上限也有待提升。

2. **视频生成缺失**：Qwen3.5-Omni 目前不支持视频生成，这是一个明显的能力缺口。视频生成需要处理时序一致性、运动物理规律等额外挑战。

3. **长视频理解**：虽然支持视频输入，但对于超长视频（>30 分钟）的理解仍受限于上下文窗口。

4. **多模态输出协调**：在同时输出文本、语音和图像时，不同模态输出之间的时序协调和内容一致性还有优化空间。

5. **实时交互延迟**：虽然流式生成降低了首字延迟，但在复杂推理场景下（如需要长时间思维链），整体响应时间仍然较长。

6. **幻觉问题**：多模态生成中仍存在幻觉，特别是图像生成中可能产生与文本指令不完全一致的内容。

### 7.2 未来方向

1. **视频生成**：将生成能力扩展到视频是最自然的下一步，需要在架构中引入时序建模模块。

2. **更高分辨率和更高质量的图像生成**：通过改进图像 tokenizer 和扩大生成训练数据来提升图像生成质量。

3. **更强的实时交互能力**：进一步降低延迟，支持真正的全双工语音对话（即同时听和说）。

4. **Agent 能力**：结合全模态感知和生成能力，使模型能够作为自主 agent 操作图形界面、处理多模态信息流。

5. **模态间对齐优化**：研究更精细的跨模态对齐方法，减少多模态输出中的不一致和幻觉。

6. **端侧部署**：研究模型压缩和加速技术，使全模态能力可以在边缘设备上运行。

## 8. 个人见解和启发

### 8.1 大一统 vs 专用化：走向融合的趋势

Qwen3.5-Omni 代表了一种重要趋势——从"专用模型拼接"走向"大一统模型"。此前的多模态系统通常是通过组装多个专用模型来实现（例如 LLM + TTS + Image Generator），这种方案的问题是模态间信息损失大、延迟高、一致性差。Qwen3.5-Omni 证明了单一模型可以在所有模态上都达到接近专用模型的水平，这意味着大一统方案在工程和产品层面的优势（更低延迟、更好一致性、更简部署）将成为决定性因素。

### 8.2 Thinker-Talker 架构的工程智慧

Thinker-Talker 的分离设计是一种非常务实的工程选择。它不是理论上最优的端到端方案，但在实践中解决了最关键的问题——训练稳定性。这提醒我们，在 AI 系统设计中，工程可行性往往比理论优雅性更重要。特别是当训练成本以百万美元计时，一个训练不稳定的"更优雅"架构远不如一个训练稳定的"更朴素"架构有价值。

### 8.3 思维链蒸馏的价值

论文中思维链蒸馏的效果令人印象深刻——28B 参数的模型通过蒸馏能够逼近 235B 模型的推理水平。这说明推理能力在很大程度上是"可转移"的，大模型的思维链中包含了可以被小模型学习到的推理模式。这对行业实践有重要启示：我们不需要在每个部署场景都运行最大的模型，通过蒸馏可以在更小、更高效的模型中保留大部分推理能力。

### 8.4 对视频生成领域的启示

作为从事视频生成相关工作的人，Qwen3.5-Omni 的架构设计给我以下几点启发：

1. **统一理解与生成是可行的**：视频生成模型不必是纯粹的生成器，它可以同时具备视频理解能力，理解能力反过来可以提升生成质量（例如通过更好地理解文本指令来指导视频生成）。

2. **思维链在生成中的应用**：让模型在生成前先"想"清楚，这种思路可以迁移到视频生成——例如在生成视频前先生成场景描述、镜头规划、运动轨迹等中间表示，然后基于这些规划进行生成，可能显著提升生成质量。

3. **流式生成的价值**：视频生成天然是序列化的，流式生成思路可以应用于视频的逐帧或逐段生成，使得用户在生成过程中就能看到部分结果，而不是等待完整视频生成完毕。

### 8.5 开放问题

尽管 Qwen3.5-Omni 取得了令人瞩目的进展，但仍有一些开放问题值得深入思考：

- **规模效应的上限**：随着参数规模和训练数据的增长，大一统模型的能力是否还能持续提升？还是会在某个点后出现瓶颈？
- **模态间干扰的解决**：尽管 Thinker-Talker 缓解了训练中的跷跷板效应，但在推理时，同时处理多模态是否会导致计算资源的竞争？
- **评估体系的完善**：现有的多模态基准大多是单模态的，缺少真正考察多模态协同能力的综合评估方案。如何设计更好的全模态评估基准是一个重要的开放问题。
- **安全性与可控性**：统一了多模态生成能力后，模型被滥用的风险也随之增大（如生成虚假图像+语音），如何在保持能力的同时确保安全可控是必须解决的问题。
