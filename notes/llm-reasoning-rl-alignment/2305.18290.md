# Direct Preference Optimization: Your Language Model is Secretly a Reward Model (Rafailov et al., 2023)

**arxiv**: 2305.18290 | **DPO 开山** | NeurIPS 2023

---

## 一、核心动机

RLHF（InstructGPT/ChatGPT 的对齐方法）虽然在实践中成功，但存在严重的工程和理论问题：

1. **训练流程复杂**：需要三个阶段（SFT → RM → PPO），每阶段都有独立的超参数
2. **PPO 训练不稳定**：RL 训练对学习率、KL 惩罚系数、价值函数初始化等非常敏感
3. **奖励模型是瓶颈**：RM 需要单独训练和保存，在 RL 过程中作为环境的一部分被反复调用，内存开销大
4. **Reward hacking**：策略可能利用 RM 的盲区，生成高分但低质的输出
5. **实现困难**：PPO 需要 4 个模型同时运行（策略、参考策略、RM、价值函数），对工程实现要求高

本文的核心问题：**能不能绕过显式的奖励模型和 RL 训练，直接从人类偏好数据优化策略？**

答案是可以——DPO（Direct Preference Optimization）通过一个巧妙的数学推导，将 RLHF 的优化目标转化为一个简单的分类损失，只需要策略模型本身，无需 RM 和 RL。

## 二、方法

### RLHF 的标准框架

回顾 RLHF 的目标：在 KL 约束下最大化奖励：

$$\max_{\pi_\theta} \mathbb{E}_{x \sim \mathcal{D}, y \sim \pi_\theta(\cdot|x)} [r(x, y)] - \beta \mathbb{D}_{KL}[\pi_\theta(\cdot|x) \| \pi_{\text{ref}}(\cdot|x)]$$

这个优化问题有闭式解：

$$\pi^*(y|x) = \frac{1}{Z(x)} \pi_{\text{ref}}(y|x) \exp\left(\frac{1}{\beta} r(x, y)\right)$$

其中 $Z(x)$ 是配分函数（与 $y$ 无关的归一化常数）。

### 关键推导：从奖励到策略

将上式重新排列，用最优策略表示奖励：

$$r(x, y) = \beta \log \frac{\pi^*(y|x)}{\pi_{\text{ref}}(y|x)} + \beta \log Z(x)$$

在 Bradley-Terry 偏好模型下（与 RM 训练相同的假设），偏好概率为：

$$p(y_w \succ y_l | x) = \sigma\left(r(x, y_w) - r(x, y_l)\right)$$

将奖励的表达式代入：

$$p(y_w \succ y_l | x) = \sigma\left(\beta \log \frac{\pi^*(y_w|x)}{\pi_{\text{ref}}(y_w|x)} - \beta \log \frac{\pi^*(y_l|x)}{\pi_{\text{ref}}(y_l|x)}\right)$$

注意 $\beta \log Z(x)$ 被消掉了！这意味着我们不需要知道配分函数，直接用策略的隐式奖励就能计算偏好概率。

### DPO 损失函数

将上述偏好概率最大化，得到 DPO 的损失函数：

$$\mathcal{L}_{\text{DPO}}(\theta) = -\mathbb{E}_{(x, y_w, y_l)} \left[\log \sigma\left(\beta \log \frac{\pi_\theta(y_w|x)}{\pi_{\text{ref}}(y_w|x)} - \beta \log \frac{\pi_\theta(y_l|x)}{\pi_{\text{ref}}(y_l|x)}\right)\right]$$

其中：
- $y_w$：人类偏好的回复（winner）
- $y_l$：人类不偏好的回复（loser）
- $\pi_\theta$：正在训练的策略模型
- $\pi_{\text{ref}}$：参考模型（通常是 SFT 模型，冻结不动）
- $\beta$：控制偏离参考模型的程度

### DPO 梯度的直觉

对 $\theta$ 求梯度：

$$\nabla_\theta \mathcal{L}_{\text{DPO}} = -\beta \mathbb{E}\left[\hat{r}(x, y_w) - \hat{r}(x, y_l)\right] \cdot \sigma(\cdot) \cdot \left[\nabla_\theta \log \pi_\theta(y_w|x) - \nabla_\theta \log \pi_\theta(y_l|x)\right]$$

直觉上：
- 增加 $y_w$ 的概率（$\nabla_\theta \log \pi_\theta(y_w|x)$ 项）
- 降低 $y_l$ 的概率（$-\nabla_\theta \log \pi_\theta(y_l|x)$ 项）
- 权重由当前隐式奖励差 $\hat{r}(x, y_w) - \hat{r}(x, y_l)$ 决定——如果模型已经更偏好 $y_w$，梯度就小（不需要再推）；反之梯度大

这与 RLHF 的 PPO 梯度类似，但无需单独的 RM 和价值函数。

### DPO vs RLHF 的对比

| 方面 | RLHF | DPO |
|------|------|-----|
| 训练阶段 | 3 阶段（SFT → RM → PPO） | 2 阶段（SFT → DPO） |
| 需要的模型数 | 4（策略、参考、RM、价值函数） | 2（策略、参考） |
| 是否需要 RL | 是（PPO） | 否（简单分类损失） |
| 训练稳定性 | 对超参数敏感 | 相对稳定 |
| 内存需求 | 高（4 个模型） | 中（2 个模型） |
| 奖励模型 | 需要显式训练 | 隐式存在于策略中 |

## 三、实验结果

### 文本摘要任务

在 TL;DR 数据集上：

| 方法 | 人类偏好胜率 vs SFT |
|------|-------------------|
| SFT | — |
| PPO (RLHF) | 67% |
| DPO | **71%** |

DPO 的偏好胜率甚至略高于 PPO。

### 对话任务

在 Anthropic HH（Helpful and Harmless）数据集上：

| 方法 | 偏好胜率 vs SFT |
|------|----------------|
| PPO | 64% |
| DPO | **66%** |

DPO 再次略优于 PPO。

### 自动评估

使用 GPT-4 作为评判：
- DPO 在摘要和对话任务上都匹配或超越 PPO
- 在更难的 prompt 上，DPO 的优势更明显

### 消融实验

1. **$\beta$ 的影响**：
   - $\beta$ 太小：策略偏离参考模型太远，可能生成低质量输出
   - $\beta$ 太大：策略几乎不偏离参考模型，学习不足
   - $\beta \approx 0.1 - 0.5$ 通常是甜点

2. **参考模型的选择**：
   - 用 SFT 模型作为参考是标准做法
   - 用预训练模型作为参考效果差（参考分布与偏好数据差异太大）

3. **数据质量**：
   - DPO 对偏好数据中的噪声比 PPO 更鲁棒
   - 但偏好标注的错误方向（$y_w$ 和 $y_l$ 标反）会显著损害 DPO

### 隐式奖励分析

通过 $\hat{r}(x, y) = \beta \log \frac{\pi_\theta(y|x)}{\pi_{\text{ref}}(y|x)}$ 提取的隐式奖励，与显式训练的 RM 的奖励高度相关（相关系数 > 0.9）。这说明 DPO 的策略确实编码了奖励信息。

## 四、个人评价

DPO 是 LLM 对齐领域的范式转换之作：

1. **数学推导的优雅**：DPO 的推导虽然只有几步，但极其精妙——将 RL 问题转化为分类问题的关键是利用了 KL 约束 RL 的闭式解和 Bradley-Terry 模型的配对消去。这个推导不是 trivial 的，它深刻理解了 RLHF 的数学结构，然后发现了一个之前被忽视的等价形式。

2. **实践影响巨大**：DPO 让 RLHF 从"只有大公司能做的复杂工程"变成了"任何人都能跑的简单训练"。这直接推动了开源 LLM 对齐的发展——几乎所有开源 LLM（LLaMA-Chat, Mistral-Instruct, Zephyr 等）都采用了 DPO 或其变体。

3. **"Your Language Model is Secretly a Reward Model"**：这个标题一语中的——DPO 揭示了策略模型本身就可以充当 RM。这不是巧合，而是 KL 约束 RL 的数学性质。这个洞察也启发了一系列后续工作。

4. **后续发展**：
   - **IPO**（Azar et al., 2023）：指出 DPO 在偏好确定性高时可能过拟合，提出用更鲁棒的损失函数
   - **KTO**（Ethayarajh et al., 2024）：只需"好/坏"信号而非成对偏好，大幅降低数据需求
   - **ORPO**：将 SFT 和偏好优化合并为一步
   - **SimPO**：去掉参考模型，用策略自身的长度归一化 logprob 作为隐式奖励
   - **DPO-Positive**：修正 DPO 中"惩罚 $y_l$"可能带来的副作用
   - **Iterative DPO**（如 DeepSeek-V2）：多轮 DPO，每轮用当前策略生成新偏好数据

5. **DPO 的局限**：
   - **离线性质**：DPO 只能用固定的偏好数据训练，不能像 PPO 那样在线采样。如果偏好数据与当前策略的输出分布差距大，DPO 效果可能受限。
   - **长度偏差**：DPO 倾向于生成更长的回复，因为较长的 $y_w$ 通常有更低的 $\log \pi_\theta$，导致隐式奖励更高。SimPO 和 R-DPO 试图修复这个问题。
   - **只适用于成对偏好**：需要 $y_w$ 和 $y_l$ 的配对数据，不能直接用"好/坏"信号。KTO 放宽了这个限制。
   - **理论上的假设**：DPO 假设偏好服从 Bradley-Terry 模型，且最优策略有闭式解。这些假设在实际中可能不完全成立。

6. **DPO vs PPO 的争论**：DPO 在学术基准上匹配 PPO，但在大规模部署中 PPO 可能仍有优势——因为 PPO 可以在线探索，发现并修正策略的弱点。目前共识是：DPO 适合中小规模对齐，PPO 适合大规模精细对齐。DeepSeek-R1 的 GRPO 则是另一种折中。

7. **更深层的影响**：DPO 的核心思想——将 RL 问题转化为更简单的监督学习问题——不限于对齐。类似的思想也被用于其他场景：
   - **Diffusion DPO**：将 DPO 用于扩散模型的人类偏好对齐
   - **RL from AI Feedback (RLAIF)**：用 AI 生成偏好数据 + DPO 训练

DPO 证明了一件事：在深度学习中，如果一个问题看起来需要 RL，也许只是因为还没有找到正确的参数化方式。换个视角，RL 可能变成简单的分类。
