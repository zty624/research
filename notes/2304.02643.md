# Segment Anything (Kirillov et al., 2023)

**arxiv**: 2304.02643 | **关键词**: 分割基础模型, 零样本迁移, 提示学习, 数据引擎 | Meta AI Research, FAIR | ICCV 2023

---

## 一、核心问题

如何为图像分割构建一个基础模型（foundation model）？NLP 领域的大语言模型通过在大规模数据上预训练，实现了强大的零样本和少样本泛化能力，但在计算机视觉的分割任务中，缺乏类似的基础模型。核心挑战在于三个方面：

1. **任务定义**：什么样的分割任务能支持零样本泛化？
2. **模型架构**：什么样的架构能支持灵活提示并实现实时推理？
3. **训练数据**：分割 mask 在互联网上并不像文本那样自然丰富，如何获取大规模分割数据？

本文提出 Segment Anything (SA) 项目，通过定义可提示分割任务（promptable segmentation）、设计 SAM 模型、以及构建数据引擎来系统性地解决这三个问题。

## 二、方法

### 2.1 可提示分割任务（Promptable Segmentation）

受 NLP 中 next token prediction 任务的启发，定义**可提示分割任务**：给定任意分割提示（前景/背景点、边界框、mask、自由文本等），返回一个有效的分割 mask。

关键设计原则：**"有效"输出**——即使提示是模糊的（如一个点可能指向衬衫或穿着衬衫的人），模型也应输出至少一个合理对象的 mask。这类似于期望语言模型对模糊提示仍能生成连贯回复。

该任务同时作为：
- **预训练目标**：模拟交互式分割的提示序列进行训练
- **零样本迁移接口**：通过 prompt engineering 解决下游分割任务

### 2.2 Segment Anything Model (SAM)

SAM 由三个组件构成：

**Image Encoder**：使用 MAE 预训练的 ViT-H/16，对每张图像编码一次，生成 image embedding。图像编码器计算成本高但可分摊。

**Prompt Encoder**：
- 稀疏提示（点、框、文本）：点/框使用位置编码 + 可学习的类型嵌入，文本使用 CLIP 的文本编码器
- 密集提示（mask）：通过卷积嵌入，逐元素加到 image embedding 上

**Mask Decoder**：轻量级设计，基于修改的 Transformer decoder block：
- Prompt self-attention + 双向 cross-attention（prompt-to-image 和 image-to-prompt）
- 运行 2 层后，上采样 image embedding，MLP 将 output token 映射为动态线性分类器
- 每个提示在 ~50ms 内生成 mask（在浏览器 CPU 上运行）

**歧义消解**：对单个模糊提示预测 **3 个 mask**（覆盖"整体-部分-子部分"三层嵌套），训练时仅回传与 GT 最小损失的 mask。

**损失函数**：$\mathcal{L} = \mathcal{L}_\text{focal} + \mathcal{L}_\text{dice}$，每个 mask 附带 IoU 置信度分数。

### 2.3 数据引擎

由于互联网上缺乏大规模分割标注，采用三阶段数据引擎迭代收集：

1. **辅助手动阶段**：专业标注员在 SAM 辅助下点击标注，每 mask 从 34s 降至 14s，收集 4.3M masks / 120k 图像
2. **半自动阶段**：用目标检测器自动检测高置信度 mask，标注员专注补充剩余对象，增加 mask 多样性，收集 5.9M masks / 180k 图像
3. **全自动阶段**：用 32×32 网格点提示 SAM，利用歧义感知模型为每个点预测多个 mask，经过 IoU 筛选 + 稳定性筛选 + NMS，平均每张图生成 ~100 个高质量 mask

### 2.4 SA-1B 数据集

最终数据集：11M 图像，1.1B masks，99.1% 由全自动阶段生成。图像平均分辨率 3300×4950。比最大已有分割数据集 Open Images 大 11× 图像、400× masks。

## 三、实验结果

### 零样本单点分割
- 在 23 个多样化分割数据集上，SAM 在 16/23 数据集上超过最强交互分割器 RITM
- 人工评分：SAM mask 质量评分 7-9 分（10 分制），显著高于 RITM
- Oracle 选择（3 个预测中选最佳）时，SAM 在所有数据集上都优于 RITM

### 零样本边缘检测
- BSDS500 上 ODS=0.768，接近专门的边缘检测方法 HED (0.788) 和 EDDET (0.840)

### 零样本实例分割
- 在 LVIS v1 上，用 ViTDet-H 检测框提示 SAM，AR@1000 达 59.3（接近 ViTDet-H 的 63.0），零样本即可用于实例分割

### 零样本目标提案
- SAM mask AR@1000 在常见/稀有类别上分别为 74.4/54.7

### 公平性分析
- SAM 在不同性别呈现、年龄组、肤色上的分割性能差异不显著（95% 置信区间重叠）

## 四、个人评价

### 影响
- **开创性的视觉基础模型**：SAM 是首个成功将"基础模型"范式引入图像分割的工作，定义了可提示分割这一新任务范式，影响了后续大量研究（如 SAM 2、Grounded SAM、Semantic SAM 等）
- **数据引擎范式**：模型-数据协同迭代的 data engine 成为后续视觉基础模型数据构建的标准范式
- **SA-1B 数据集**：1.1B masks 的数据集是目前最大的分割数据集，成为训练分割模型的重要资源
- **生态影响**：SAM 的开源极大地推动了分割相关应用的发展，从医学影像到遥感分析

### 局限
- **文本提示能力有限**：论文承认文本到 mask 的能力仅为初步探索，远未成熟
- **精细分割不足**：对极小物体、高度遮挡物体的分割质量有待提升
- **实时性**：Image encoder 推理较慢（ViT-H），对实时应用场景需要更小的编码器
- **语义理解缺失**：SAM 不理解分割对象的语义类别，仅做 mask 生成
- **数据偏见**：SA-1B 中非洲和低收入国家图像仍然不足，尽管 mask 绝对数量远超此前数据集
