# Self-Instruct: Aligning Language Models with Self-Generated Instructions (Wang et al., 2023)

**arxiv**: 2302.04761 | **关键词**: 自指令, 指令微调, 自举, 数据生成, LLM 对齐 | University of Washington | ACL 2023

---

## 一、核心问题

大语言模型虽具备强大的语言能力，但若不经指令微调（instruction tuning），往往无法按用户指令行事。指令微调的关键瓶颈在于**高质量的指令-回复配对数据**，这类数据通常依赖人工编写，成本高、规模有限。本文提出核心问题：

**能否用 LLM 自身生成的指令数据来提升其指令遵循能力？**

答案是可以。Self-Instruct 提出了一种自举（bootstrapping）框架，从少量人工种子指令出发，让 LLM 自行生成大量指令及对应实例，构建指令微调数据集。

---

## 二、方法

### 2.1 Self-Instruct 流水线

整体流程为四步循环：

1. **指令生成**：从种子指令池中采样少量指令作为 prompt，让 LLM 生成新指令
2. **指令过滤**：用 ROUGE-L 去重，过滤与已有指令过于相似的新指令；同时过滤非指令类输出
3. **实例生成**：对每条通过过滤的指令，让 LLM 生成对应的输入-输出实例（支持 input-first 和 output-first 两种策略）
4. **实例过滤**：过滤空实例和重复实例，保留高质量数据

通过多轮迭代，从 175 条种子指令自举生成约 52K 条指令数据。

### 2.2 种子指令集

- 175 条人工编写的高质量指令，覆盖 35 个任务类别
- 每条指令包含：指令文本 + 一个输入-输出实例
- 作用：为 LLM 提供指令格式的示范，引导生成风格一致的新指令

### 2.3 生成细节

- **指令生成**：从种子池采样 6 条指令（few-shot prompt），让 GPT-3 生成新指令
- **分类判断**：先让 LLM 判断指令属于分类任务还是非分类任务
- **实例生成**：
  - 分类任务：使用 input-first 策略（先生成输入，再生成输出）
  - 非分类任务：使用 output-first 策略（先生成输出，再反推输入）
- **去重阈值**：ROUGE-L > 0.7 时视为重复

### 2.4 微调

将 Self-Instruct 生成的数据格式化为 SUPERNATURAL INSTRUCTIONS 的格式，用标准监督微调训练 GPT-3 (175B)。整个过程中，种子指令仅占训练数据的极小比例，绝大部分数据由模型自身生成。

## 三、实验结果

### 3.1 数据集统计

| 阶段 | 数量 |
|------|------|
| 种子指令 | 175 |
| 生成指令总数 | ~52K |
| 过滤后指令 | 52,445 |
| 过滤后实例 | 82,639 |

### 3.2 核心对比

在 SUPERNATURAL INSTRUCTIONS 基准上：

| 模型 | Rouge-L |
|------|---------|
| GPT-3 (原始，无指令微调) | 0.297 |
| InstructGPT (人工标注数据微调) | 0.733 |
| Self-Instruct (自生成数据微调) | 0.459 |

Self-Instruct 相比原始 GPT-3 有显著提升（0.297 → 0.459），虽与 InstructGPT 的差距仍存在，但证明了自生成数据的有效性。

### 3.3 人工评估

- 生成的指令中约 92% 是有效的指令
- 约 54% 的实例被标注为正确
- 数据质量虽不如人工标注，但以极低成本获得大规模数据

## 四、关键洞察

1. **自举可行性**：LLM 可以作为指令数据的生成器，用自身输出改善自身能力。这是"AI 自我改进"思想在指令微调领域的早期实践。

2. **少量种子的杠杆效应**：仅需 175 条种子指令即可触发大规模数据生成，种子指令的作用是锚定格式和风格，而非提供知识覆盖。

3. **质量与数量的权衡**：自生成数据的质量不及人工标注，但规模可以弥补部分质量差距。这暗示了数据质量的"最低门槛"——只要足够好，更多数据仍有帮助。

4. **通用指令能力**：Self-Instruct 生成的数据覆盖多样化的任务类型，使模型获得泛化的指令遵循能力，而非仅限于特定任务。

5. **对后续工作的影响**：Self-Instruct 直接催生了 Alpaca（用 GPT-3.5 生成指令数据微调 LLaMA）、Vicuna、Dolly 等一系列工作，开启了"合成数据微调"的范式。

---

## 五、实验复现 (52_self_instruct.py)

模拟 Self-Instruct 流水线，验证自训练循环的效果：

- 使用温度控制（temperature）调节生成多样性
- 实现指令生成 → 过滤 → 实例生成 → 质量评估的自训练循环
- 关键结果：多轮自训练后，生成质量从 **0.30 提升至 0.51**，验证了自举机制的有效性
- 温度参数对生成多样性至关重要：过低则缺乏多样性，过高则质量下降

## 六、评价

**影响**：Self-Instruct 是指令微调领域的关键转折点。它证明了"用模型生成数据来训练模型"的可行性，直接开启了合成数据微调的浪潮，是 Alpaca、Vicuna、Open-Assistant 等工作的直接先驱。

**优点**：
- 思路简洁优雅：用 LLM 自身作为数据生成器，无需外部标注
- 极低成本：175 条种子指令即可启动整个流水线
- 实验充分：既有自动评估，也有人工评估，全面验证了数据质量
- 为开源社区提供了可行的指令微调路径

**局限**：
- 生成的数据存在偏差，倾向于模型已有知识领域的指令
- 数据质量上限受限于基础模型能力——"模型不能教自己不知道的东西"
- ROUGE-L 去重可能遗漏语义相似但措辞不同的指令
- 依赖 GPT-3 等 API，成本虽低于人工标注但仍不可忽略

**后续发展**：
- **Alpaca**：用 GPT-3.5 (text-davinci-003) 生成 52K 指令数据微调 LLaMA-7B，效果显著
- **Evol-Instruct / WizardLM**：通过进化策略逐步提升指令复杂度，克服 Self-Instruct 的多样性不足
- **DPO / RLHF**：从仅用指令数据到引入偏好对齐，进一步提升了模型质量
- **合成数据的规模化**：Llama 3 等模型大量使用合成数据进行训练，Self-Instruct 的思想已成为行业标准实践
