## 一、核心动机

State Space Models（如 Mamba）在推理时内存消耗更低、生成吞吐量更高，是 Attention 的有吸引力的替代。但社区已经积累了大量 Transformer 预训练知识，且有大量现成的预训练 Transformer 模型。为了在享受 SSM 推理优势的同时利用现有 Transformer 资产，需要有效的跨架构蒸馏方法。

然而，朴素地从 Transformer 直接蒸馏到 Mamba 会失败（PPL > 100），现有工作往往退回到 Attention+SSM 混合架构。本文的核心论点：**通过为 Mamba 提供原理性初始化，可以恢复更好的跨架构蒸馏配方**。

## 二、两阶段蒸馏方法

### 数学基础：Linear Attention 与 SSM 的对应

Softmax Attention：
$$Y_{Attn} = \text{softmax}\left(\frac{QK^T}{\sqrt{d}}\right)V$$

Linear Attention（去掉 softmax）：
$$Y_{LinAttn} = \hat{Q}\hat{K}^T \hat{V} = \hat{Q}(\hat{K}^T \hat{V})$$

SSM 的矩阵形式：
$$Y_{SSM} = A_{SSM} X, \quad [A_{SSM}]_{i,j} = C_i^T \prod_{k=i}^{j+1} \Lambda_k B_j$$

当 $\Lambda \equiv I$ 时，SSM 中的 $B, C, X$ 对应 Linear Attention 中的 $\hat{K}, \hat{Q}, \hat{V}$。这一对应关系是两阶段蒸馏的理论基础。

### Stage 1：Softmax Attention → Linear Attention（Hedgehog）

利用 Mercer 定理，将指数核 $e^{x^T x'}$ 重写为特征映射的内积 $\phi(x)^T \phi(x')$。Hedgehog 方法不使用固定的 Taylor 展开近似，而是**学习特征映射**：

$$\phi(x) \approx \phi_{MLP}(x) := \sigma(Wx + b)$$

通过匹配每个 Transformer 层的输出与 Hedgehog 线性化版本的余弦相似度来训练。仅学习特征映射参数，其余参数（MLP、LayerNorm、embedding）从教师模型直接复制并冻结。

### Stage 2：Linear Attention → Mamba（HedgeMamba）

将第一阶段学到的 Hedgehog 参数作为 Mamba 的初始化：

$$B(X) \to \hat{K}(X) := \phi_{MLP}(K(X)), \quad C(X) \to \hat{Q}(X) := \phi_{MLP}(Q(X))$$
$$\Lambda \to I, \quad X \to \hat{V}(X) := V(X)$$

关键修改：
1. Mamba 原本没有 value 变换 $X \to V(X)$，需修改实现以支持
2. 加入归一化因子（Attention scores 的分母），通过扩展 $V$ 为 $\text{concat}[V; \mathbf{1}]$ 并复制 $\Lambda$ 实现
3. Gate 分支和卷积层初始化为单位算子，不影响第一阶段学到的 Hedgehog 特征映射

然后解冻所有参数（除 embedding），用交叉熵损失微调整个架构。

## 三、实验结果

### 主结果（Pythia-1B → HedgeMamba-1B，10B 蒸馏 token）

| 模型 | PPL↓ | Arc-C↑ | SIQA↑ | Lambada↑ | HSwag↑ |
|------|-------|--------|-------|----------|--------|
| Pythia (Teacher) | 13.86 | 27.04 | 39.86 | 42.07 | 47.16 |
| Hedgehog (Baseline) | 14.89 | 26.45 | 38.38 | 30.60 | 40.79 |
| **HedgeMamba (Ours)** | **14.11** | **27.13** | **39.76** | **32.31** | **41.87** |

HedgeMamba 的 PPL 为 14.11（vs 教师 13.86），在仅用教师训练 token 量的 2.7%（10B vs 334B）下保留了大部分性能。

### Mamba 组件消融

逐步添加 Mamba 组件的效果：

| 组件 | PPL↓ | Lambada↑ | HSwag↑ |
|------|-------|----------|--------|
| Hedgehog 基线 | 14.89 | 30.60 | 40.79 |
| +SSM（可学习因果掩码 $\Lambda$） | 14.89 | 31.24 | 51.38 |
| +Conv（短卷积） | 14.89 | 31.63 | 51.78 |
| +Gate（SiLU 门控） | **14.58** | **32.16** | **50.99** |

Gate 分支贡献最大，与近期研究（gated attention 改善性能）一致。

### Token 预算分配

| S1/S2 分配 | PPL↓ |
|-------------|-------|
| 100/0（仅 Stage 1） | 25.71 |
| 50/50 | 14.58 |
| 25/75 | 14.25 |
| **10/90** | **14.11** |
| 0/100（仅 Stage 2） | 17.08 |

最优分配为 10/90——Stage 1 只需少量 token 学习特征映射，大部分预算用于 Stage 2 微调。但两个阶段都不可或缺。

### 蒸馏 Token 规模缩放

| Token 预算 | PPL↓ |
|------------|-------|
| 1B | 16.56 |
| 2B | 15.61 |
| 3B | 15.15 |
| **10B** | **14.11** |

随 token 预算增加持续改善，10B 时尚未饱和。

## 四、与相关工作的对比

- **MOHAWK**（Bick et al., 2024）：三阶段配方，更复杂，使用 Phi 架构和 C4 数据集（包含版权材料 Book3）。HedgeMamba 是两阶段配方，更直接，理论基于 Mercer 定理
- **LoLCATs**：同样基于 Hedgehog，但加入窗口注意力和 LoRA，规模和设置不同，不可直接比较
- **朴素蒸馏**：直接从 Transformer 蒸馏到 Mamba，PPL > 100，完全失败

## 五、个人评价

这篇论文的核心贡献是**原理性初始化**的想法——不是强行跨架构迁移知识，而是找到一个数学上的桥梁（Linear Attention），让两个架构的参数空间有意义的对齐：

1. **两阶段设计理论扎实**：基于 Mercer 定理和 SSM-LinearAttention 的对应关系，不是经验性的 hack
2. **纯 SSM 学生**：不依赖 Attention-SSM 混合，实现了完全的推理效率优势
3. **Token 效率**：仅用教师训练量的 2.7% 就能保留大部分性能
4. **消融实验详尽**：混合架构组件、token 分配、规模缩放三个维度

不足：
- 仅在 Pythia 系列上验证，尚未在其他 Transformer 架构上测试
- Mamba 的 selective scan 实现有模型维度 256 的硬限制，导致 1B 模型训练时间膨胀 8x+
- 训练数据仅用 OpenWebText，未研究数据质量对蒸馏效果的影响
- 下游任务提升不均匀（Lambada 从 42→32 仍有较大差距）

这项工作为跨架构蒸馏提供了一个清晰的范式：找到架构间的数学桥梁，利用桥梁进行初始化，再端到端微调。这个思路可以推广到其他架构对之间的蒸馏。
