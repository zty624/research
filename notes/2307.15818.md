# RT-2: Vision-Language-Action Models Transfer Web Knowledge to Robotic Control (Brohan et al., 2023)

**arxiv**: 2307.15818 | **VLA 开山** | Google DeepMind | 2023

---

## 一、核心问题

大视觉语言模型（VLM）在语义理解、视觉推理上展现出强大能力，但机器人控制需要的是低层动作输出（如笛卡尔末端执行器命令），而非自然语言。核心问题：

**能否将 VLM 的网络规模预训练知识直接迁移到低层机器人控制中，实现更强的泛化和涌现能力？**

此前的方案要么用 VLM 做高层规划（输出文本指令给底层控制器，底层控制器不受益于 VLM 的语义知识），要么从头设计新架构。RT-2 提出了一条更简洁的路径：**直接将机器人动作表示为文本 token，与自然语言一起训练**。

## 二、方法

### 核心思路

将机器人动作编码为文本 token，使 VLM 在输出语言和输出动作之间没有架构差异。动作成为"另一种语言"。

### 动作 tokenization

6DoF 末端执行器位移 + 夹爪开合 + 终止命令，共 8 个维度：
- 连续维度均匀离散化为 256 个 bin
- 每个 bin 对应一个 token
- 输出格式：`"terminate Δpos_x Δpos_y Δpos_z Δrot_x Δrot_y Δrot_z gripper_extension"`
- 示例：`"1 128 91 241 5 101 127"`

### 模型架构

基于两个预训练 VLM：
- **RT-2-PaLI-X**：5B / 55B 参数，数字 0-1000 各有独立 token
- **RT-2-PaLM-E**：12B 参数，覆盖 256 个最低频 token 作为动作 token

### Co-Fine-Tuning

关键训练细节：**不是只用机器人数据 finetune，而是将机器人数据与原始 web 数据混合训练**。这避免了对 VLM 语义知识的灾难性遗忘，同时学习动作生成。

每个 batch 中通过增加机器人数据的采样权重来平衡比例。

### 输出约束

推理时，当输入是机器人任务 prompt 时，限制输出词表只采样合法动作 token；对于标准 VQA 任务仍允许完整词表。

### 实时推理

55B 模型部署在云 TPU 服务上，通过网络查询，控制频率 1-3 Hz。5B 模型约 5 Hz。

### Chain-of-Thought 推理

在 PaLM-E 版本上，增加"Plan"步骤：先输出自然语言规划，再输出动作 token。

示例：`"Instruction: I'm hungry. Plan: pick rxbar chocolate. Action: 1 128 124 136 121 158 111 255."`

## 三、实验结果

### 泛化性能（6K 评估轨迹）

| 方法 | Seen Tasks | Unseen Objects | Unseen Backgrounds | Unseen Environments |
|------|-----------|---------------|-------------------|-------------------|
| RT-1 | ~62% | ~55% | ~50% | ~30% |
| MOO | ~55% | ~45% | ~45% | ~25% |
| VC-1 | ~40% | ~25% | ~25% | ~10% |
| R3M | ~25% | ~15% | ~15% | ~5% |
| **RT-2-PaLI-X** | **~62%** | **~62%** | **~60%** | **~45%** |
| **RT-2-PaLM-E** | **~60%** | **~60%** | **~55%** | **~45%** |

关键发现：在 seen tasks 上 RT-2 与 RT-1 相当，但在所有泛化场景下 RT-2 比 RT-1 和 MOO 提升约 **2x**，比 VC-1/R3M 提升约 **6x**。

### 涌现能力

RT-2 展现了三类涌现能力（均不在机器人训练数据中）：

| 能力类别 | 示例 | RT-2 vs RT-1 |
|---------|------|-------------|
| 符号理解 | "move apple to 3"、"push coke can on top of heart" | **3x+** |
| 推理 | "move apple to cup with same color"、"move X near sum of 2+1" | **3x+** |
| 人物识别 | "move coke can to person with glasses" | **3x+** |

RT-2 平均成功率是最接近基线的 **3 倍以上**。

有趣的发现：PaLM-E 版本在数学推理任务上优于 PaLI-X，归因于 PaLM-E 的预训练数据混合更偏向数学。

### 模型规模与训练策略消融

| 训练方式 | 泛化性能 |
|---------|---------|
| From scratch（5B） | 极差 |
| Fine-tune（5B） | 中等 |
| **Co-fine-tune（5B）** | **更好** |
| **Co-fine-tune（55B）** | **最好** |

- 从头训练大模型效果极差
- Co-fine-tuning 在所有规模上都优于纯 fine-tuning
- 更大模型 = 更好泛化

### Language Table 仿真

| 方法 | 成功率 |
|------|-------|
| BC-Zero | 72% |
| RT-1 | 74% |
| LAVA | 77% |
| **RT-2-PaLI-3B** | **90%** |

### Chain-of-Thought 示例

- "I need to hammer a nail" → Plan: pick rock → Action: ...
- "I'm tired, bring me a drink" → Plan: pick energy drink → Action: ...
- "Move all objects together" → Plan: move green can near green bag → Action: ...

## 四、个人评价

RT-2 的核心思想极其简洁——"动作就是另一种语言"——但效果惊人。它第一次证明了大模型从 web 数据学到的语义知识可以直接迁移到低层机器人控制中。

影响：
1. **VLA 范式的开创**：RT-2 定义了 Vision-Language-Action 模型这一新类别，此后 OpenVLA、Octo、pi0 等工作都沿用此范式
2. **Co-fine-tuning 的重要性**：证明了在机器人数据 finetune 时保留 web 数据的关键价值
3. **VLM → 机器人的直接迁移**：无需中间层或专用架构，VLM 的进步自动惠及机器人
4. **Chain-of-thought 在控制中的初步探索**：为后续"推理再行动"的机器人策略提供了起点

局限：
- **不学新动作**：VLM 预训练只迁移语义知识，机器人的物理技能仍限于训练数据中的分布
- **推理速度慢**：55B 模型仅 1-3 Hz，高频控制场景受限
- **动作离散化的信息损失**：256 bin 的离散化对精细操作可能不足
- **仅验证了移动操作场景**：未在灵巧手、导航等更广泛场景中测试
- **后续 pi0 / OpenVLA 的改进**：更小的模型、流匹配替代自回归、更丰富的数据
