# VALL-E: Neural Codec Language Models are Zero-Shot Text to Speech Synthesizers (Wang et al., 2023)

**arxiv**: 2301.02111 | **Codec LLM 开山** | ICML 2023 / arxiv 2023

---

## 一、核心问题

传统 TTS 系统采用级联架构（phoneme → mel-spectrogram → waveform），存在三个根本缺陷：

1. **中间表示瓶颈**：mel-spectrogram 是连续信号回归目标，训练需要高质量录音棚数据，网络爬取的大规模噪声数据会导致性能退化
2. **泛化能力差**：训练数据规模小（通常 ≤ 600 小时），zero-shot 场景下说话人相似度和自然度急剧下降
3. **缺少 in-context learning**：已有的 zero-shot TTS 方法要么需要 fine-tuning（speaker adaptation），要么需要复杂的 speaker encoder（speaker encoding），无法像 GPT-3 那样通过 prompting 直接适配新任务

核心洞察：**把 TTS 从连续信号回归任务转化为离散 codec 码的条件语言建模任务**，用大规模数据训练，像 GPT-3 一样涌现 in-context learning 能力。

## 二、方法

### 2.1 语音量化：Neural Audio Codec

采用预训练的 EnCodec（Défossez et al., 2022）作为 tokenizer：
- 输入 24kHz 音频，编码器输出 75Hz 的 embedding（320 倍下采样）
- 每帧 embedding 经过 **残差向量量化 (RVQ)**：8 层量化器，每层 1024 个码本条目
- 对应 6K bitrates 的配置
- 10 秒波形 → 750×8 的离散码矩阵

RVQ 的关键特性：**第一层量化器恢复声学属性（说话人身份等），后续量化器逐步建模残差细节**。这使得分层建模成为可能。

### 2.2 条件 Codec 语言建模

将 zero-shot TTS 形式化为条件语言建模：给定音素序列 $x$ 和声学 prompt 矩阵 $\tilde{C}$，最大化 $p(C|x, \tilde{C})$，其中 $C$ 是目标声学码矩阵。

**分层建模策略**——AR + NAR：

**第一阶段：自回归 (AR) 模型**，生成第一层量化器码 $c_{:,1}$：

$$p(c_{:,1} | x, \tilde{C}_{:,1}; \theta_{AR}) = \prod_{t=0}^{T} p(c_{t,1} | c_{<t,1}, \tilde{c}_{:,1}, x; \theta_{AR})$$

- Decoder-only Transformer，输入为音素序列与第一层声学码的拼接
- 音素 prompt 和声学 prompt 分别计算位置编码
- 输出投影层与声学 embedding 共享权重
- 训练时是纯因果 LM 训练（任何前缀都可视为 prompt），推理时使用 enrolled 语音的声学码作为前缀
- 使用 sampling-based 解码（beam search 会导致无限循环）

**第二阶段：非自回归 (NAR) 模型**，生成第 2-8 层量化器码：

$$p(C_{:,2:8} | x, \tilde{C}; \theta_{NAR}) = \prod_{j=2}^{8} p(c_{:,j} | C_{:,<j}, x, \tilde{C}; \theta_{NAR})$$

- 每步随机采样 stage $i \in [2, 8]$，预测第 $i$ 层码
- 8 个独立的声学 embedding 层，前 $i-1$ 层码的 embedding 求和作为输入
- 声学 prompt $\tilde{C}$ 的全部 8 层码 embedding 求和作为 prompt
- 使用 **AdaLN** 注入当前 stage 信息：$\text{AdaLN}(h, i) = a_i \cdot \text{LayerNorm}(h) + b_i$
- NAR 模型允许每个 token attend 到所有输入 token（双向 attention）
- 使用 greedy 解码

**AR + NAR 的设计动机**：
- 语速因人而异，AR 模型天然灵活处理变长序列
- 后续 7 层的输出长度已由第一层确定，NAR 可以 O(1) 生成而非 O(T)

### 2.3 In-Context Learning via Prompting

两种推理模式：
- **VALL-E**：给定文本 + 3 秒 enrolled 语音，合成该说话人说出给定文本的语音
- **VALL-E-continual**：给定 3 秒语音前缀，续写后续内容

### 2.4 训练配置

- **数据**：LibriLight，60K 小时英语语音，约 7000 个说话人（比现有 TTS 数据大数百倍）
- **标注**：原始数据无标注，用 DNN-HMM ASR 生成音素级别对齐（30ms 帧移）
- **模型**：12 层 Transformer，16 heads，embedding dim 1024，FFN dim 4096，dropout 0.1
- **训练**：随机裁剪 10-20 秒波形，16×V100 32GB，batch size 6K acoustic tokens/GPU，800K steps
- **优化**：AdamW，warmup 32K steps 至 $5 \times 10^{-4}$，然后线性衰减

## 三、实验结果

### 3.1 LibriSpeech 评测

| 模型 | WER ↓ | 说话人相似度 (SPK) ↑ |
|------|-------|---------------------|
| Ground Truth | 2.2 | 0.754 |
| GSLM (speech-to-speech) | 12.4 | 0.126 |
| AudioLM (speech-to-speech) | 6.0 | 0.337* |
| YourTTS (baseline) | 7.7 | 0.580 |
| **VALL-E** | **5.9** | **0.508** |
| VALL-E-continual | 3.8 | — |

*AudioLM 未开源，无法用同一工具评测 SPK

人类评测（40 个说话人，3 秒 enrolled）：

| 模型 | SMOS ↑ | CMOS (vs VALL-E) |
|------|--------|------------------|
| YourTTS | 3.45±0.09 | -0.12 |
| VALL-E | **4.38±0.10** | 0.00 |
| Ground Truth | 4.5±0.10 | +0.17 |

VALL-E 在 SMOS 上比 YourTTS 高 **+0.93**，CMOS 高 **+0.12**。

### 3.2 VCTK 评测

VCTK 含 108 个说话人，YourTTS 训练时见过其中 97 个，VALL-E 全部未见。

3 秒 prompt 自动评测（108 个说话人）：

| 模型 | SPK |
|------|-----|
| YourTTS* | 0.382 |
| VALL-E | **0.423** |
| Ground Truth | 0.546 |

人类评测（60 个说话人，3 秒 enrolled）：

| 模型 | SMOS ↑ | CMOS (vs VALL-E) |
|------|--------|------------------|
| YourTTS* | 3.70±0.09 | -0.23 |
| VALL-E | **3.81±0.09** | 0.00 |
| Ground Truth | 4.29±0.09 | -0.04 |

VALL-E 在 CMOS 上甚至 **超过 ground truth +0.04**（虽然统计上不显著），说明合成语音自然度与真人录音无显著差异。

### 3.3 消融实验

**NAR 模型消融**（输入为 ground truth 第一层码）：

| 设置 | WER | SPK |
|------|-----|-----|
| NAR-no prompt | 19.6 | 0.518 |
| NAR-phn prompt | 3.0 | 0.541 |
| NAR-2 prompts | **2.8** | **0.732** |

音素 prompt 主要贡献内容信息（WER 19.6→3.0），声学 prompt 主要贡献说话人信息（SPK 0.541→0.732）。

**AR 模型消融**：

| 设置 | WER | SPK |
|------|-----|-----|
| VALL-E (完整) | 5.9 | 0.585 |
| w/o acoustic prompt | 5.9 | 0.236 |

去掉 AR 模型的声学 prompt 导致说话人相似度暴跌（0.585→0.236），说明 AR 阶段的声学 prompt 对 speaker identity 至关重要。

### 3.4 定性发现

1. **多样性**：相同输入 + 不同随机种子可产生不同语速、不同重音的输出，有利于生成 ASR 伪数据
2. **声学环境保持**：prompt 有混响 → 合成也带混响（因为训练数据包含多种声学条件）
3. **情感保持**：prompt 带愤怒情绪 → 合成也保持愤怒情绪（无需情感 TTS 训练数据）

## 四、局限性

1. **合成鲁棒性**：AR 模型存在注意力对齐错误，导致偶尔出现词的遗漏、重复或替换
2. **数据覆盖不足**：60K 小时仍不足以覆盖所有口音（VCTK 上比 LibriSpeech 差），且 LibriLight 主要是朗读风格
3. **模型结构**：AR + NAR 两个模型较重，未来方向是统一大模型或全 NAR 加速
4. **安全风险**：可被滥用于语音伪造和冒充，需要检测模型配合

## 五、个人评价

VALL-E 的核心贡献不仅是"把 TTS 做成语言模型"——更重要的是**证明了语音领域也存在 scaling law + in-context learning 的范式**。具体来说：

1. **范式转换**：从 mel-spectrogram 回归到离散 codec 码的语言建模，打通了 NLP 和语音的技术路线。这一思路直接影响了后续的 VALL-E X（多语言）、SpeechGPT、AudioPaLM 等工作，乃至 2024-2025 年的语音大模型浪潮
2. **Scaling 的力量**：简单地把数据从数百小时扩展到 60K 小时，不需要精心设计的 speaker encoder 或 fine-tuning，就能实现 zero-shot voice cloning。这和 GPT-3 在 NLP 中的发现完全一致——规模涌现能力
3. **EnCodec 作为语音 tokenizer 的选择非常关键**：RVQ 的层级结构天然适合分层建模，且保留了说话人身份（不像 HuBERT codes 会丢失 speaker 信息）。后来 VALL-E 2、Bark、ChatTTS 等都采用了类似的 codec-based 方案

但也存在明显不足：
- 60K 小时相比 LLM 的训练数据仍然很少，zero-shot 的说话人相似度（0.508 vs GT 的 0.754）还有很大提升空间
- AR 模型的鲁棒性问题（WER 5.9% vs GT 2.2%）是 codec LLM 的通病，后续工作（如 VALL-E 2 的持续采样策略）才逐步缓解
- 声学环境和情感的保持是涌现出来的，但不可控——没有显式的控制机制

总体而言，VALL-E 是语音合成领域的 GPT-3 时刻，开创了"speech as language modeling"的研究范式，后续几年的语音大模型几乎都沿着这条路走。
