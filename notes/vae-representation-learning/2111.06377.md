**标记**: 开山 — Masked Autoencoder (MAE) 的奠基性工作

## 1. 理论推导

### MAE 框架概述

MAE 是一种简单的自编码方法：给定部分观测信号，重建原始信号。与经典自编码器不同，MAE 采用**非对称编码器-解码器**设计，并结合极高掩码率，形成一种高效的自监督学习框架。

### 图像分块与掩码

按照 ViT 的方式，将输入图像划分为规则的非重叠 patch（如 16x16 像素）。然后以均匀分布随机采样一个 patch 子集，不重复地移除（掩码）其余 patch。采样策略为"随机采样"——均匀随机、无替换。这种策略搭配高掩码率可以大量消除图像的空间冗余，使任务无法通过简单的外推邻近 patch 来解决。

### 非对称编码器-解码器设计

**编码器**：仅处理可见的（未被掩码的）patch 子集，不使用掩码 token [M]。编码器是标准 ViT 结构：通过线性投影加位置嵌入将 patch 映射为 token，再通过一系列 Transformer block 处理。由于只处理约 25% 的 patch，计算量和显存占用大幅降低。

**解码器**：输入是完整的 token 集合，包括：(i) 编码后的可见 patch token，(ii) 掩码 token。每个掩码 token 是一个共享的、可学习的向量，表示该位置需要预测缺失的 patch。所有 token 都加入位置嵌入（否则掩码 token 无法获知自身在图像中的位置）。解码器由另一组 Transformer block 组成，最后一层是线性投影，输出通道数等于每个 patch 的像素值数量。解码器的输出重塑为重建图像。

**预训练后丢弃解码器**：解码器仅在预训练阶段用于图像重建，推理时只保留编码器，对完整图像（全部 patch）进行特征提取。

### 重建损失

损失函数计算重建图像与原始图像在像素空间中的**均方误差（MSE）**。关键点：**损失仅在掩码 patch 上计算**，不在可见 patch 上计算（类似 BERT 的做法）。这不同于传统去噪自编码器在所有像素上计算损失。实验表明，在所有像素上计算损失会导致精度轻微下降（约 0.5%）。

论文还研究了归一化像素值作为重建目标：计算每个 patch 内所有像素的均值和标准差，用它们对该 patch 做归一化。使用归一化像素作为重建目标可以提升表征质量，因为它增强了局部对比度。

### 为什么高掩码率（75%）能迫使模型学习好的表征

论文从三个角度分析了视觉与语言在掩码自编码上的差异：

1. **架构差异**：过去卷积网络难以融入掩码 token 或位置嵌入，但 ViT 解决了这个问题。
2. **信息密度差异**：语言是人类生成的高度语义化、信息密集的信号，掩码少量词就能驱动复杂语言理解。而图像是自然信号，具有**严重的空间冗余**——缺失的 patch 可以仅通过邻近 patch 的低级统计信息恢复，无需对物体、场景的高层理解。为了克服这种冗余，需要**极高比例的随机掩码**（如 75%），这样才能大量消除冗余，创造出一个需要整体理解（beyond low-level image statistics）的自监督任务。
3. **解码器角色差异**：在视觉中，解码器重建的是像素，其输出语义层次低于识别任务；在语言中，解码器预测缺失的词，本身就包含丰富语义。因此在 BERT 中解码器可以很简单（一个 MLP），但在图像中解码器设计对学到的潜在表征的语义层次有重要影响。

实验验证：掩码率为 75% 时，linear probing 和 fine-tuning 都达到最优。从 10% 到 75%，linear probing 准确率差距高达 ~20%（54.6% vs. 73.5%）。这说明高掩码率确实迫使编码器学到了更有用的表征。

### 简洁实现

MAE 的实现不需要专门的稀疏操作：(1) 为所有 patch 生成 token；(2) 随机打乱 token 列表并按掩码率移除末尾部分；(3) 编码后，将掩码 token 追加到编码 patch 列表后，再反打乱（逆随机排列）使所有 token 与目标对齐；(4) 解码器处理完整列表。shuffle/unshuffle 操作开销可忽略。

## 3. 重要idea

### 3.1 非对称编码器-解码器是效率的关键

编码器不处理掩码 token，只处理可见 patch（25%），将 mask token 推迟到轻量级解码器中处理。这使得整体预训练计算量减少 3.3x FLOPs，实际墙钟加速 2.8x–4.1x，显存也大幅降低。编码器始终只看到真实 patch，消除了预训练（大量 mask token 输入）与部署（无掩码完整图像）之间的分布差距，对 linear probing 准确率提升达 14%。

### 3.2 75% 掩码率是最优的，与 NLP 的 15% 截然不同

BERT 的掩码率为 15%，但 MAE 的最优掩码率为 75%。根本原因在于图像与语言的信息密度差异：图像空间冗余严重，15% 掩码几乎不需要高级语义理解即可完成，75% 才能迫使模型学到真正有用的表征。在 75% 掩码率下，仅 25% patch 可见（如 196 个 patch 中仅 49 个可见），模型必须进行整体推理才能重建。

### 3.3 编码器仅处理可见 patch → 巨大计算节省

75% 掩码率下编码器只处理 25% 的 patch。由于 self-attention 复杂度为二次方，实际加速可超过 4x。这使得训练 ViT-Huge 等超大模型变得可行。同时显存消耗大幅降低，可以进一步增大 batch size 或模型规模。

### 3.4 Linear probing vs. Fine-tuning 行为差异

Linear probing 和 fine-tuning 遵循截然不同的趋势：
- **Linear probing**：准确率随掩码率增加持续提升直至最优点（10%→75% 差距 ~20%），需要足够深的解码器（解码器深度对 linear probing 影响达 8%），因为更深的解码器能承担更多重建特化，使编码器输出停留在更抽象的层次。
- **Fine-tuning**：对掩码率不那么敏感，40%–80% 范围内都表现良好，对解码器深度也不敏感（1 层解码器即可达 84.8%）。

**部分微调（Partial fine-tuning）**实验揭示关键洞察：仅微调 1 个 Transformer block 就可将精度从 73.5% 提升到 81.0%；微调最后半个 block（即 MLP 子模块）即可达 79.1%。MAE 的表征线性可分性不如 MoCo v3，但只要微调少量非线性层就全面超越 MoCo v3。这说明**线性可分性不是衡量表征质量的唯一指标**。

### 3.5 MAE 作为去噪自编码器（DAE）的视角

MAE 本质上是一种去噪自编码器（denoising autoencoder），其"噪声"即为随机掩码。但与传统 DAE 的区别在于：
- 传统 DAE 在所有像素上计算损失，MAE 仅在掩码 patch 上计算损失
- MAE 采用非对称设计，编码器不处理 mask token
- 掩码率极高（75%），远超传统 DAE 的噪声水平
- 重建目标为像素（而非 dVAE token），无需额外的 tokenizer 预训练阶段

实验表明，像素重建目标与 dVAE token 重建目标在迁移学习上表现无显著差异（Table 7），但像素方案更简单、无需额外预训练、无额外计算开销。

### 3.6 其他发现

- **数据增强**：MAE 对数据增强的需求极低，即使不做增强（仅 center-crop，无翻转）也表现不错。这与对比学习方法（MoCo、SimCLR、BYOL）形成鲜明对比——对比学习严重依赖数据增强，不用增强性能暴跌。
- **掩码采样策略**：随机采样最优。block-wise 采样（BEiT 的策略）在 75% 掩码率下退化严重；grid-wise 采样任务过于简单，表征质量较低。
- **训练时长**：MAE 在 1600 epochs 时仍未饱和（linear probing），而对比学习（如 MoCo v3）在 300 epochs 即饱和。但即使 1600 epochs，MAE 的总训练时间仍短于 MoCo v3 的 300 epochs。
- **规模扩展**：MAE 展现出优秀的扩展性——模型越大，MAE 相对于从零训练的提升越大，呈现出类似 NLP 自监督预训练的扩展规律。
