# A ConvNet for the 2020s (ConvNeXt) (Liu et al., 2022)

**arxiv**: 2201.03545 | **纯卷积现代化 开山之作** | Meta AI | CVPR 2022

---

## 一、核心问题

2020-2021 年，Vision Transformer（ViT、Swin Transformer）在视觉任务上全面超越传统卷积网络。一个核心问题浮现：**Transformer 的优势到底来自注意力机制本身，还是来自其他架构设计改进？**

ConvNeXt 的回答：纯卷积网络只要借鉴 Transformer 的现代设计选择，就能匹配甚至超越 Swin Transformer——注意力并非必要。

## 二、关键创新

### 2.1 逐步现代化策略

以 ResNet-50 为起点，逐步引入 Swin Transformer 的设计选择，每一步观察 ImageNet 精度变化。这种控制变量法清晰地揭示了每个改进的独立贡献。

### 2.2 关键改动（共 5 大类）

**1. 宏观架构比例调整**

- ResNet 原始阶段比例：3-4-6-3（每阶段的 block 数）
- 借鉴 Swin：改为 3-3-9-3，增加深层 block 数量
- 后续进一步调整为 3-3-27-3（ConvNeXt-T）

**2. 改用 Depthwise 卷积**

- 将标准 3×3 卷积替换为 Depthwise 7×7 卷积
- Depthwise 卷积每个输入通道独立做卷积，类似 Self-Attention 中每个 head 独立计算
- 7×7 感受野比 3×3 更大，且 depthwise 使得 FLOPs 增长可控
- 代价：密度降低，需要更多通道补偿

**3. Inverted Bottleneck**

- ResNet：宽 → 窄 → 宽（标准 bottleneck）
- 改为：窄 → 宽 → 窄（inverted bottleneck），与 MobileNetV2 和 Transformer 的 MLP 结构一致
- MLP 扩展比设为 4（与 Transformer FFN 一致）

**4. 激活与归一化层**

- ReLU → **GELU**（Transformer 标配）
- BatchNorm → **LayerNorm**（消除对 batch 统计的依赖）
- 大幅**减少激活和归一化层**：从每个 conv 后都有，改为仅在 inverted bottleneck 的中间层保留一个 GELU 和 LN
- 极简设计：1×1 conv → LN → 1×1 conv → GELU → 3×3 DW conv → 1×1 conv，只在开头一个 LN 和中间一个 GELU

**5. 分组/层归一化与下采样**

- 用 2×2 stride-2 卷积替代 max pooling 做空间下采样
- 下采样层前加 LayerNorm 稳定训练

## 三、方法细节

### 3.1 最终 ConvNeXt Block

```
输入
  │
  ├─ 7×7 Depthwise Conv
  │
  ├─ LayerNorm
  │
  ├─ 1×1 Conv (升维 4×)
  │
  ├─ GELU
  │
  ├─ 1×1 Conv (降维)
  │
  +─ 残差连接
```

对比 ResNet Block 的 4 个 BN + 2 个 ReLU，ConvNeXt Block 只有 1 个 LN + 1 个 GELU，极为精简。

### 3.2 模型家族

| 变体 | 通道数 C | Block 数 [3,3,27,3] | 参数量 |
|------|---------|---------------------|--------|
| ConvNeXt-T | 96 | [3,3,9,3] | 28M |
| ConvNeXt-S | 128 | [3,3,27,3] | 50M |
| ConvNeXt-B | 192 | [3,3,27,3] | 89M |
| ConvNeXt-L | 256 | [3,3,27,3] | 198M |
| ConvNeXt-XL | 384 | [3,3,27,3] | 350M |

### 3.3 训练策略现代化

除架构外，训练策略也全面对齐 ViT：
- 训练 epoch：90 → 300（加 EMA）
- 优化器：SGD → AdamW
- 数据增强：简单 RandomResizedCrop → Mixup + CutMix + RandAugment + Random Erasing
- 正则化：加 Label Smoothing + DropPath + Stochastic Depth

## 四、实验结果

### 4.1 ImageNet 分类

| 模型 | ImageNet Top-1 | 参数量 | FLOPs |
|------|---------------|--------|-------|
| Swin-T | 81.3% | 29M | 4.5G |
| **ConvNeXt-T** | **82.1%** | 28M | 4.5G |
| Swin-B | 83.5% | 88M | 15.4G |
| **ConvNeXt-B** | **83.8%** | 89M | 15.4G |
| Swin-L (384) | 86.3% | 197M | 103G |
| **ConvNeXt-L (384)** | **87.8%** | 198M | 103G |

同等 FLOPs 下，ConvNeXt 全面超越 Swin Transformer。

### 4.2 下游任务

- **COCO 目标检测/实例分割**：ConvNeXt-B 在 Cascade Mask R-CNN 上 54.0 box AP / 47.0 mask AP，优于 Swin-B 的 53.2 / 46.3
- **ADE20K 语义分割**：ConvNeXt-B 49.1 mIoU，优于 Swin-B 的 48.1
-ConvNeXt 在稠密预测任务上同样全面匹配或超越 Swin

### 4.3 逐步改动的贡献

从 ResNet-50 (76.9%) 逐步改动到 ConvNeXt-T (82.1%)：
- 改宏观比例 → +0.4%
- 改 depthwise 7×7 → +0.2%
- 改 inverted bottleneck → +0.5%
- 减少 BN/ReLU → +0.3%
- 改 GELU/LN → +0.1%
- 训练策略现代化 → +1.5%（贡献最大！）

## 五、影响

1. **Transformer 优势的解构**：ConvNeXt 证明 Transformer 的性能优势并非来自注意力机制，而是来自一系列架构设计改进（宏观比例、深度可分离卷积、inverted bottleneck、GELU/LN、精简激活层）和训练策略升级。这些改进是架构无关的

2. **纯卷积的复兴**：在 ViT 占据视觉研究主流的背景下，ConvNeXt 证明纯卷积架构仍然极具竞争力，且没有注意力机制的 $O(n^2)$ 复杂度问题，更适合高分辨率视觉任务

3. **设计方法论**：逐步改造 + 控制变量的方法论被后续工作广泛采用（如如何将 Transformer 设计迁移到其他架构）

4. **实践影响**：ConvNeXt 已成为视觉 backbone 的重要选择，尤其在需要高分辨率输入、长序列的场景（如医学影像、遥感）中比 ViT 更实用
