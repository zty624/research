**Caron, Touvron, Misra, Jegou, Mairal, Bojanowski, Joulin (Facebook AI Research / Inria, 2021)**

arxiv: 2104.14294 | 开山 — 自蒸馏无标签自监督学习框架

## 1. 理论推导

### 自蒸馏框架 (Self-Distillation with No Labels)

DINO 将自监督学习重新表述为知识蒸馏的形式：训练一个 student 网络 g_{θ_s} 去匹配 teacher 网络 g_{θ_t} 的输出，但 teacher 并非预先给定的，而是由 student 的历史版本构建。两个网络共享完全相同的架构，参数不同。

**数据增强与多裁剪 (Multi-Crop)：** 给定一张输入图像 x，通过 multi-crop 策略生成一组视图 V：包含 2 个全局视图 (x^g_1, x^g_2, 分辨率 224^2，覆盖 >50% 区域) 和若干局部视图 (分辨率 96^2，覆盖 <50% 区域)。所有裁剪送入 student，仅全局视图送入 teacher，从而鼓励 "local-to-global" 对应关系。

**损失函数：** 对每一对 (teacher 全局视图, student 不同视图) 计算交叉熵：

$$\min_{θ_s} \sum_{x \in \{x^g_1, x^g_2\}} \sum_{x' \in V, x' \neq x} H(P_t(x), P_s(x'))$$

其中 H(a, b) = -a log b。Student 和 teacher 的输出概率通过带温度的 softmax 得到：

$$P_s(x)(i) = \frac{\exp(g_{θ_s}(x)^{(i)} / τ_s)}{\sum_k \exp(g_{θ_s}(x)^{(k)} / τ_s)}$$

Teacher 端类似，使用温度 τ_t。

### Centering 与 Sharpening

**Centering (中心化)：** 在 teacher 输出上减去一个均值向量 c，防止某一维度主导输出：

$$g_t(x) \leftarrow g_t(x) + c$$

c 通过指数移动平均更新：

$$c \leftarrow m c + (1-m) \frac{1}{B} \sum_{i=1}^{B} g_{θ_t}(x_i)$$

Centering 防止 "一维坍缩" (某一维度主导)，但会鼓励输出趋向均匀分布。

**Sharpening (锐化)：** 通过给 teacher 的 softmax 使用低温 τ_t (0.04~0.07) 实现输出锐化。Sharpening 的效果与 centering 相反——鼓励输出集中到少数维度。

**两者互补：** 将交叉熵分解为熵 h 和 KL 散度：

$$H(P_t, P_s) = h(P_t) + D_{KL}(P_t \| P_s)$$

缺少 centering 时 KL→0 且熵→0 (坍缩到单一维度)；缺少 sharpening 时 KL→0 且熵→-log(1/K) (坍缩到均匀分布)。两者结合才能平衡，避免坍缩。

### Stop-Gradient 的关键作用

Teacher 网络上的 stop-gradient 操作 (t = t.detach()) 是框架的核心设计之一：梯度只通过 student 传播，teacher 不接收来自损失的梯度。Teacher 参数通过 EMA 更新：

$$θ_t \leftarrow λ θ_t + (1-λ) θ_s$$

λ 从 0.996 余弦退火到 1。如果去掉 stop-gradient（即 teacher 直接复制 student 权重），模型无法收敛，直接坍缩。这是因为 stop-gradient 打破了对称性：student 通过梯度更新快速适应，而 teacher 通过 EMA 缓慢演变，形成一种 "Mean Teacher" 式的模型集成。作者观察到 teacher 在整个训练过程中始终优于 student——这可以理解为 Polyak-Ruppert 平均的在线版本，持续提供更高质量的目标特征来指导 student 学习。

### 无监督分割特征的涌现

DINO 最惊人的发现：ViT 最后一层 [CLS] token 的 self-attention map 自动学会关注语义对象区域，无需任何像素级标注或分割监督。不同 attention head 关注不同对象或部件，直接阈值化 attention map 即可得到高质量的分割掩码。在 PASCAL VOC12 上，DINO ViT-S/8 的 attention map 与 ground truth 的 Jaccard 相似度达 45.9%，远超 supervised ViT 的 27.3%。这种分割特征在所有自监督方法中均有出现，但在 DINO+ViT 中最为显著。

### 网络架构

g = h ∘ f，其中 f 是 backbone (ViT 或 ResNet)，h 是 projection head。Projection head 为 3 层 MLP (隐藏维度 2048，GELU 激活) + L2 归一化 + 权重归一化全连接层 (输出 K=65536 维)。Student 和 teacher 架构完全相同，不使用 predictor (与 BYOL 不同)，ViT 版本完全不用 Batch Normalization。

## 2. 最小实验复现资源

| 项目 | 规格 |
|------|------|
| **预训练数据集** | ImageNet-1k (1.28M 图像，无需标签) |
| **模型架构** | ViT-S/16 (21M 参数), ViT-B/8 (85M 参数) 等 |
| **优化器** | AdamW, lr = 0.0005 × batchsize/256, cosine schedule |
| **Batch size** | 1024 (ViT-S/16, 分布在 16 GPU) |
| **硬件** | 两台 8-GPU 服务器 (16 GPU, V100) |
| **训练时间** | ViT-S/16: 300 epochs 约 60.9h (2×224²+6×96² crops); 快速版: 2×224²+10×96² crops, 100 epochs 约 24h → 74.6% linear top-1; 3天可达到 76.1% |
| **GPU 显存** | ViT-S/16: 约 9.3G~15.4G/GPU (取决于 crop 数量) |
| **关键超参** | τ_s=0.1, τ_t: 0.04→0.07 线性 warmup (前 30 epochs), λ: 0.996→1 cosine, weight decay: 0.04→0.4 cosine |
| **小 batch 可行** | batch size 128 也能训练 (单 GPU), k-NN 57.9% (100 epochs) |

**最低复现方案：** 单卡 V100, ViT-S/16, batch size 128, 2×224² crops, 100 epochs。虽然性能有所下降，但框架不需要大 batch 即可工作。

## 3. 重要idea

1. **自监督学习 = 无标签的知识蒸馏：** 将 SSL 重新表述为 student-teacher 蒸馏，teacher 由 EMA 构建，用标准交叉熵损失训练。无需负样本对 (contrastive)、无需 predictor、无需聚类约束，形式极其简洁。这统一了 BYOL 的 Mean Teacher 视角和 SwAV 的聚类分配视角。

2. **ViT 中涌现分割特征：** Self-supervised ViT 的 self-attention 自动学会语义分割，这是 supervised ViT 和自监督 CNN 都不具备的涌现特性。CNN 的分割信息需要专门方法从权重中提取，而 ViT 的 attention map 直接可视化即为分割掩码。这暗示 ViT + SSL 的组合可能接近一种 "视觉 BERT"。

3. **无需负样本 (不同于 Contrastive Learning)：** SimCLR/MoCo 依赖大量负样本对 (需要大 batch 或 memory bank)，DINO 仅靠 centering + sharpening + EMA 即可避免坍缩，对 batch size 不敏感。

4. **Centering + Sharpening 防坍缩：** 两者效果互补——centering 防止单维主导但鼓励均匀坍缩，sharpening 鼓励集中但可能导致单维坍缩。结合使用在 momentum teacher 下足够稳定。Centering 仅依赖一阶 batch 统计量 (EMA 更新)，不依赖 batch 大小，这是 DINO 对小 batch 友好的关键。

5. **与 BYOL 的联系与区别：** DINO 受 BYOL 启发，但关键差异：(a) 使用交叉熵损失而非 MSE；(b) 不需要 predictor（BYOL 中 predictor 对防坍缩至关重要）；(c) Student 和 teacher 架构完全相同；(d) Teacher 在训练中始终优于 student（BYOL 未观察到此现象）。DINO 可被视为 BYOL 的简化与推广——将 SSL 理解为 Mean Teacher 自蒸馏。

6. **Patch size 对特征质量的关键影响：** 减小 patch size (16→8→5) 不增加参数但大幅提升性能，因为更细粒度的 patch 带来更长的 token 序列，让 self-attention 有更精细的空间建模能力。这对分割等密集任务尤为重要。

7. **k-NN 友好特征：** DINO+ViT 的特征天然适合最近邻分类（ImageNet 78.3% k-NN top-1），无需线性分类器或微调即可实现强分类，这在检索场景中极具实用价值。
