# Momentum Contrast for Unsupervised Visual Representation Learning (He et al., 2020)

**arxiv**: 1911.05722 | **关键词**: 对比学习, MoCo, 动量编码器, 字典查找, 自监督学习 | Facebook AI Research (FAIR) | CVPR 2020

---

## 一、核心问题

无监督视觉表示学习的核心挑战: 如何构建**大型且一致的字典**来驱动对比学习？

对比学习可以看作**字典查找**: 编码后的查询 $q$ 应与匹配的正键 $k^+$ 相似，与所有负键 $\{k^-\}$ 不相似。好的对比学习需要:

1. **大字典**: 更多负键 → 更好的对比信号 → 更好地采样高维视觉空间
2. **一致性**: 字典中的键应由相同或相似的编码器生成，保证比较的一致性

现有方法的局限:
- **端到端 (End-to-end)**: 字典 = 当前 mini-batch，受 GPU 内存限制，字典大小受限
- **Memory Bank**: 维护整个数据集的表示，字典大但键来自不同训练步的编码器，不一致

MoCo 的核心思想: 用 **队列 (queue)** 构建大字典，用 **动量编码器 (momentum encoder)** 保证一致性。

## 二、方法

### 2.1 对比学习作为字典查找

给定编码后的查询 $q$ 和键集合 $\{k_0, k_1, \ldots\}$，假设 $k^+$ 是 $q$ 的正键。使用 **InfoNCE** 损失:

$$\mathcal{L}_q = -\log \frac{\exp(q \cdot k^+ / \tau)}{\sum_{i=0}^{K} \exp(q \cdot k_i / \tau)}$$

其中 $\tau$ 是温度超参数，$K$ 是负键数量。

### 2.2 MoCo 的两个核心设计

#### (1) 字典作为队列 (Dictionary as a Queue)

- 维护一个先进先出队列存储编码后的键
- 当前 mini-batch 的键入队，最旧的 mini-batch 出队
- **字典大小与 mini-batch 大小解耦**: 可以设置远大于 mini-batch 的字典大小
- 出队的旧键本身也有益: 它们由最过时的编码器产生，移除它们提升一致性

#### (2) 动量更新 (Momentum Update)

由于键来自不同 mini-batch，无法通过反向传播更新键编码器 $f_k$。简单地将 $f_k$ 设为 $f_q$ 效果很差，因为编码器变化太快，导致键不一致。

MoCo 使用动量更新:

$$\theta_k \leftarrow m \cdot \theta_k + (1 - m) \cdot \theta_q$$

其中 $m \in [0, 1)$ 是动量系数，$\theta_k$ 和 $\theta_q$ 分别是键编码器和查询编码器的参数。只有 $\theta_q$ 通过反向传播更新。

- $m = 0.999$ (默认): 键编码器缓慢演进，保证键的一致性
- $m = 0.9$: 进化过快，效果差
- $m = 0$: 训练发散，无法收敛

### 2.3 Pretext Task: 实例判别

使用简单的实例判别 (instance discrimination) 任务:
- 对同一图像应用两种不同的数据增强，得到查询 $x_q$ 和正键 $x_k$
- $q = f_q(x_q)$, $k^+ = f_k(x_k)$
- 负键来自队列中其他图像的编码

数据增强: 随机裁剪 → 颜色抖动 → 随机翻转 → 随机灰度化

### 2.4 Shuffling BN

Batch Normalization 在对比学习中会导致信息泄漏: 同一 mini-batch 内的样本通过 BN 共享统计量，模型可以通过 BN 统计量"作弊"来识别正键。

解决方案: **Shuffling BN**
- 训练使用多 GPU，每个 GPU 独立计算 BN 统计量
- 对键编码器的输入，在分配到 GPU 前打乱样本顺序
- 编码后恢复原始顺序
- 确保查询和正键的 BN 统计量来自不同子集

### 2.5 与其他机制的对比

| 机制 | 字典大小 | 一致性 | 计算效率 |
|------|---------|--------|---------|
| 端到端 | 受限于 mini-batch | 高 (相同参数) | 低 (需要大 batch) |
| Memory Bank | 大 (整个数据集) | 低 (不同步的编码器) | 中 |
| **MoCo** | **大 (队列)** | **高 (动量编码器)** | **高** |

## 三、实验结果

### 3.1 ImageNet 线性分类协议

| 方法 | 架构 | Top-1 |
|------|------|-------|
| InstDisc (Memory Bank) | R50 | 54.0% |
| LocalAgg | R50 | 58.8% |
| CMC | R50w2x | 63.5% |
| AMDIM-large | - | 68.1% |
| **MoCo** | **R50** | **60.6%** |
| **MoCo** | **R50w2x** | **65.4%** |
| **MoCo** | **R50w4x** | **68.6%** |

### 3.2 消融实验: 对比机制比较

| 机制 | ImageNet Top-1 |
|------|---------------|
| 端到端 (K=1024) | 54.9% |
| Memory Bank (K=65536) | 58.0% |
| **MoCo (K=65536)** | **60.6%** |

MoCo 在所有 K 值下都优于端到端和 Memory Bank。

### 3.3 消融实验: 动量系数

| 动量 $m$ | 0 | 0.9 | 0.99 | **0.999** | 0.9999 |
|-----------|---|------|-------|-----------|---------|
| Top-1 | 失败 | 55.2% | 57.8% | **59.0%** | 58.9% |

高动量（0.999）效果最好，验证了缓慢演进的编码器对一致性的重要性。

### 3.4 下游任务迁移

**PASCAL VOC 目标检测**:

| 预训练 | AP50 | AP | AP75 |
|--------|------|-----|------|
| 随机初始化 | 60.2% | 33.8% | 33.1% |
| ImageNet 监督 | 81.3% | 53.5% | 58.8% |
| **MoCo IN-1M** | 81.5% (+0.2) | 55.9% (+2.4) | 62.6% (+3.8) |
| **MoCo IG-1B** | **82.2% (+0.9)** | **57.2% (+3.7)** | **63.7% (+4.9)** |

**MoCo 在目标检测上超越了 ImageNet 监督预训练**，这是里程碑式的结果。

**COCO 检测/分割**: MoCo IG-1B 在 Mask R-CNN 上超越监督预训练 +0.5~1.7 点。

**7 个下游任务** 中 MoCo 超越监督预训练，包括 VOC/COCO 检测、COCO 实例分割、关键点检测、密集姿态估计等。

### 3.5 大规模数据

MoCo 在 Instagram-1B (约 10 亿张图片) 上预训练，效果持续提升，证明了方法在真实世界大规模数据上的可扩展性。

## 四、个人评价

**影响**: MoCo 是自监督/对比学习在视觉领域的里程碑工作，首次在多个下游任务上证明了无监督预训练可以超越有监督 ImageNet 预训练，标志着计算机视觉无监督学习的新纪元。

**优点**:
- 动量编码器 + 队列的设计优雅，解决了字典大小与一致性的矛盾
- 计算高效: 不需要超大 mini-batch (SimCLR 需要 4096+)
- 即插即用: 可与各种 pretext task 结合
- 在多种下游任务上超越监督预训练

**局限性**:
- Pretext task (实例判别) 较简单，后续 SimCLR/MoCo v2 通过更好的增强和投影头大幅提升
- 队列大小仍受内存限制
- 动量更新引入了额外的超参数 ($m$, 队列大小)
- 与 BYOL/SimSiam 等无需负样本的方法相比，MoCo 仍依赖负样本

**后续发展**:
- **MoCo v2** (Chen et al., 2020): 加入 MLP 投影头和更强增强，R50 达到 71.1%
- **MoCo v3** (Chen et al., 2021): 适配 ViT 架构
- **SimCLR** (Chen et al., 2020): 大 batch + 强增强的端到端方案
- **BYOL** (Grill et al., 2020): 无需负样本的自监督学习
- **DINO/DINOv2**: 自蒸馏方法，进一步推动视觉基础模型
- **MAE** (He et al., 2022): 掩码自编码器，与 MoCo 互补的范式
