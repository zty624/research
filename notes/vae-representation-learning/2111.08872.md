# TorchGeo: Deep Learning With Geospatial Data (Stewart et al., 2021)

**arxiv**: 2111.08872 | **遥感, 地理空间数据, PyTorch生态** | UIUC / Microsoft AI for Good | NeurIPS 2021 Workshop

---

## 一、核心问题

遥感地理空间数据对精准农业、城市规划、灾害监测和气候研究至关重要。深度学习在计算机视觉任务上取得了巨大成功，但将其应用于遥感数据面临特殊挑战：

1. 多光谱数据：卫星图像通常包含 RGB 之外的额外光谱波段
2. 地理元数据：坐标系、边界、分辨率各异，需要精确的空间对齐
3. 数据源多样：不同卫星、传感器的数据格式和特性差异大
4. 缺乏统一框架：现有工具分散，难以与 PyTorch 生态系统集成

## 二、方法

### 1. TorchGeo 整体架构

TorchGeo 是一个 Python 库，核心设计原则是**无缝集成 PyTorch 生态**：

```
torchgeo/
├── datasets/       # 预定义数据集类
├── samplers/       # 地理空间采样器
├── transforms/     # 数据增强和预处理
├── models/         # 预训练模型
└── trainers/       # 训练流程
```

### 2. 数据集设计

**预定义数据集**：覆盖 20+ 个常用遥感基准数据集，每个自动下载、解压、缓存

**可组合数据集**：无边界数据集类，支持：
- 多光谱波段选择和重采样
- 不同 CRS（坐标参考系）自动转换
- 自动裁剪和拼接
- 基于经纬度的空间查询

### 3. 地理空间采样器

标准随机/顺序采样器不适用于地理空间数据，因为：
- 训练样本不应来自同一区域的相邻像素（空间自相关导致泄漏）
- 需要保证训练/测试集在地理空间上分离

TorchGeo 提供地理感知采样器：
- **GeoSampler**：基于地理边界和分辨率采样
- 支持按区域划分 train/val/test，避免空间泄漏

### 4. 多光谱处理

- 自动归一化不同传感器的波段值范围
- 支持 Landsat、Sentinel-2、NAIP 等多种传感器
- 波段选择和重采样到统一分辨率

### 5. 与 PyTorch Lightning 集成

提供预定义的训练任务：
- 语义分割
- 目标检测
- 变化检测
- 多标签分类

## 三、实验结果

### 1. 基准实验

在多个遥感数据集上提供了标准化基准：
- LandCover.ai 语义分割
- EuroSAT 场景分类
- So2Sat 城市功能分类
- SpaceNet 建筑物检测

### 2. 可复现性

- 统一的数据预处理流程
- 标准化的训练/验证/测试划分
- 预定义的模型配置

### 3. 生态系统集成

- 与 PyTorch、torchvision、PyTorch Lightning 无缝配合
- 支持分布式训练
- 兼容 Weights & Biases 等实验管理工具

## 四、个人评价

**优点**：
1. 填补了重要空白——遥感领域长期缺乏与主流 DL 框架深度集成的工具库
2. 地理感知采样器解决了遥感 DL 中的空间泄漏问题，这是常被忽视但极其重要的问题
3. 可组合数据集设计灵活，支持跨传感器、跨分辨率的数据融合
4. 开源社区活跃，持续更新新数据集和模型

**不足**：
1. 论文更像工具文档而非研究论文，缺少方法论上的创新
2. 基准实验规模偏小，未与专业遥感工具（如 Google Earth Engine）进行性能对比
3. 对超大尺度数据（如全球范围卫星图像）的分布式处理支持有限
4. 缺少与 R 语言生态（如 terra、sf）的互操作性

**影响**：TorchGeo 已成为遥感深度学习事实上的标准 Python 库，被广泛应用于学术论文和工业项目中。它降低了遥感 DL 的入门门槛，促进了该领域的标准化和可复现性。其"地理感知采样"的理念对其他空间数据领域也有参考价值。
