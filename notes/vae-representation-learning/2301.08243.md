## 一、核心动机

Transformer 推理时的主要瓶颈是**自注意力的 KV cache**。

标准 Multi-Head Attention (MHA) 中，每个注意力头都有独立的 Key 和 Value 投影：
- Query heads: $h$ 个，每个维度 $d_k$
- Key heads: $h$ 个
- Value heads: $h$ 个

在自回归解码时，每一步需要将新 token 的 KV 向量追加到 cache 中。对于 $h$ 个头、每头 $d_k$ 维、序列长度 $L$：

$$\text{KV cache} = 2 \times h \times d_k \times L \times \text{batch\_size} \times \text{bytes}$$

以 LLaMA-2 70B 为例：$h=64$, $d_k=128$, $L=4096$, batch_size=1, FP16：
$$2 \times 64 \times 128 \times 4096 \times 2 \text{ bytes} = 128\text{MB/seq}$$

当 batch_size 增大或序列更长时，KV cache 迅速成为内存瓶颈。

**Multi-Query Attention (MQA)**（Shazeer, 2019）的方案：所有 Query 头共享一组 Key 和 Value：
- KV cache 减少到 $1/h$，即 2MB/seq
- 但质量有明显下降

核心问题：**MHA 质量好但 KV cache 大，MQA KV cache 小但质量差。能否找到折中？**

## 二、方法

### Grouped-Query Attention (GQA)

GQA 的核心思想：将 $h$ 个 Query 头分成 $G$ 组，每组共享一组 Key-Value 头。

- $G = h$：退化为 MHA（每头独立 KV）
- $G = 1$：退化为 MQA（所有头共享 KV）
- $1 < G < h$：GQA，在 MHA 和 MQA 之间插值

具体地，设 $h_q = h$ 为 Query 头数，$h_{kv} = G$ 为 KV 头数：

$$\text{head}_i^{(q)} = x W_i^Q, \quad i = 1, \ldots, h_q$$
$$\text{head}_j^{(k)} = x W_j^K, \quad j = 1, \ldots, h_{kv}$$
$$\text{head}_j^{(v)} = x W_j^V, \quad j = 1, \ldots, h_{kv}$$

第 $i$ 个 Query 头对应的 KV 头索引为 $\lfloor i \cdot h_{kv} / h_q \rfloor$。

KV cache 减少到原来的 $G/h$ 倍。例如 $h_q = 64, G = 8$ 时，KV cache 只有 MHA 的 1/8。

### 从 MHA Checkpoint 到 GQA 的上采样

本文的另一贡献：**如何将已有的 MHA 模型转换为 GQA 模型**，而无需从头训练。

很多场景下已有训练好的 MHA 模型（如 T5, LLaMA 等），想用 GQA 但不想完全重新训练。方法：

**Key/Value 上采样**：将 $h_{kv}$ 个 KV 头复制扩展到 $h_q$ 个，然后做少量额外的预训练。

具体步骤：
1. 取原始 MHA 模型的 $h_q$ 个 KV 头
2. 将它们分成 $G$ 组，每组 $h_q/G$ 个头
3. 每组用组内均值（mean pooling）初始化该组的共享 KV 头
4. 用原始训练数据继续预训练一小段（original pretraining 的很小比例）

这种"从 MHA 到 GQA 的转换"方法在实践中非常有效——只需很少的额外训练就能恢复质量。

### 实验配置

- 基于 T5 和 PaLM 架构进行实验
- 比较三种配置：MHA ($G=h$), GQA ($G=8$), MQA ($G=1$)
- 模型规模从 base 到 XXL

## 三、实验结果

### T5 架构

| 配置 | KV 头数 | BLEU (WMT) | SQuAD F1 |
|------|--------|------------|----------|
| MHA | 64 | baseline | baseline |
| GQA-8 | 8 | -0.1 | -0.2 |
| MQA | 1 | -1.2 | -0.8 |

GQA-8 与 MHA 质量几乎相同，KV cache 只有 1/8。MQA 质量明显下降。

### 上采样实验

从 MHA checkpoint 转换为 GQA：

| 方法 | 额外训练量 | 质量恢复 |
|------|-----------|---------|
| 直接上采样（mean pool 初始化） | 0 | 接近 MHA |
| 上采样 + 5% 原始预训练 | 5% | 完全恢复 |
| 从头训练 GQA | 100% | 同等 |

关键发现：从 MHA 到 GQA 的上采样 + 少量预训练（5%）就能完全恢复 MHA 的质量。这意味着已有的大规模 MHA 模型可以低成本转换为 GQA。

### 推理速度

| 配置 | KV cache 大小 | 解码速度 |
|------|-------------|---------|
| MHA | 1× | 1× |
| GQA-8 | 1/8× | 2-3× |
| MQA | 1/64× | 3-4× |

GQA 在质量和速度之间取得了好的平衡。

### 生成质量

在摘要生成和翻译任务上：
- GQA 与 MHA 的 ROUGE/BLEU 差异在统计误差范围内
- MQA 在长文本生成上有明显的质量下降
- GQA 的生成多样性与 MHA 相当

## 四、个人评价

GQA 是一个看似简单但影响深远的工程优化：

1. **简单到极致**：GQA 的核心思想——"把 Query 头分组共享 KV"——几乎不增加任何实现复杂度，但带来了 8 倍的 KV cache 减少和 2-3 倍的推理加速。这种"简单但有效"的工作在 LLM 部署中价值巨大。

2. **改变了 LLM 架构设计**：GQA 之后，几乎所有主流 LLM 都采用了 GQA 或类似的 KV 共享方案：
   - **LLaMA-2 70B**：GQA-8
   - **LLaMA-3**：GQA-8
   - **Mistral 7B**：GQA（KV heads = 8, Q heads = 32）
   - **Mixtral 8x7B**：GQA
   - **Gemma 2**：GQA
   这说明 GQA 已经成为标准实践，正如 LayerNorm 之于 BatchNorm。

3. **与 MQA 的本质区别**：MQA 把所有头共享一组 KV，信息瓶颈太严重。GQA 允许不同组关注不同的 KV 子空间，保留了多头注意力的核心优势。$G=8$ 这个甜点在多个模型和任务上被验证。

4. **上采样的实用价值**：从 MHA 转换到 GQA 只需 5% 额外训练，这对已有大规模模型的迁移非常重要。但后来从头训练 GQA 成为更常见的做法——因为新模型直接设计 GQA 架构更简洁。

5. **与 KV cache 优化的其他方向**：
   - **PagedAttention**（vLLM）：操作系统式的 KV cache 内存管理
   - **Sliding Window Attention**（Mistral）：只 cache 最近的 KV
   - **KV cache 压缩**：量化、剪枝、蒸馏
   - **Cross-layer KV 共享**：相邻层共享 KV（如 DeepSeek-V2 的 MLA）
   GQA 与这些方法正交且互补，通常同时使用。

6. **$G$ 的选择**：$G=8$ 在实践中被广泛采用，但最优 $G$ 取决于具体场景：
   - 推理密集型场景（大 batch、长序列）：倾向更小的 $G$
   - 质量优先场景：倾向更大的 $G$
   - $G/h_{kv}$ 的比值通常在 4-8 之间

7. **局限**：
   - GQA 减少了 KV cache 但不减少计算量（attention 计算仍是 $O(n^2 d)$）
   - 训练时的加速不如推理时明显（训练时 KV cache 不是瓶颈）
   - 极大规模模型（>1T 参数）可能需要更激进的 KV 共享

GQA 的成功再次验证了深度学习中一个反复出现的模式：最实用的创新往往不是理论突破，而是在约束条件（内存、速度、质量）之间找到好的平衡点。
