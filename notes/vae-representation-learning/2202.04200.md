# MaskGIT: Masked Generative Image Transformer (Chang et al., 2022)

**arxiv**: 2202.04200 | **图像生成, 双向Transformer, 迭代解码** | Google Research | CVPR 2022

---

## 一、核心问题

当前视觉生成 Transformer 的主流范式是**自回归解码**——按光栅扫描顺序逐 token 生成。这存在两个问题：

1. **非最优**：光栅扫描顺序不是最优的生成顺序，图像的 2D 结构被忽略
2. **低效**：严格逐 token 串行生成，无法利用并行计算

核心问题：能否设计一种非自回归的图像生成方法，既保持 Transformer 的建模能力，又大幅提升推理速度？

## 二、方法

### 1. MaskGIT 核心思想

借鉴 BERT 的 Masked Language Modeling，MaskGIT 采用**双向 Transformer 解码器**，通过迭代精化（iterative refinement）生成图像。

**训练**：随机 mask 部分图像 token，让模型预测被 mask 的 token（类似 MAE）

**推理**：从全 mask 开始，每步预测并保留置信度最高的 token，迭代精化

### 2. 训练过程

给定图像 token 序列 $\mathbf{y} = (y_1, \ldots, y_N)$：

1. 随机选择 mask 比例 $r \sim \text{Uniform}(0, 1)$
2. 对每个 token 以概率 $r$ 替换为 [MASK]
3. 双向 Transformer 预测所有被 mask 的 token
4. 损失仅在被 mask 的位置计算交叉熵

与 BERT 的关键区别：mask 比例 $r$ 是连续随机采样的，而非固定的 15%。这让模型学会在不同程度的遮蔽下预测，为推理时的渐进解码做准备。

### 3. 迭代解码（Cosine Schedule）

推理时从全 mask 开始，$K$ 步迭代：

1. **步 $k$**：当前 mask 比例 $r_k = \cos\left(\frac{\pi k}{2K}\right) \cdot \frac{k}{K}$...（余弦调度）
2. 模型预测所有 [MASK] 位置的 token 分布
3. 对每个预测位置计算**置信度**（最大概率值）
4. 保留置信度最高的 $(1 - r_k)$ 比例的 token
5. 其余位置重新设为 [MASK]，进入下一步

余弦调度使得前期大步填充，后期精细修整。

### 4. 温度调节

预测时使用温度 $T$ 控制随机性：
- 高 $T$：更多样但低质量的填充
- 低 $T$：更保守但高质量的填充
- 论文采用**自适应温度**：早期步用较高温度增加多样性，后期用较低温度保证质量

### 5. 条件生成与图像操控

**类条件生成**：在输入序列前添加 class token

**图像操控**：
- 将需要编辑的区域 mask 掉
- 给定目标类别
- 模型迭代填充被 mask 的区域

**图像外推**：将已有图像 token 保持，其余设为 [MASK]，迭代生成

## 三、实验结果

### 1. 类条件 ImageNet 生成

| 模型 | 分辨率 | FID↓ | IS↑ | 步数 |
|------|--------|------|-----|------|
| VQGAN + AR | 256×256 | 18.65 | 80.5 | 256 |
| VQGAN + Transformer | 256×256 | 15.78 | 78.3 | 256 |
| **MaskGIT** | **256×256** | **6.18** | **182.1** | **8-16** |
| VQ-Diffusion | 256×256 | 7.89 | 175.1 | 10 |

MaskGIT 在 FID 和 IS 上都大幅超越自回归方法，同时推理步数从 256 减少到 8-16。

### 2. 512×512 分辨率

FID 7.32，仍然保持高质量。

### 3. 推理速度

| 模型 | 步数 | 相对速度 |
|------|------|---------|
| VQGAN + AR | 256 | 1x |
| VQGAN + Transformer | 256 | 1x |
| MaskGIT (K=8) | 8 | **32x** |
| MaskGIT (K=12) | 12 | **21x** |

### 4. 图像操控

展示了高质量的条件替换（如将鸟替换为火烈鸟）和图像外推（扩展图像边界）。

## 四、个人评价

**优点**：
1. 推理速度提升巨大——从 256 步 AR 到 8 步迭代精化，32x 加速
2. 生成质量反而更好——FID 6.18 vs AR 的 18.65，说明双向注意力优于单向
3. 余弦调度 + 置信度选择的设计直觉且有效——先填充容易的，再处理难的
4. 图像操控和外推能力是 AR 方法难以实现的独特优势
5. 训练简单——只需随机 mask + 预测，无需复杂的调度

**不足**：
1. 依赖 VQGAN 量化器的质量——量化器的信息损失是瓶颈
2. 迭代步数 K 需要手动选择
3. 8 步 FID 6.18 在 2022 年已不如同期扩散模型（如 Guided Diffusion FID ~3.6）
4. 生成多样性受置信度选择策略限制——倾向于选择"安全"的高置信度 token

**影响**：MaskGIT 开创了非自回归视觉生成的迭代精化范式，直接影响了后续的 MAGE、Magvit 等工作。其核心洞察——"双向注意力 + 迭代解码比单向自回归更高效更优质"——改变了视觉生成的范式偏好。在视频生成领域，Magvit-v2 也采用了类似的 mask 建模策略。
