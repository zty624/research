## 1. 理论推导

### 对比学习框架与 NT-Xent 损失

SimCLR 的核心思想是：对同一样本施加两种不同的数据增强，得到一对正样本 (positive pair)，然后在潜在空间中最大化这对正样本的一致性，同时将其与其余负样本区分开。

给定一个 minibatch 中的 N 个样本，经两种增强后产生 2N 个数据点。对于正样本对 (i, j)，损失函数定义为：

$$\ell_{i,j} = -\log \frac{\exp(\text{sim}(z_i, z_j) / \tau)}{\sum_{k=1}^{2N} \mathbf{1}_{[k \neq i]} \exp(\text{sim}(z_i, z_k) / \tau)}$$

其中 $\text{sim}(u, v) = u^\top v / \|u\| \|v\|$ 为余弦相似度（即 l2 归一化后的点积），$\tau$ 为温度参数，$\mathbf{1}_{[k \neq i]}$ 为指示函数。最终损失是对所有正样本对 $(i,j)$ 和 $(j,i)$ 求平均。该损失被称为 **NT-Xent**（Normalized Temperature-scaled Cross Entropy Loss）。

### 温度缩放的意义

温度参数 $\tau$ 对学习效果至关重要。论文表 5 的实验表明：

- **$\tau$ 过大**（如 1.0）：对比任务准确率低（59.1%），表征质量差（top-1 58.0%）。模型难以区分难负样本。
- **$\tau$ 过小**（如 0.05）：对比任务准确率高（90.5%），但表征质量反而下降（top-1 59.7%）。模型过度关注难负样本，导致训练不稳定。
- **$\tau = 0.1$** 是最优选择，达到 64.4% top-1。

从梯度的角度理解（表 2）：NT-Xent 的梯度为 $(1 - \exp(u^\top v^+/\tau))/\tau \cdot v^+ - \sum_{v^-} \exp(u^\top v^-/\tau)/Z(u)\tau \cdot v^-$。l2 归一化配合温度参数可以有效地根据负样本的相对难度进行加权——不同于 margin triplet loss 或 logistic loss，NT-Xent 天然地对难负样本赋予更大梯度，无需显式的半难负样本挖掘。

### 数据增强的组合

论文系统研究了单一增强与增强组合的效果（图 5），核心发现：

- **没有任何单一增强足以学到好的表征**——即使模型能近乎完美地解决对比预测任务（因为任务太简单），但学到的表征泛化性差。
- **组合增强使任务变难但表征质量大幅提升**。最关键的组合是 **随机裁剪 + 颜色扰动**。
- 仅用随机裁剪时，同一图像的不同 patch 往往具有相似的颜色分布（图 6），网络可以利用颜色直方图这一捷径来区分正负样本，而无需学习高级语义特征。加入颜色扰动后，网络被迫学习形状、纹理等更本质的特征。
- 对比学习比监督学习**更需要强增强**：增强颜色扰动强度从 1/8 提升到 1（+blur），SimCLR 从 59.6% 升至 64.5%，而监督学习反而从 77.0% 降到 75.4%（表 1）。

默认增强策略：随机裁剪（resize 到 224x224，含随机水平翻转）→ 随机颜色扰动（color jitter 概率 0.8，grayscale 概率 0.2）→ 随机高斯模糊（概率 0.5，sigma 从 [0.1, 2.0] 随机采样）。

### 大 batch size 与更长训练为何有效

- 更大的 batch size 提供更多负样本（batch size 8192 时每个正样本对有 16382 个负样本），加速收敛。
- 更长的训练等价于见到更多负样本对，持续提升性能（图 9、B.2），这与监督学习不同——监督 ResNet 训练更长并无显著收益（表 B.3）。
- 当训练步数足够多时，不同 batch size 的差距缩小甚至消失（图 9），但大 batch size 在更少 epoch 时优势显著。
- 使用 LARS 优化器稳定大 batch 训练，学习率采用线性缩放（LR = 0.3 x BatchSize/256）或平方根缩放（LR = 0.075 x sqrt(BatchSize)）。

### Projection Head 设计

投影头 $g(\cdot)$ 将编码器输出 $h$ 映射到对比损失施加的空间：

$$z_i = g(h_i) = W^{(2)} \sigma(W^{(1)} h_i)$$

其中 $\sigma$ 为 ReLU，即一个两层 MLP（一个隐藏层 + 一个输出层）。关键发现：

- **非线性投影头比线性投影头好约 +3%，比无投影头好 >10%**（图 8）。
- 更反直觉的是：**下游任务应使用 $h$（投影头之前），而非 $z = g(h)$**。$h$ 的线性评估比 $z$ 好 >10%。
- 原因解释：$z$ 被训练为对数据变换不变，因此 $g$ 会移除对下游任务有用但与变换不变性无关的信息（如颜色、方向）。表 3 验证了这一点：用 $h$ 和 $g(h)$ 分别预测预训练时施加的变换，$h$ 的准确率远高于 $g(h)$（如颜色预测 99.3% vs 67.6%，旋转预测 97.4% vs 25.6%）。非线性投影头使 $h$ 可以保留更多信息，而将不变性约束"吸收"在 $g$ 中。

投影头输出维度对性能影响不大（从 32 到 2048 均可，论文默认 128 维）。

## 2. 最小实验复现资源

| 项目 | 规格 |
|------|------|
| **预训练数据集** | ImageNet ILSVRC-2012（约 128 万张图） |
| **编码器架构** | ResNet-50（输出 $h \in \mathbb{R}^{2048}$） |
| **投影头** | 两层 MLP：2048 → 2048（ReLU）→ 128 |
| **Batch size** | 4096（默认），大模型训练用 4096-8192 |
| **训练 Epochs** | 100（消融实验），1000（最终报告结果） |
| **优化器** | LARS，LR = 0.3 x BatchSize/256 = 4.8，weight decay = $10^{-6}$ |
| **学习率调度** | 前 10 epoch 线性 warmup，之后 cosine decay 无重启 |
| **温度 $\tau$** | 0.1 |
| **硬件** | Cloud TPU v3，32-128 cores |
| **训练时间** | ResNet-50，batch size 4096，100 epochs：约 **1.5 小时**（128 TPU v3 cores） |
| **线性评估** | 冻结编码器，训练线性分类器，90 epochs，LR = 1.6 |

GPU 复现参考：使用 8x V100 (32GB)，batch size 4096，100 epochs 约需 2-3 天。更实际的单卡设置（如 batch size 256-512，100-200 epochs）可在 1-2 天内完成，但性能会有下降。

## 3. 重要idea

1. **对比学习可以极其简单**：SimCLR 不需要 memory bank（如 MoCo/InstDisc），不需要定制架构（如 AMDIM/CPCv2），不需要互信息估计的复杂理论框架。仅通过同 batch 内的负样本、标准 ResNet、简单的 NT-Xent 损失即可达到 SOTA。

2. **数据增强的组合是决定性因素**：单一增强不够，组合增强（尤其是裁剪+颜色扰动）才能定义出足够难且有意义的对比预测任务。增强定义了"要预测什么"，将预测任务与网络架构解耦——之前的 CPC 用 patch 分割+context 网络，AMDIM 用修改的 receptive field，SimCLR 仅用随机裁剪就隐含了 global-to-local 和相邻视图预测。

3. **非线性投影头是关键改进**：投影头之前 $h$ 的表征质量远优于投影头之后 $z$。投影头充当信息瓶颈，将变换不变性约束限制在 $z$ 空间，使 $h$ 可以保留颜色、方向等对下游有用的信息。这是 SimCLR 相比使用线性投影头的 MoCo/PIRL 的重要设计差异。

4. **大 batch + 长训练一致有效**：对比学习对负样本数量和多样性的需求远高于监督学习。更大的 batch 和更多的训练步数持续提升性能，这与监督学习的饱和趋势形成鲜明对比。

5. **自监督达到监督水平**：ResNet-50 (4x) 线性评估 76.5% top-1，匹配监督 ResNet-50；仅用 1% 标签微调达到 85.8% top-5，超越全标签 AlexNet。在 12 个迁移学习数据集上，微调后的 SimCLR 在 10 个数据集上与监督基线持平或更优。这证明了自监督表征的强大泛化能力，也暗示了"self-supervised learning remains undervalued"。
