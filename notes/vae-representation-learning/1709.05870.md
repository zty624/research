# ZhuSuan: A Library for Bayesian Deep Learning (Shi et al., 2017)

**arxiv**: 1709.05870 | **关键词**: 概率编程, 贝叶斯深度学习, 变分推断, MCMC, TensorFlow | Tsinghua University | arXiv 2017

---

## 一、核心问题

2017 年前后，深度学习和贝叶斯方法各有优势但缺乏统一工具：

1. **深度学习的局限**：确定性神经网络给出点估计，无法建模不确定性；需要大量标注数据；难以融入先验知识；容易受对抗攻击
2. **贝叶斯方法的优势**：可建模不确定性、融入先验、小样本学习；但传统贝叶斯模型的变换通常是简单的解析形式（如指数函数、内积）
3. **贝叶斯深度学习 (BDL) 的核心特征**：确定性变换可以是深度神经网络自动学到的表达力参数化形式，突破了传统贝叶斯模型的表达力瓶颈
4. **工具缺口**：现有深度学习框架（TensorFlow, PyTorch）面向确定性网络和监督任务设计；现有概率编程库（PyMC3, Edward）要么不支持灵活变分后验，要么建模与推理耦合过紧

ZhuSuan 的目标：基于 TensorFlow 构建一个专为贝叶斯深度学习设计的概率编程库，支持从传统层次贝叶斯模型到深度生成模型的广泛模型类型。

## 二、方法

### 2.1 贝叶斯深度学习背景

论文系统梳理了 BDL 需要的推断方法：

**变分推断 (VI)**：
- 传统 VI 使用解析更新（共轭模型），但 BDL 中非共轭依赖常见
- 随机近似 + 数据子采样使 VI 可扩展到大数据集
- 直接优化变分下界替代传统解析更新
- 关键挑战是梯度估计：SGVB/重参数化技巧（连续变量）、REINFORCE（离散变量）
- 更好的下界：IWAE 的 importance weighted bound、VIMCO（离散变量多样本估计）

**Monte Carlo 方法**：
- 重要性采样：用于模型学习和对数似然评估，neural adaptive proposals 提升效率
- HMC：利用目标分布的几何信息高效探索，仅需密度和梯度；leapfrog 积分器保持体积守恒和近似哈密顿守恒

### 2.2 设计原则

ZhuSuan 的两大设计原则：

1. **模块化 (Modularity)**：建模和推断完全分离，各部分可独立使用。不像 PyMC3 那样将采样算法封装在 `sample()` 调用中
2. **透明性 (Transparency)**：推理过程不隐藏在抽象背后，用户可直接操作 surrogate cost，支持重度定制

### 2.3 模型原语

#### 确定性节点

直接使用 TensorFlow 的 Tensor（任意 TF 操作），不做额外抽象：
- 算术操作（`tf.add`, `tf.tensordot`）
- 神经网络层（`tf.layers.fully_connected`, `tf.layers.conv2d`）
- 控制流（`tf.while_loop`, `tf.cond`）

#### 随机节点

`StochasticTensor` 抽象，继承 Tensor 行为：
- 分布类型：Normal, Bernoulli, Categorical, Concrete/Gumbel-Softmax 等
- 可直接传入 TF 操作，自动转换为 Tensor
- 值可设置为采样值或观测值，或由上下文自动决定

#### 图上下文

`BayesianNet` 通过 Python `with` 语句管理所有命名的 `StochasticTensor`：
- 查询当前状态输出
- 查询局部概率（CPD 的当前值）
- **模型复用**：通过 `observed` 参数传递观测值，同一模型定义即可用于先验采样和后验推断，无需复制计算图（Edward 需要复制）

### 2.4 推理算法

#### 变分推断

支持多种目标函数和梯度估计器：

| 目标函数 | 梯度估计器 | 支持的隐变量类型 |
|----------|-----------|----------------|
| ELBO | SGVB（重参数化） | 连续可重参数化 + Concrete 松弛 |
| ELBO | REINFORCE | 所有类型 |
| Importance Weighted | SGVB (IWAE) | 连续可重参数化 + Concrete 松弛 |
| Importance Weighted | VIMCO | 所有类型 |
| $KL(p \| q)$ | RWS (Reweighted Wake-Sleep) | 所有类型 |

变分后验可通过 `BayesianNet` 构建任意结构的分布，不限于简单的高斯——这是相对 PyMC3 的关键优势。

#### Monte Carlo 方法

- **重要性采样**：支持模型学习和对数似然评估
- **HMC**：并行链运行、自动调参（步长和质量矩阵）、API 类似 TensorFlow 优化器

### 2.5 运行示例

论文提供了四个完整的示例，代码与图模型定义一一对应：

1. **Bayesian Logistic Regression**：权重 $w \sim \mathcal{N}(0, \alpha^2 I)$，推理使用 SGVB
2. **VAE**：标准 VAE，编码器输出高斯参数，解码器为 Bernoulli
3. **Deep Sigmoid Belief Network (DSBN)**：多层离散隐变量，支持 Concrete 松弛或 VIMCO
4. **Bayesian RNN**：LSTM 权重为随机变量，使用 `tf.scan` 处理变长序列

### 2.6 与 PyMC3 和 Edward 的比较

| 特性 | PyMC3 | Edward | ZhuSuan |
|------|-------|--------|---------|
| 基于 | Theano | TensorFlow | TensorFlow |
| 可定制变分后验 | 仅重参数化 | 支持 | 支持 |
| 控制流支持 | 支持 | 不完善 | 支持 |
| 模块化 | 耦合（采样器封装在 sample()） | 耦合 | **独立** |
| 透明性 | 低（隐藏推断过程） | 低 | **高**（用户直接操作 surrogate cost） |
| 模型复用 | — | 需复制计算图 | **observed 参数实现多态** |

## 三、实验结果

本文主要是工具介绍论文，实验以示例形式展示：

- 四个模型的完整代码实现，代码与图模型定义一一对应
- HMC 实现支持 GPU 并行，API 简洁（类似 TF 优化器的 `minimize()` 接口）
- 变分推断支持 SGVB、REINFORCE、VIMCO、RWS 等多种算法，覆盖连续和离散隐变量

## 四、个人评价

### 核心贡献

1. **统一了变分推断和 MCMC 的接口**：在同一框架下支持 SGVB、REINFORCE、VIMCO、HMC 等多种算法，且模块化设计允许灵活组合
2. **模型复用设计**：通过 `observed` 参数实现多态，避免了 Edward 的图复制问题——同一模型定义既可先验采样又可后验推断
3. **透明性原则**：让用户直接操作推断目标函数（surrogate cost），而非将其隐藏在抽象背后。这对于 BDL 研究者特别重要——他们经常需要定制推断算法
4. **推动贝叶斯深度学习的易用性**：让研究者可以像写深度学习代码一样写概率模型，降低了 BDL 的入门门槛

### 局限

1. **基于 TensorFlow 静态图**：在 TF 2.x 动态图时代已不太适用，虽然理论上可以迁移到 eager mode，但从未实际完成
2. **社区活跃度不如后来的 Pyro**：Pyro（基于 PyTorch）受益于 PyTorch 在研究社区的统治地位，最终成为 BDL 工具的主流选择
3. **缺少自动推断**：不像 PyMC 的 NUTS 自动采样，ZhuSuan 需要用户手动选择推断算法和配置参数
4. **工具类论文，方法贡献有限**：主要价值在工程实现，对 BDL 理论没有新贡献
5. **Concrete/Gumbel-Softmax 的支持有限**：虽然列出了对离散变量的支持，但实际使用中 REINFORCE 的高方差问题仍然存在

### 影响

ZhuSuan 是早期将贝叶斯推断与深度学习范式统一的重要尝试，其设计哲学（模块化、透明性）对后续工具设计有参考价值。虽然最终被 Pyro/NumPyro 等工具取代，但它代表了 2017 年前后 BDL 社区对"正确抽象"的探索。论文中系统梳理的 BDL 推断方法（从 SGVB 到 VIMCO 到 HMC）本身也是一份有价值的技术参考。
