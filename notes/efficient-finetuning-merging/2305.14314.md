# QLoRA: Efficient Finetuning of Quantized LLMs (Dettmers et al., 2023)

**arxiv**: 2305.14314 | **QLoRA 开山之作** | NeurIPS 2023

---

## 一、核心动机

大语言模型的微调虽然有效，但**内存成本极高**：16-bit 微调 LLaMA 65B 需要 >780GB GPU 内存。现有的量化方法（如 LLM.int8()）只能用于推理，训练时会崩溃。LoRA 虽然减少了可训练参数，但基础模型仍以 16-bit 存储，整体内存消耗仍然巨大。

本文的核心问题：**能否在 4-bit 量化模型上进行微调，而不损失任何性能？**

答案是肯定的——QLoRA 将 65B 模型的微调内存从 >780GB 降至 <48GB（单张 A100-48G 即可），且**完全恢复 16-bit 微调的性能**。

## 二、方法

### QLoRA 整体思路

$$Y = XW + sXL_1L_2$$

- $W$：冻结的 4-bit 量化预训练权重，使用时反量化到 BFloat16 做矩阵乘法
- $L_1 \in \mathbb{R}^{h \times r}, L_2 \in \mathbb{R}^{r \times o}$：低秩适配器（LoRA），BFloat16 精度训练
- $s$：标量缩放因子
- 梯度通过反量化的权重回传到 LoRA 参数

### 三大创新

#### 1. 4-bit NormalFloat (NF4)

**核心思想**：对于正态分布的数据，分位数量化（quantile quantization）是信息论最优的——每个量化桶被等概率地占据。

**为什么 NF4 优于 Int4/FP4？**

- **Int4**：均匀量化，对正态分布的权重效果差（大权重附近桶太稀疏，零附近桶太密集）
- **FP4**：浮点格式（E2M1 或 E3M0），指数位分配不匹配权重分布
- **NF4**：直接按正态分布的分位数设置量化值，确保每个量化桶被等概率使用

**NF4 的构造**：
1. 预训练神经网络的权重近似服从零均值正态分布
2. 将权重缩放至 [-1, 1] 范围
3. 计算标准正态分布 $N(0,1)$ 在 [-1, 1] 上的分位数作为量化值
4. 这些分位数值就是 NF4 的 16 个量化水平

实际效果（Pile Common Crawl 平均困惑度）：

| 数据类型 | 平均 PPL |
|---------|---------|
| Int4 | 34.34 |
| Float4 (E2M1) | 31.07 |
| Float4 (E3M0) | 29.48 |
| **NF4 + DQ** | **27.41** |

NF4 + Double Quantization 的困惑度比 Int4 低 20%，比 FP4 低 12%。

#### 2. Double Quantization（双重量化）

**问题**：分块量化中，每个大小为 B 的块需要一个量化常数 $c$（FP32）。对于 65B 模型，这些常数本身占 2 GB 内存。

**方法**：对量化常数再做一次量化——把 FP32 的 $c$ 量化为 Int8，并存储第二级量化常数。

**节省**：每个参数平均节省 0.37 bits。对 65B 模型约节省 3 GB。

具体计算：
- 原始：每个块大小 64，每个参数的常数开销 = 32/64 = 0.5 bits
- Double Quant：第一级常数量化为 Int8 (8/64)，第二级常数为 FP32 (32/(64*256)) = 0.127 bits/param
- 净节省：0.5 - 0.127 = 0.373 bits/param

#### 3. Paged Optimizers

**问题**：梯度检查点（gradient checkpointing）在处理长序列时会产生内存峰值（spike），导致 OOM。

**方法**：使用 NVIDIA 统一内存（Unified Memory），当 GPU 内存不足时自动将优化器状态分页到 CPU 内存，GPU 内存充足时再分页回来。避免了内存峰值导致的 OOM 错误。

### LoRA 配置的关键改进

传统 LoRA 只在注意力层添加适配器，QLoRA 的关键发现：**在所有线性层（包括 FFN）添加 LoRA 适配器**对恢复 16-bit 性能至关重要。

理由：虽然 LoRA 参数本身只占很小内存（0.2% 原始权重 = 26MB for 7B），但 LoRA 的**输入梯度**才是内存大头（567MB without checkpointing, 18MB with checkpointing）。既然梯度是大头，多加一些 LoRA 参数对总内存影响很小，但对性能提升明显。

## 三、实验结果

### QLoRA 是否恢复 16-bit 性能？

**GLUE + Super-NaturalInstructions（RoBERTa-large, T5 系列）**：

| 方法 | GLUE Acc. | T5-780M SI | T5-3B SI |
|------|-----------|------------|----------|
| BF16 full finetune | 88.6 | 48.0 | 54.3 |
| LoRA BF16 | 88.8 | 47.1 | 55.4 |
| QLoRA NF4+DQ | - | 42.7 | 55.3 |
| QLoRA FP4 | 88.6 | 42.4 | 55.6 |

**MMLU 5-shot（LLaMA 7B-65B）**：

| 方法 | 7B | 13B | 33B | 65B |
|------|-----|------|------|------|
| BF16 LoRA | 38.4/47.2 | 50.6/57.7 | 60.5/61.8 | 62.5/63.9 |
| FP4 | 37.2/47.3 | 50.0/55.9 | 58.5/61.3 | 63.3 |
| **NF4+DQ** | **39.0/47.5** | **50.7/57.3** | **59.2/61.8** | **63.9** |

- **NF4+DQ 在所有规模上完全恢复 BF16 性能**
- FP4 始终比 BF16 差约 1 个百分点
- 有趣：7B + Alpaca 上 NF4 甚至超过了 BF16（39.0 vs 38.4）

### Guanaco：SOTA 开源 Chatbot

**Vicuna Benchmark（相对 ChatGPT 的百分比）**：

| 模型 | 参数 | 精度 | 内存 | ChatGPT% |
|------|------|------|------|----------|
| GPT-4 | - | - | - | 114.5% |
| **Guanaco 65B** | 65B | 4-bit | 41GB | **99.3%** |
| **Guanaco 33B** | 33B | 4-bit | 21GB | **97.8%** |
| Open Assistant 33B | 33B | 16-bit | 66GB | 94.9% |
| Vicuna 13B | 13B | 16-bit | 26GB | - |
| **Guanaco 7B** | 7B | 4-bit | 6GB | - |
| Alpaca 65B | 65B | 4-bit | 41GB | 70.7% |

- Guanaco 65B 达到 ChatGPT 的 99.3%，**几乎追平**
- Guanaco 33B 仅用 21GB 内存就达到 97.8%
- Guanaco 7B 仅 5-6GB 内存就超过 Alpaca 13B（26GB）20+ 个百分点

**Elo 排名（GPT-4 评判）**：

| 模型 | Elo |
|------|-----|
| GPT-4 | 1348 ± 1 |
| **Guanaco 65B** | **1022 ± 1** |
| **Guanaco 33B** | **992 ± 1** |
| Vicuna 13B | 974 ± 1 |
| ChatGPT | 966 ± 1 |
| Guanaco 13B | 916 ± 1 |
| Bard | 902 ± 1 |

- Guanaco 33B/65B 的 Elo 超过 ChatGPT（注意这是 GPT-4 评判的 Elo，有偏差）

### 关键发现：数据质量 > 数据量

- OASST1（约 9k 样本）在 chatbot 性能上**全面超越** FLAN v2（450k 样本）
- FLAN v2 在 MMLU 上强但 chatbot 能力差，OASST1 反之
- **MMLU 高不代表 chatbot 好，反之亦然**——任务适配性比数据量更重要

### GPT-4 评估 vs 人类评估

- 两者在整体排名上**大致一致**
- 但存在个别情况严重不一致——GPT-4 评估并非完全可靠
- 论文推荐使用 Elo 排名法而非绝对评分法，避免"8/10 分在不同场景含义不同"的问题

## 四、个人评价

QLoRA 是 2023 年大模型民主化最关键的技术之一，其影响可以从多个维度理解：

1. **民主化微调**：将 65B 模型的微调门槛从"多卡服务器"降到"单张消费级 GPU"，使得学术实验室和个人研究者都能参与大模型研究
2. **NF4 数据类型的理论优雅性**：基于分数量化的信息论最优性，针对正态分布权重量身设计，比直觉式的 Int4/FP4 更有理论支撑
3. **"所有层加 LoRA"的实用发现**：打破了"LoRA 只加注意力层"的惯例，证明在计算-内存权衡中，多加适配器几乎免费但性能提升显著
4. **Guanaco 的训练效率**：24 小时单卡训练出接近 ChatGPT 的模型，这在当时是震撼性的结果

局限性：
1. **推理速度**：4-bit 存储需要反量化到 16-bit 才能计算，矩阵乘法本身仍以 16-bit 进行——QLoRA 节省的是内存而非计算
2. **量化精度的天花板**：NF4 虽然信息论最优，但 4-bit 终究是 4-bit，对于分布偏离正态的权重（如 embedding 层）效果可能下降
3. **Chatbot benchmark 的可信度**：论文自己也指出 Vicuna benchmark 的置信区间很宽，GPT-4 评估存在位置偏差（先出现的回答得分更高）
4. **Elo 排名的解释**：Guanaco 超过 ChatGPT 的 Elo 更多反映了 benchmark 的局限而非模型的真实能力——lemon-pick 分析显示 Guanaco 在推理和复杂任务上仍明显弱于 ChatGPT
5. **后续发展的冲击**：QLoRA 之后，GPTQ、AWQ 等方法在推理端量化上更进一步，LoRA 本身也被 DoRA、PiSSA 等变体挑战

总体而言，QLoRA 的核心贡献是**证明了"量化+适配器"可以完全恢复全精度微调性能**，消除了低精度训练的理论顾虑。这个结论为后续所有量化感知训练工作奠定了信心基础。同时，"数据质量远比数据量重要"的发现对整个 instruction tuning 社区产生了深远影响。
