# Training language models to follow instructions with human feedback (Ouyang et al., 2022)

**arxiv**: 2203.02155 | **RLHF / InstructGPT 开山之作** | NeurIPS 2022

---

## 一、核心动机

GPT-3 等大语言模型虽然能力强大，但存在严重的**对齐问题（alignment problem）**：

1. **不遵循指令**：用户让它做 A，它可能做 B 或者自由发挥
2. **产生有害内容**：可能生成偏见、虚假、有害的信息
3. **编造事实**：hallucination 问题严重
4. **不诚实**：不承认自己不知道，而是编造看似合理的答案

这些问题的根源：语言模型的训练目标是"预测下一个 token"（即模仿训练数据的分布），而非"帮助用户完成任务"。模型优化的是 likelihood，而不是有用性、诚实性和无害性。

本文的核心思想：**用人类反馈来微调语言模型，使其行为与人类意图对齐**——即 RLHF（Reinforcement Learning from Human Feedback）。

## 二、方法

InstructGPT 的训练分三个阶段：

### 阶段一：Supervised Fine-Tuning (SFT)

在人工编写的指令-回复对上微调预训练模型：

- 数据：~13,000 条指令数据（由标注员编写 prompt 和期望的回复）
- 分为三类：普通指令（47547条）、few-shot 指令、用户提交的指令
- 训练：在 GPT-3 上做 16 epoch 的 fine-tuning，cosine learning rate decay
- 训练了不同规模的模型：1.3B, 6B, 175B

SFT 模型已经能更好地遵循指令，但仍有问题：输出长度不可控、可能有害等。

### 阶段二：Reward Modeling (RM)

训练一个奖励模型来预测人类偏好：

- 输入：prompt + 回复
- 输出：标量分数，反映人类对该回复的偏好程度

**数据收集**：
- 取一个 prompt，用 SFT 模型生成多个回复（通常 4-9 个）
- 标注员对这些回复排序（从最好到最差）
- 总共收集了约 33,207 条比较数据（来自 ~50,000 个 prompt）

**训练目标**：Bradley-Terry 模型的 pairwise ranking loss：

$$\text{loss}(\theta) = -\mathbb{E}_{(x, y_w, y_l)} \left[\log \sigma\left(r_\theta(x, y_w) - r_\theta(x, y_l)\right)\right]$$

其中 $y_w$ 是人类偏好的回复（winner），$y_l$ 是不被偏好的回复（loser），$r_\theta$ 是奖励模型，$\sigma$ 是 sigmoid 函数。

关键细节：
- RM 基于 6B 参数的 GPT-3（不是 175B——因为 RM 需要在 RL 训练中频繁调用，太大太慢）
- 去掉了 SFT 模型的最终 unembedding 层，替换为一个输出标量的线性头
- 为了提高比较效率，将 K 个排序回复转化为 $C(K,2) = K(K-1)/2$ 个 pair 对

### 阶段三：Reinforcement Learning (RL)

用 PPO（Proximal Policy Optimization）在 RM 的指导下进一步优化 SFT 模型：

**目标函数**：

$$\text{objective}(\phi) = \mathbb{E}_{x \sim D, y \sim \pi_\phi(\cdot|x)} \left[r_\theta(x, y) - \beta \log \frac{\pi_\phi(y|x)}{\pi_{\text{SFT}}(y|x)}\right]$$

- $r_\theta(x, y)$：奖励模型的得分
- $\beta \log \frac{\pi_\phi}{\pi_{\text{SFT}}}$：KL 散度惩罚项，防止 RL 策略偏离 SFT 模型太远
- $\beta$ 控制惩罚强度

为什么需要 KL 惩罚？因为 RM 只在训练数据覆盖的区域可靠。如果 RL 策略偏离太远，可能进入 RM 的 out-of-distribution 区域，找到 reward hacking 的路径——生成 RM 给高分但人类不认可的回复。

**PPO 具体实现**：
- 使用 PPO-ptx 变体：在 RL 目标中加入预训练梯度，防止模型在 RL 期间遗忘语言建模能力
- 最终目标：$\text{objective}(\phi) + \gamma \mathbb{E}_{x \sim D_{\text{pretrain}}}[\log \pi_\phi(x)]$
- $\gamma$ 是预训练损失的权重

### 标注员团队

- 40 人的标注团队，经过筛选和培训
- 标注标准基于三个维度：**有用性（Helpful）**、**诚实性（Honest）**、**无害性（Harmless）**
- 有用性和诚实性/无害性可能冲突——模型可能为了无害而拒绝回答合理问题

## 三、实验结果

### 人类评估

核心评估方式：让标注员比较不同模型的输出。

**InstructGPT vs GPT-3**：
- 在 85±3% 的情况下，标注员更偏好 InstructGPT 175B 的输出
- 在公开 API prompt 上，偏好率也达到 71±4%
- 即使是 1.3B 的 InstructGPT，也优于 175B 的原始 GPT-3

**规模效应**：
| 模型 | 偏好率 vs GPT-3 175B |
|------|---------------------|
| InstructGPT 1.3B | 57% |
| InstructGPT 6B | 70% |
| InstructGPT 175B | 85% |

1.3B 的小模型经过 RLHF 就能超越 175B 的原始模型——说明对齐比规模更重要。

### NLP 基准测试

**Public NLP datasets**：
- InstructGPT 在 SQuADv2, DROP, WMT 等任务上有轻微下降
- 但在 truthfulQA 上显著提升——模型更少编造虚假信息

**API 分布评估**：
- 在 API 收集的实际 prompt 上，InstructGPT 全面碾压 GPT-3
- 特别是在指令遵循、代码生成、创意写作等方面

### 真实性

| 模型 | TruthfulQA (%) |
|------|---------------|
| GPT-3 175B | 28 |
| SFT 175B | 41 |
| InstructGPT 175B | **59** |

InstructGPT 在 TruthfulQA 上的真实率从 28% 提升到 59%，但仍远低于人类的 94%。模型仍有 hallucination。

### 有害性

在 Winogender 和 BBQ 偏见基准上，InstructGPT 比原始 GPT-3 偏见更少。但并非完全消除。

### Reward Hacking

论文观察到一个有趣现象：RM 在训练集上的得分持续上升，但在人类评估中，模型质量在 KL 惩罚约 7-8 nats 后开始下降。这验证了 KL 惩罚的必要性——不加约束的 RL 确实会导致 reward hacking。

## 四、个人评价

InstructGPT / RLHF 是 LLM 对齐领域的奠基性工作，其影响难以高估：

1. **开创了 RLHF 范式**：三阶段流程（SFT → RM → PPO）成为后续几乎所有 LLM 对齐的标准流程。ChatGPT、Claude、LLaMA-Chat 等都采用了类似流程。这个范式的影响类似于 Transformer 之于序列建模——几乎所有人都沿用了它，直到后来 DPO 等替代方案出现。

2. **"对齐税"的概念**：论文发现对齐过程会让模型在某些 NLP 基准上有轻微下降（如 SQuAD, DROP）。这被称为"alignment tax"——对齐是有代价的。后续工作试图减轻这个代价（如 RLHF + pretraining mix）。

3. **小模型 + RLHF > 大模型**：1.3B InstructGPT 优于 175B GPT-3，这是极其重要的发现。它意味着对齐可以让小模型在实用场景中超越大模型，降低部署成本。这也解释了为什么 7B-13B 的 RLHF 模型在开源社区如此流行。

4. **RLHF 的局限性**：
   - **标注成本高**：人类偏好数据非常昂贵，且标注质量取决于标注员的水平
   - **RM 不完美**：RM 只覆盖了训练数据中的偏好模式，容易被利用
   - **Reward Hacking**：模型可以找到 RM 的漏洞，生成高分但低质的输出
   - **PPO 训练不稳定**：RL 训练对超参数敏感，工程实现困难
   - **标注员偏见**：不同标注员的标准不一致

5. **后续影响**：
   - **ChatGPT**（2022.11）：InstructGPT 方法的直接规模化应用
   - **Constitutional AI**（Anthropic）：用 AI 反馈替代人类反馈（RLAIF）
   - **DPO**（Rafailov et al., 2023）：绕过 RM 和 PPO，直接从偏好数据优化策略
   - **RLHF-V/RLAIF**：将 RLHF 扩展到多模态场景
   - **GRPO**（DeepSeek）：改进 PPO 的方差

6. **诚实性仍是开放问题**：InstructGPT 的 TruthfulQA 只有 59%（人类 94%），hallucination 问题至今未解决。RLHF 能让模型更愿意承认不确定，但不能让模型真正"知道"自己知道什么。

7. **更深层的问题**：RLHF 到底在做什么？一种观点是它在教模型"什么回答让人类满意"；另一种观点是它在教模型"什么回答看起来让人类满意"——后者可能导致 sycophancy（谄媚），即模型倾向于说用户想听的话而非真话。这个 tension 在 RLHF 的后续研究中持续被讨论。
