# YaRN: Efficient Context Window Extension of Large Language Models (Peng et al., 2023)

**arxiv**: 2309.00071

---

## 一、核心动机

使用 Rotary Position Embeddings (RoPE) 的 Transformer 模型无法泛化到训练时见过的序列长度之外。Position Interpolation (PI) 虽然可以通过微调扩展上下文窗口，但存在两个问题：(1) 均匀缩放所有 RoPE 维度会丢失高频信息；(2) 扩展比超过 8x 后模型质量明显退化。本文系统性地分析了 PI 的问题，并提出了 YaRN 方法。

## 二、关键技术点

### RoPE 回顾

RoPE 将位置 $m$ 编码为复数旋转 $e^{im\theta_d}$，每个隐藏维度 $d$ 对应一个频率 $\theta_d = 10000^{-2d/|D|}$。RoPE 的优势是注意力权重只依赖相对位置 $m-n$，但在训练上下文 $L$ 之外泛化困难。

### 问题 1：高频信息丢失 → "NTK-aware" 插值

PI 对所有维度均匀缩放 $s = L'/L$，导致高频分量被过度压缩。NTK 理论表明，深度神经网络在缺少高频分量的嵌入时难以学习高频信息。

解决方案（"NTK-aware"）：不均匀缩放——高频少缩放、低频多缩放。最简单的实现是修改 RoPE 的 base $b$。Code Llama 就采用了此方法（base 调至 1M）。

**局限**：最优 base 需要经验搜索，难以确定。

### 问题 2：局部相对距离丢失 → "NTK-by-parts" 插值

关键观察：RoPE 的不同维度编码了不同类型的位置信息：
- **短波长**（$\lambda \ll L$）：一个上下文窗口内旋转多圈，只编码**相对位置**信息——不应插值！
- **长波长**（$\lambda \geq L$）：整个上下文窗口内不足一圈，编码了**绝对位置**信息——必须插值

定义旋转比 $r(d) = L / \lambda_d$（维度 $d$ 在训练上下文中旋转的圈数），引入参数 $\alpha, \beta$：
- $r(d) < \alpha$：纯插值（像 PI 一样缩放）
- $r(d) > \beta$：不插值（保持原始频率）
- $\alpha \leq r(d) \leq \beta$：线性混合

对 LLaMA 系列，$\alpha=1, \beta=32$ 效果好。

### 问题 3：注意力熵增加 → YaRN 的温度缩放

扩展上下文后，注意力分布变得更平坦（熵增加），模型不确定该关注哪个位置。YaRN 引入温度 $t$：

$$\text{softmax}\left(\frac{q_m^T k_n}{t \sqrt{|D|}}\right)$$

推荐值：$1/\sqrt{t} = 0.1 \ln(s) + 1$，由 LLaMA 系列模型上的实验拟合得到。

**实现技巧**：通过缩放复数旋转位置嵌入本身来实现，无需修改注意力代码，零额外开销，且与 Flash Attention 2 兼容。

### YaRN = NTK-by-parts + 温度缩放

### Dynamic Scaling（推理时技巧）

在自回归生成中，每步的序列长度递增。动态缩放将 $s$ 设为 $\max(1, l'/L)$ 而非固定的 $L'/L$，使模型在未超出训练长度时保持原始行为，超出后优雅退化而非突然崩溃。Dynamic-YaRN 可在不微调的情况下实现 2x 以上的上下文扩展。

## 三、实验结果

- YaRN 在 LLaMA 7B/13B/33B/65B 和 Llama 2 7B/13B/70B 上测试
- 微调数据量仅需原始预训练数据的 ~0.1%（比 PI 少 10x token、2.5x 训练步数）
- 在 perplexity 评估上，YaRN 在所有测试长度上均优于 PI 和 NTK-aware
- 64x 扩展（从 4K 到 256K）仍能保持合理性能
- Dynamic-YaRN 可在不微调的情况下实现 2x 以上扩展
- 温度参数 $t$ 在不同模型大小间具有"普适性"，暗示注意力熵增加是 RoPE 扩展的内在问题

## 四、个人思考/影响

1. **对 RoPE 的深刻理解**：本文最重要的贡献是系统性地分析了 RoPE 各维度的行为——高频编码相对位置、低频编码绝对位置。这种分析框架不仅指导了 YaRN 的设计，也帮助理解了为什么简单的 PI 会失败
2. **温度缩放是点睛之笔**：单独的 NTK-by-parts 解决了插值问题，但注意力熵增加是一个独立的问题。温度缩放优雅地解决了这个问题，且通过"长度缩放"技巧实现了零开销
3. **实用价值极高**：YaRN 已被多种开源模型采用（Code Llama 用了 NTK-aware 的前身），是当时扩展上下文窗口的最佳实践
4. **与后续工作的关系**：YaRN 的思想影响了后来的上下文扩展方法。不过，随着模型在训练时就使用更长上下文（如 Llama 3 的 128K），对推理时扩展的需求有所降低
5. **启发**：RoPE 频率维度的"分层处理"思想（不同维度不同策略）可以推广到其他位置编码的设计和分析中
