## 1. 理论推导

### 核心问题

RAE (Zheng et al., 2025) 证明了可以直接在 DINOv2 patch 特征上做扩散，绕过像素空间 VAE，获得更快的收敛和有竞争力的生成质量。但 DINOv2 ViT-B/14 产生 256 个 768 维的 patch embedding，序列长度与像素空间潜扩散相同，计算上没有优势。关键观察：相邻 patch 共享大量语义信息，存在显著空间冗余。

### FlatDINO 架构设计 ("Lamination")

"Lamination"（层压/压扁）的核心思想：将 DINOv2 的二维 patch 网格压缩为一维连续 token 序列，丢弃原始二维空间结构。

**编码器**：ViT-B（768维，12层，12头），输入为 256 个 DINOv2 patch embedding + T 个可学习 register token。处理后只保留 register token 作为潜表示（latent）。CLS token 和 DINOv2 的 register token 被丢弃。

**解码器**：ViT-L（1024维，24层，16头），使用可学习 register 作为 query，对潜 token 做 cross-attention，重建 256 个 patch embedding。

**默认配置**：T=32 个 token，d=128 维，即将 256×768 的表示压缩为 32×128——序列长度减少 8×，总维度压缩 48×。

### 数学公式

**β-VAE 目标函数**：

$$\mathcal{L} = \mathbb{E}_{q_\phi(z|x)}[-\log p_\theta(x|z)] + \beta \, D_{KL}(q_\phi(z|x) \| p(z))$$

其中 $x$ 为 DINOv2 patch embedding，$z \in \mathbb{R}^{T \times d}$ 为潜 token，$q_\phi$ 和 $p_\theta$ 分别为编码器和解码器分布。KL 项对潜空间施加正则化，使潜分布与标准高斯对齐，保证与扩散过程的噪声调度兼容。

**KL 权重归一化**：由于 KL 散度随潜维度线性增长，直接用相同 β 会导致大潜空间被过度正则化。因此 β 按潜维度归一化：

$$\beta \propto \frac{1}{T \cdot d}$$

参考值：在 $T \cdot d = 512$ 时 $\beta = 10^{-6}$（遵循 Kim et al., 2025 / TA-TiTok 的设定）。β 过大会导致自编码器坍缩。

**Flow Matching 目标**：在 FlatDINO 潜空间上训练流匹配模型：

$$z_t = (1-t)z_0 + t z_1$$

$$\mathcal{L}_{FM} = \mathbb{E}_{t, z_0, z_1} \| v_\theta(z_t, t) - (z_1 - z_0) \|^2$$

从标准高斯 $z_0 \sim \mathcal{N}(0, I)$ 到数据分布 $z_1$ 沿直线路径插值，训练速度场 $v_\theta$。

### Lamination 相对标准 Representation Autoencoder 的改进

1. **计算效率**：序列长度从 256 降至 32，DiT-XL 前向 FLOPs 从 118.4 GFLOPs 降至 14.3 GFLOPs（约 8× 减少）。含编码开销的完整训练步 FLOPs 也减少 4.1×（DiT-XL）。
2. **保持语义结构**：压缩 patch embedding 而非像素，避免了像素重建 VAE 中重建-生成的优化困境。
3. **保留可扩散性**：KL 正则化保证潜空间与扩散噪声调度兼容。

## 3. 重要idea

### 3.1 Lamination：从二维 patch 网格到一维 token 序列

核心洞察是 DINOv2 patch 网格中存在大量空间冗余——相邻 patch 的余弦相似度随空间距离递减（Figure 5）。TiTok 证明图像可以编码为 32 个一维 token 而不失质量，FlatDINO 将此原则从像素空间迁移到 SSL 特征空间。这使扩散模型可以在语义组织良好的紧凑潜空间上运行，同时大幅降低计算开销。

### 3.2 Token 的自组织空间分区行为

Token 消融实验揭示了一个有趣现象：32 token 模型中，大多数 token 自动学习到局部化的 blob 形感受野，形成对图像的空间分区覆盖。一个特殊 token（token 26）编码全局背景信息。而 16 token 模型出现相变：token 从局部 blob 退化为编码整个水平条带，导致重建质量骤降。这表明存在一个 token 数量的临界阈值，低于此阈值则空间分区策略失效。

### 3.3 特征维度 vs token 数量的不对称性

PCA 分析表明 DINOv2 特征在特征维度上几乎没有线性冗余（768 维中需 594 维才能解释 95% 方差，仅 1.3× 压缩），而像素 patch 可达 28× 压缩。因此 DINO 特征的冗余主要集中在空间维度而非特征维度，压缩应优先减少 token 数而非特征维度。实验证实：同等总维度下，更多 token + 更小特征维度的配置优于更少 token + 更大特征维度。

### 3.4 压缩-扩散的权衡

FlatDINO 32×128 在 gFID 1.85 (w/ CFG) 的同时实现 8× 前向 FLOPs 减少和 4.5× 训练 FLOPs 减少。但生成质量仍不及 RAE 直接在未压缩 DINOv2 特征上的结果（gFID 1.51），部分原因是训练不充分（600 vs 800 epochs），部分是压缩语义潜空间的扩散配方仍需优化。32×768 的潜空间虽然保留了判别性（k-NN ImageNet 77.2% vs 原始 74.3%），但扩散模型无法在此高维潜空间上收敛，说明压缩特征维度对扩散可行性至关重要。

### 3.5 与像素空间 VAE 的根本区别

像素重建 VAE 会将潜容量分配给纹理和高频细节，这可能不利于生成。而 FlatDINO 在 DINOv2 特征空间训练自编码器，潜空间自然编码高层语义内容，绕过了重建-生成的优化困境。
