# An Image is Worth 16x16 Words: Transformers for Image Recognition at Scale (Dosovitskiy et al., 2021)

**arxiv**: 2103.14030 | **ViT / Vision Transformer 开山之作** | ICLR 2021

---

## 一、核心问题

Transformer 在 NLP 领域取得了统治性地位（BERT、GPT-3），但在计算机视觉领域，CNN 仍然是绝对主流。之前将 Transformer 应用于视觉的尝试（如 ViT 之前的 iGPT、DETR）要么效率低下，要么仍严重依赖 CNN 组件。

核心问题：**纯 Transformer 架构能否直接处理图像，并在大规模数据上超越 CNN？**

CNN 的归纳偏置（inductive bias）——局部性（locality）和平移等变性（translation equivariance）——一直被认为是视觉模型的必要条件。但 NLP 的经验表明，大规模预训练可以克服归纳偏置的不足。ViT 的核心假设：**在大规模数据上预训练的纯 Transformer 可以学到与 CNN 等价甚至更好的视觉表示**。

## 二、关键创新

### 2.1 Patch Embedding：图像 → 序列

ViT 最核心的设计决策：将图像分割为固定大小的 patch，每个 patch 线性投影为一个向量，作为 Transformer 的输入 token。

$$x \in \mathbb{R}^{H \times W \times C} \to x_p \in \mathbb{R}^{N \times (P^2 \cdot C)}$$

- $H, W, C$：图像高度、宽度、通道数
- $P$：patch 大小（默认 16×16）
- $N = HW / P^2$：patch 数量（即序列长度）

然后通过可训练的线性投影 $E \in \mathbb{R}^{(P^2 \cdot C) \times D}$ 将每个 patch 映射到 $D$ 维空间。

这个设计的直觉："An Image is Worth 16×16 Words"——每个 16×16 的 patch 类比 NLP 中的一个 word，整个图像就是一句话。

### 2.2 [CLS] Token

与 BERT 类似，ViT 在 patch embedding 序列前加一个可学习的 [class] token ($z_0^0 = x_{\text{class}}$)。该 token 的最终表示 $z_L^0$ 用于分类（通过一个 MLP head）。

### 2.3 位置编码

使用标准的可学习 1D 位置编码 $E_{pos} \in \mathbb{R}^{(N+1) \times D}$，加到 patch embedding 上：

$$z_0 = [x_{\text{class}}; x_p^1 E; x_p^2 E; \ldots; x_p^N E] + E_{pos}$$

论文实验表明 1D 位置编码效果与 2D 位置编码相当——模型可以隐式地从 1D 位置编码中学到 2D 位置关系。

### 2.4 最小化归纳偏置

ViT 相比 CNN 的关键区别：**几乎不引入视觉特定的归纳偏置**。

- CNN 天生有局部性（卷积核只看局部）和平移等变性
- ViT 的 self-attention 是全局的，从头开始训练时缺乏局部先验
- 初始层 self-attention 的注意力模式几乎不关注空间邻域——位置关系需要从头学习

这解释了为什么 ViT 在小数据集上不如 CNN，但在大规模数据上可以超越。

## 三、方法细节

### 3.1 模型变体

| 模型 | 层数 | 隐藏维度 D | MLP 维度 | 注意力头数 | 参数量 |
|------|------|-----------|---------|-----------|--------|
| ViT-B/16 | 12 | 768 | 3072 | 12 | 86M |
| ViT-L/16 | 24 | 1024 | 4096 | 16 | 307M |
| ViT-H/14 | 32 | 1280 | 5120 | 16 | 632M |

命名规则：ViT-{Base/Large/Huge}/{patch 大小}

### 3.2 预训练

**大规模预训练是 ViT 成功的关键**：

- **JFT-300M**：Google 内部数据集，3 亿张标注图像——这是 ViT 发挥优势的前提
- **ImageNet-21k**：1400 万张图像，2 万 1 千类
- **ImageNet-1k**：130 万张图像，1 千类

预训练方式：在大型数据集上做分类预训练（移除 MLP head，换为对应类别数的线性层），然后在目标数据集上微调。

### 3.3 微调

微调时通常使用更高分辨率的图像：
- 预训练：224×224
- 微调：384×384 或 512×512

提高分辨率后，patch 大小不变但 patch 数量增加（序列变长）。位置编码需要插值——使用 2D 双三次插值将预训练的位置编码扩展到新的序列长度。

### 3.4 训练细节

- 优化器：Adam（β₁=0.9, β₂=0.999），weight decay 0.1
- 学习率：base 0.001，cosine 衰减，warmup 10k steps
- 批大小：4096
- 数据增强：RandAugment、Mixup、CutMix、Erasing
- 正则化：Dropout（微调时在 attention 和 MLP 后加）、Stochastic Depth

## 四、实验结果

### 4.1 与 SOTA 比较

**ImageNet 1K 微调结果**：

| 模型 | 预训练数据 | ImageNet Top-1 |
|------|-----------|---------------|
| BiT-ResNet-152 (22B) | JFT-300M | 87.54% |
| ViT-H/14 | **JFT-300M** | **88.55%** |
| ViT-L/16 | ImageNet-21k | 85.30% |
| ViT-L/16 | JFT-300M | 87.76% |

ViT-H/14 在 ImageNet 上达到 88.55%，超越所有 CNN 模型，且预训练计算量更少。

### 4.2 预训练数据规模的影响

这是论文最重要的实验：ViT 与 CNN (BiT) 在不同预训练数据规模下的比较：

- **小数据（ImageNet-1k）**：ViT 表现不如 BiT（ResNet）
- **中等数据（ImageNet-21k）**：两者相当
- **大数据（JFT-300M）**：ViT 显著优于 BiT

结论：**ViT 需要大规模预训练数据才能发挥优势**。在小数据上，CNN 的归纳偏置更有效；在大数据上，ViT 可以从数据中学到更优的表示，且计算效率更高。

### 4.3 计算效率

在相同的预训练 FLOPs 下，ViT 比 BiT (ResNet) 达到更高的精度。ViT-H/14 的训练成本仅为 BiT-XL 的 0.37 倍，但精度更高。

这是因为 ViT 的计算效率几乎不受 patch 大小影响（序列长度短），而 CNN 的计算量与分辨率成二次关系。

### 4.4 自注意力分析

对 ViT 学到的 self-attention 进行可视化：

- **浅层**：注意力模式接近 CNN——关注局部邻域，类似卷积核
- **深层**：注意力变得全局化——远距离的 patch 之间建立连接
- 整体上，**ViT 先学局部后学全局**，这暗示 CNN 的局部先验只是"浅层"的

### 4.5 位置编码分析

1D 位置编码与 2D 位置编码效果几乎相同。可视化学到的位置编码发现：相近位置的编码相似（余弦相似度高），形成了类似 2D 网格的结构——模型隐式地学到了 2D 空间结构。

## 五、影响

1. **Transformer 统一了 NLP 和 CV 的架构**：ViT 之后，视觉和语言的模型架构趋于统一，为多模态模型（CLIP、Flamingo、LLaVA）奠定了基础。

2. **ViT 成为标准视觉骨干网络**：后续几乎所有的视觉 Transformer 工作（Swin Transformer、DeiT、DINO、MAE）都以 ViT 为起点。ViT + MAE (He et al., 2022) 的组合成为自监督视觉预训练的标准方案。

3. **证明了规模的力量**：ViT 验证了 NLP 中的缩放定律在 CV 中同样适用——足够大的数据和模型可以让纯 Transformer 超越带有强归纳偏置的 CNN。这与 Chinchilla、GPT-3 的核心洞见一致。

4. **推动了自监督视觉预训练**：ViT 在小数据上不如 CNN 的问题，后来被 MAE（Masked Autoencoder）解决——MAE 在 ImageNet-1k 上自监督预训练 ViT，就能达到甚至超越有监督 CNN。

5. **CNN 与 Transformer 的融合**：ViT 的成功并不意味着 CNN 的终结。Swin Transformer 引入了层次化结构和窗口注意力，将 CNN 的局部性与 Transformer 的全局性结合。ConvNeXt (2022) 则表明纯 CNN 在借鉴 ViT 设计后也能达到相当的性能。

6. **开放问题**：
   - ViT 为什么在大数据上能超越 CNN？是因为 Transformer 更灵活（更少的归纳偏置 = 更大的假设空间），还是因为全局注意力提供了更好的信息传播路径？
   - Patch embedding 是不是最优的视觉 tokenization 方式？后续工作探索了不同粒度和方式的图像 tokenization
