# xLSTM: Extended Long Short-Term Memory (Beck et al., 2024)

**arxiv**: 2405.04517 | **关键词 序列模型 LSTM 扩展 开山** | JKU Linz / NXAI | 2024

---

## 一、核心问题

LSTM 曾是序列建模的主力（Hochreiter, 1991），在文本生成、机器翻译、强化学习等领域取得巨大成功。但 Transformer 的出现使 LSTM 逐渐被边缘化。LSTM 有三个根本局限：

1. **无法修正存储决策**：sigmoid 门控使得存储更新是渐进的——当需要替换已存储的值时，LSTM 难以完全覆盖旧值（最近邻搜索实验证实）
2. **存储容量有限**：信息必须压缩到标量单元状态 $c_t \in \mathbb{R}$，导致低频 token 预测困难（稀有 token 预测实验证实）
3. **无法并行化**：隐藏状态之间的递归连接（memory mixing）强制顺序处理，训练效率低

核心问题：**如果克服这些局限并将 LSTM 扩展到数十亿参数，在语言建模中能达到什么水平？**

## 二、方法

### 2.1 LSTM 回顾

原始 LSTM 的更新规则：

$$c_t = f_t c_{t-1} + i_t z_t, \quad h_t = o_t \tilde{h}_t, \quad \tilde{h}_t = \psi(c_t)$$

其中 $i_t, f_t, o_t$ 均为 sigmoid 门，$z_t$ 为单元输入。常数误差轮转（CEC）通过加性更新避免梯度消失。

### 2.2 sLSTM：指数门控 + 记忆混合

**指数门控**是 sLSTM 的核心创新。将输入门和遗忘门的激活从 sigmoid 改为指数函数：

$$i_t = \exp(\tilde{i}_t), \quad f_t = \sigma(\tilde{f}_t) \text{ 或 } \exp(\tilde{f}_t)$$

指数门控使模型可以**急剧增加**某个输入的存储权重，从而实现存储决策的完全修正。

**归一化状态**（normalizer state）追踪门控强度的累积：

$$n_t = f_t n_{t-1} + i_t$$

**稳定化状态**（stabilizer state）防止指数溢出：

$$m_t = \max(\log(f_t) + m_{t-1}, \log(i_t))$$

$$i'_t = \exp(\log(i_t) - m_t), \quad f'_t = \exp(\log(f_t) + m_{t-1} - m_t)$$

sLSTM 的完整前向传播：

$$c_t = f_t c_{t-1} + i_t z_t, \quad n_t = f_t n_{t-1} + i_t, \quad h_t = o_t \tilde{h}_t, \quad \tilde{h}_t = c_t / n_t$$

**新记忆混合**：sLSTM 可以有多个记忆单元和多个头。头内各单元通过递归权重矩阵 $R_z, R_i, R_f, R_o$ 进行记忆混合；头间不混合。指数门控使这种混合产生新的动态——一个头的指数输入门可以急剧改变整个头的信息流。

### 2.3 mLSTM：矩阵记忆 + 协方差更新 + 可并行化

为增强存储容量，mLSTM 将标量记忆 $c \in \mathbb{R}$ 扩展为矩阵记忆 $C \in \mathbb{R}^{d \times d}$。

**协方差更新规则**（Sejnowski, 1977; Dayan & Willshaw, 1991）：

$$C_t = f_t C_{t-1} + i_t v_t k_t^\top$$

其中 $k_t, v_t, q_t$ 分别为 key、value、query（借用 Transformer 术语）。在 key 和 value 投影前施加 LayerNorm 使其零均值，因此称为协方差更新。该规则在检索二值向量时具有最大信噪比。

**检索**：通过矩阵乘法查询：

$$\tilde{h}_t = \frac{C_t q_t}{\max(n_t^\top q_t, 1)}, \quad n_t = f_t n_{t-1} + i_t k_t$$

归一化状态 $n_t$ 追踪加权的 key 向量和，用点积归一化。下界为 1 防止除零。

mLSTM 前向传播（Eq. 19-27）：

$$q_t = W_q x_t + b_q, \quad k_t = \frac{1}{\sqrt{d}} W_k x_t + b_k, \quad v_t = W_v x_t + b_v$$

$$i_t = \exp(\tilde{i}_t), \quad f_t = \sigma(\tilde{f}_t) \text{ 或 } \exp(\tilde{f}_t), \quad o_t = \sigma(\tilde{o}_t)$$

注意：mLSTM **没有记忆混合**（无 $h_{t-1}$ 到门的递归连接），因此递推可重写为并行形式，类似 FlashAttention 的方式。

### 2.4 xLSTM 块与架构

两种残差块设计：

1. **sLSTM 块（post up-projection）**：类似 Transformer，先在原始空间用 sLSTM 总结过去，再通过 gated MLP 上投影 → 激活 → 下投影回原始空间
2. **mLSTM 块（pre up-projection）**：类似 SSM，先线性上投影到高维空间，在高维空间用 mLSTM 总结过去，再线性下投影

设计原理（Cover 定理）：非线性地嵌入到高维空间中更可能线性可分。mLSTM 的矩阵记忆容量在高维空间更大，因此用 pre up-projection。

xLSTM 架构：预 LayerNorm 残差骨干 + 交替堆叠 sLSTM 和 mLSTM 块。记号 xLSTM[a:b] 表示 a/b 的 mLSTM/sLSTM 块比例。

### 2.5 计算与内存

- 推理：线性计算复杂度 + 常数内存（相对序列长度），适合边缘部署
- mLSTM 的矩阵记忆 $d \times d$ 无需额外参数但计算昂贵，不过可在 GPU 上并行
- sLSTM 不可并行（因 memory mixing），但开发了快速 CUDA 实现，通常比 mLSTM 慢不到 2 倍

## 三、实验结果

### 3.1 合成任务

**形式语言任务（状态追踪）**：
- Transformer、Mamba、RWKV 等无记忆混合的模型无法解决奇偶性（parity）等正则语法任务
- 只有带 memory mixing 的 xLSTM[0:1]（纯 sLSTM）和 LSTM 能解决
- 证实了 RNN 的状态追踪能力 > SSM/Transformer（与 Merrill et al., 2024 一致）

**多查询关联回忆（MQAR）**：
- xLSTM[1:1] 在所有非 Transformer 模型中表现最佳
- sLSTM 块不削弱记忆容量，反而在 256 KV 对的困难设置中有所增益

### 3.2 15B Token 语言建模

在 SlimPajama 15B tokens 上的验证集困惑度：

| 模型 | 参数量(M) | 困惑度 |
|------|----------|--------|
| GPT-3 | 356 | 14.26 |
| Llama | 407 | 14.25 |
| Mamba | 423 | 13.70 |
| RWKV-5 | 456 | 14.25 |
| RWKV-6 | 442 | 15.03 |
| **xLSTM[1:0]** | **409** | **13.43** |
| **xLSTM[7:1]** | **408** | **13.48** |

xLSTM 超越所有方法。消融实验证实：指数门控和矩阵记忆**都**是必要贡献。

**消融**：从 vanilla LSTM（ppl 2417.86）到 xLSTM[7:1]（ppl 13.48）的提升来源：
- 加 ResNet 骨干：2417 → 35.46
- 加上投影块：35.46 → 26.01
- 加指数门控：26.01 → 17.70
- 加矩阵记忆：17.70 → 13.48

### 3.3 300B Token 大规模实验

**序列长度外推**：训练在 2048 上下文，测试到 16384：
- Llama 在 16k 时困惑度 337.83
- Mamba 在 16k 时 14.00
- **xLSTM[7:1] 在 16k 时 8.92**

xLSTM 在长上下文外推上远优于其他方法。

**下游任务**（1.3B 模型）：

| 模型 | SlimPajama ppl | LAMBADA acc | HellaSwag acc | 平均 acc |
|------|---------------|------------|---------------|---------|
| Llama | 9.44 | 57.44 | 57.81 | 56.99 |
| Mamba | 9.14 | 55.64 | 60.45 | 58.41 |
| **xLSTM[1:0]** | **8.89** | **57.83** | **60.91** | **58.48** |

**PALOMA 基准**：571 个文本领域的困惑度评估。xLSTM[1:0] 在 99.5% 的领域上优于 Mamba。

### 3.4 缩放律

在 125M 到 1.3B 范围内，xLSTM 在所有尺度上都保持优势。

## 四、个人评价

### 影响

1. **LSTM 复兴**：证明了经过关键改造的 LSTM 可以在现代语言建模中与 Transformer 和 SSM 竞争，挑战了"LSTM 已过时"的普遍认知
2. **指数门控**：指数激活函数使模型可以"硬"切换存储内容，解决了 LSTM 的存储修正问题。这一设计后来影响了其他 RNN 变体
3. **矩阵记忆**：将 LSTM 的标量记忆扩展为矩阵记忆，用协方差更新规则实现 key-value 存储，大幅提升记忆容量。与线性注意力的隐藏状态形式类似但通过门控增强了控制
4. **sLSTM + mLSTM 互补**：sLSTM 提供状态追踪能力（memory mixing），mLSTM 提供大规模存储和并行化。两者的组合在架构层面实现了 expressiveness 和 efficiency 的平衡
5. **长上下文外推**：xLSTM 在序列长度外推上的表现（16k 时困惑度仅 8.92）远超同期模型，这对长文档处理有重要意义

### 局限

1. **训练效率**：sLSTM 不可并行，训练速度受限。虽然 CUDA 优化后慢不到 2 倍，但在大规模训练中仍是瓶颈
2. **矩阵记忆的计算开销**：mLSTM 的 $d \times d$ 矩阵更新计算量 $O(d^2)$，虽然可并行但 FLOPs 高。文章未与 Mamba 2 等更高效的矩阵隐藏状态方法对比
3. **仅验证到 1.3B**：所有实验最大到 1.3B 参数，未在 7B+ 规模上验证。7B 以上 xLSTM 是否仍优于 Transformer 尚不清楚
4. **工程复杂度**：sLSTM 的稳定化、指数门控的 CUDA 实现复杂，比 Mamba 的简单选择性扫描难部署
5. **缺乏与最新基线的对比**：未与 Gated DeltaNet、Mamba 2 等同期工作对比
6. **mLSTM 的理论分析不足**：协方差更新规则为什么对语言建模有效？矩阵记忆的容量上界是多少？论文未给出深入分析
7. **sLSTM 在大规模中贡献不明**：xLSTM[1:0]（纯 mLSTM）在 PALOMA 上优于 xLSTM[7:1]，sLSTM 的实际价值在大规模场景下需要进一步验证
