# Llama 2: Open Foundation and Fine-Tuned Chat Models (Touvron et al., 2023)

**arxiv**: 2307.09288 | **Llama 2 / 开源 LLM 里程碑** | arxiv 2023

---

## 一、核心问题

Llama 1 证明了开源模型可以在性能上接近闭源模型（如 GPT-3），但仍存在明显差距：
- 训练数据量不足（1.4T tokens vs 闭源模型的数万亿）
- 上下文长度较短（2048）
- 缺乏可靠的对齐方法，对话能力薄弱
- 没有系统性的安全评估和缓解措施

Llama 2 的核心目标：**将开源 LLM 的性能推到与闭源模型（ChatGPT/PaLM）同等的水平，同时提供完整的对齐方案和安全保障**。

## 二、关键创新

### 2.1 预训练改进

**数据量**：2T tokens（Llama 1 的 1.4T → 2T），数据来源更广泛，元数据质量更高。

**上下文长度**：从 2048 扩展到 4096 tokens。

**GQA（Grouped-Query Attention）**：34B 和 70B 模型使用 GQA，将 KV 头数减少为 8 个（query 头数不变），大幅降低推理时 KV cache 的内存占用，提升推理速度。

GQA 是 MHA（Multi-Head Attention）和 MQA（Multi-Query Attention）的折中：
- MHA：每个 query 头有独立的 KV 头，效果最好但推理慢
- MQA：所有 query 头共享一组 KV，推理快但效果有损
- GQA：每 g 个 query 头共享一组 KV，在效果和效率间取得平衡

### 2.2 模型家族

| 模型 | 参数量 | 注意力 | 训练数据 | 上下文长度 |
|------|--------|--------|---------|-----------|
| Llama 2 7B | 7B | MHA | 2T tokens | 4096 |
| Llama 2 13B | 13B | MHA | 2T tokens | 4096 |
| Llama 2 34B | 34B | GQA | 2T tokens | 4096 |
| Llama 2 70B | 70B | GQA | 2T tokens | 4096 |

注：34B 模型未公开发布，开源版本为 7B、13B、70B。

### 2.3 系统性对齐方案

Llama 2-Chat 是 Llama 2 的对话版本，通过完整的 RLHF 流程对齐：

**阶段一：SFT（Supervised Fine-Tuning）**
- 使用高质量的对话数据（优先质量而非数量）
- 27,540 条标注对话
- 关键发现：少而精的 SFT 数据优于大量低质量数据

**阶段二：RLHF（Reinforcement Learning from Human Feedback）**
- 训练两个独立的奖励模型：Helpfulness RM 和 Safety RM
- 使用 PPO（Proximal Policy Optimization）优化
- 引入 Ghost Attention（GAtt）技术解决多轮对话中系统指令的"遗忘"问题

### 2.4 Ghost Attention（GAtt）

多轮对话中的一个常见问题：模型在第一轮遵守系统指令（如"回答要简洁"），但在后续轮次逐渐"遗忘"。Ghost Attention 的解决方法：

1. 在训练数据的每轮对话中都拼接系统指令
2. 对除了第一轮之外的所有轮次，将系统指令的 token loss 设为 0（不计算梯度）
3. 效果：模型学会在整个对话中持续遵守系统指令

## 三、方法细节

### 3.1 预训练

- **数据**：2T tokens，来自公开来源。数据清理：去重、安全过滤、去除 PII
- **Tokenizer**：BPE，词表大小 32K（与 Llama 1 相同）
- **训练**：AdamW（β₁=0.9, β₂=0.95），cosine LR schedule，warmup 2000 steps
- **硬件**：Meta 的 RSC 集群和合作伙伴集群

### 3.2 奖励模型

训练两个独立的奖励模型：

**Helpfulness RM**：
- 训练数据：Meta Helps 和 Meta HHH 数据集
- 目标：模型输出应有用、准确、相关

**Safety RM**：
- 训练数据：Meta Safety 和 Meta HHH 数据集
- 目标：模型输出应安全、无害

奖励模型架构：Llama 2 预训练模型 + 分类头（标量输出）。

训练方式：与 InstructGPT 类似的 pairwise ranking loss。

### 3.3 RLHF 训练

**迭代训练**：Llama 2-Chat 经历了多轮 RLHF 迭代（而非一次性训练）：

1. SFT → RLHF v1 → RLHF v2 → RLHF v3 → RLHF v4 → RLHF v5
2. 每轮迭代收集新的偏好数据，重新训练 RM，再优化策略
3. 后续迭代中引入了 Rejection Sampling fine-tuning：从模型采样多个回复，用 RM 选最好的，再在这些好的回复上做 SFT

**PPO 实现细节**：
- PPO 优化 + KL 散度惩罚（防止偏离参考模型太远）
- 同时使用 Helpful RM 和 Safety RM 的加权组合作为奖励
- Safety 权重在训练过程中动态调整

### 3.4 安全措施

Llama 2 在安全方面做了系统性工作：

- **预训练数据安全**：过滤有害内容、移除 PII
- **安全红队测试（Red Teaming）**：人工测试模型的有害输出
- **Safety RM**：专门训练安全奖励模型，与有用性 RM 协同工作
- **上下文蒸馏（Contextual Distillation）**：使用安全提示来引导模型拒绝有害请求
- **系统级安全**：发布 Responsible Use Guide，提供部署安全建议

## 四、实验结果

### 4.1 预训练模型性能

| 模型 | MMLU (5-shot) | HumanEval | GSM8K (8-shot) |
|------|---------------|-----------|----------------|
| Llama 1 65B | 63.4 | 23.7 | 46.0 |
| Llama 2 70B | **68.9** | **29.9** | **56.8** |
| MPT 30B | 46.9 | - | - |
| Falcon 40B | 55.4 | - | - |
| PaLM 540B | 68.9 | - | 56.9 |

Llama 2 70B 在 MMLU 上达到 68.9%，与 PaLM-540B 持平——一个 70B 开源模型追平了 540B 闭源模型。

### 4.2 对话模型性能

**人类评估 vs 闭源模型**：

| 模型对比 | Llama 2-Chat 70B 胜率 |
|---------|----------------------|
| vs ChatGPT (GPT-3.5) | 36% 胜 / 31.5% 平 / 32.5% 负 |
| vs PaLM-2 | 接近或略优 |

Llama 2-Chat 70B 与 ChatGPT 的表现接近，这是开源模型首次在对话能力上接近 ChatGPT。

**有用性 vs 安全性的权衡**：

Helpfulness RM 和 Safety RM 之间存在张力——过度强调安全可能导致模型拒绝合理请求（"过度拒绝"）。Llama 2 通过动态调整两个 RM 的权重来平衡。

### 4.3 缩放行为

- Llama 2 在所有规模上优于 Llama 1
- 70B 模型在推理、编码、数学等任务上显著优于更小的变体
- GQA 在 70B 模型上带来了约 2 倍的推理加速，性能损失可忽略

### 4.4 RLHF 迭代的效果

随着 RLHF 迭代轮次增加：
- 有用性评分持续上升
- 安全性评分也持续上升
- 输出长度略有增加（模型学会给出更详细的回答）
- 但后期迭代中提升幅度递减

## 五、影响

1. **开源 LLM 的民主化**：Llama 2 以免费研究和商业使用许可发布（对大企业有限制），让更多研究者和开发者能使用强大的 LLM。这直接催生了开源 LLM 生态的繁荣——Llama 2 成了微调、对齐、应用开发的标准基座。

2. **GQA 成为标准**：Llama 2 验证了 GQA 在大规模模型上的有效性，此后 GQA 成为几乎所有主流 LLM 的标配（Mistral、Qwen、DeepSeek 等均采用）。

3. **完整的对齐方法论**：Llama 2 论文详细描述了 SFT + RLHF 的完整流程，包括迭代式 RLHF、Ghost Attention、双 RM 策略等。这为开源社区提供了可复现的对齐方案。

4. **安全作为一等公民**：Llama 2 是第一个将安全评估和缓解措施作为论文核心部分的 LLM 论文。双 RM 架构、红队测试、上下文蒸馏等安全措施成为后续开源 LLM 的参考标准。

5. **开源 vs 闭源的竞争格局**：Llama 2-Chat 与 ChatGPT 接近的表现，证明了开源模型可以与闭源模型竞争。这改变了行业格局——此后闭源模型的护城河不再是模型能力本身，而是数据飞轮、产品体验和生态系统。

6. **局限与后续**：
   - 4096 的上下文长度仍然偏短（同期 GPT-4 已支持 32K+）
   - 34B 模型未开源，留下空白
   - RLHF 训练过程工程复杂度高，复现困难
   - 训练数据细节不够透明
   - Llama 2 Chat 在代码和数学推理上仍弱于 GPT-4
   - 这些问题在 Llama 3（2024）中得到了部分解决：更长的上下文、更大的数据量、更好的推理能力
