# Improved Baselines with Momentum Contrastive Learning (Chen et al., 2020)

**arxiv**: 2003.04297 | **关键词**: MoCo v2, 对比学习, 自监督, SimCLR | Facebook AI Research (FAIR) | 2020

---

## 一、核心问题

SimCLR 通过三个设计改进（大 batch、MLP projection head、更强数据增强）在对比学习上取得了显著进展，但需要 4k-8k 的超大 batch，依赖 TPU 硬件。本文研究 SimCLR 的改进是否可迁移到 MoCo 框架，以及 MoCo 是否能在不使用大 batch 的情况下达到或超过 SimCLR 的性能。

## 二、方法

### 2.1 背景对比

**InfoNCE 损失**：

$$\mathcal{L}_{q,k^+,\{k^-\}} = -\log \frac{\exp(q \cdot k^+ / \tau)}{\exp(q \cdot k^+ / \tau) + \sum_{k^-} \exp(q \cdot k^- / \tau)}$$

- **End-to-end 机制**（SimCLR）：负样本来自同一 batch，需要大 batch
- **MoCo 机制**：负样本维护在队列中，仅 query 和正 key 在 batch 内编码，动量编码器保持一致性

### 2.2 两个关键改进

从 SimCLR 迁移到 MoCo 的改进：

1. **MLP Projection Head**：将 MoCo 中的 fc head 替换为 2 层 MLP（隐藏层 2048 维 + ReLU），仅影响自监督训练阶段，下游评估不使用

2. **更强的数据增强**：在 MoCo 原有增强基础上添加 SimCLR 的 blur 增强（发现更强的颜色失真在 MoCo v2 更高基线上收益递减）

### 2.3 其他调整

- 学习率 schedule：采用 SimCLR 使用的余弦退火（cosine decay）schedule
- 温度参数 $\tau$：MLP head 最优 $\tau$ 从 0.07 变为 0.2

## 三、实验结果

**ImageNet 线性分类（ResNet-50）**：

| 方法 | Batch | Epochs | Top-1 |
|------|-------|--------|-------|
| MoCo v1 | 256 | 200 | 60.6 |
| + MLP | 256 | 200 | 66.2 |
| + aug+ | 256 | 200 | 67.3 |
| + MLP + aug+ + cos | 256 | 200 | 67.5 |
| + MLP + aug+ + cos | 256 | 800 | 71.1 |
| SimCLR | 256 | 200 | 61.9 |
| SimCLR | 4096 | 200 | 66.6 |
| SimCLR | 4096 | 1000 | 69.3 |

MoCo v2 在 batch=256 下 200 epochs 即达 67.5%，超过 SimCLR batch=4096 的 66.6%；800 epochs 达 71.1%，超过 SimCLR 1000 epochs 的 69.3%。

**VOC 目标检测**：MoCo v2 的 AP 从 55.9% 提升到 57.4%（800 epochs），超过监督预训练的 53.5%。

**计算成本**：MoCo batch=256 仅需 5.0G GPU 内存，而 SimCLR batch=4096 需 93.0G（8 GPU 机器无法运行）。

## 四、个人评价

### 影响

MoCo v2 是一篇简洁但影响深远的论文，证明了大 batch 并非对比学习好效果的必要条件，关键在于算法设计。这使得最先进的自监督学习从需要 TPU 的特权变为在普通 8-GPU 机器上即可复现，极大降低了研究门槛。MoCo v2 的消融实验也为后续对比学习研究提供了清晰的设计指南。

### 局限

1. 论文本身较短，未深入分析为何 MLP head 和增强改进在 MoCo 中有效
2. 仅迁移了 SimCLR 的部分改进，未探索更多可能性
3. MoCo 的队列和动量编码器增加了工程复杂度
