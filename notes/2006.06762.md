# Ansor: Generating High-Performance Tensor Programs for Deep Learning (Zheng et al., 2020)

**arxiv**: 2006.06762 | **Auto Schedule 开山** | UC Berkeley / OSDI 2020

---

## 一、核心问题

深度学习框架需要为不同硬件平台生成高性能的算子实现（tensor program）。当前方法要么依赖手写内核库（cuDNN, MKL），要么用搜索策略自动生成——但现有搜索空间的覆盖率不够，且探索策略效率低。

TVM 依赖人工模板定义搜索空间，覆盖有限。Halide auto-scheduler 虽然自动化但过早剪枝，错过了很多高性能程序。

## 二、方法

Ansor 有三个核心组件：

### 1. Program Sampler（程序采样器）

**层次化搜索空间表示**：Sketch + Annotation

- **Sketch**：程序的高层结构（循环嵌套、计算顺序等），通过递归应用灵活的推导规则自动生成
- **Annotation**：底层的具体选择（tile size、并行化、unroll 等），有数十亿种组合

先枚举高层结构，再随机采样底层细节。随机采样保证每个程序都有被探索到的机会——不依赖启发式剪枝。

### 2. Performance Tuner（性能调优器）

随机采样的程序性能不一定好，需要微调：
- **进化搜索**：以之前的好程序 + 新采样的程序为初始种群，通过变异和交叉生成新程序
- **学习型代价模型**：用神经网络预测程序性能（比实际测量快几个数量级），评估数千个程序只需几秒
- 进化搜索解决了 Halide 的"顺序构建"限制——变异可以乱序重写

### 3. Task Scheduler（任务调度器）

DNN 被分成多个子图，需要分配搜索时间资源：
- 基于梯度下降的调度算法
- 把更多资源分配给更可能提升端到端性能的子图

## 三、实验结果

| 平台 | 加速比（vs 之前最佳） |
|------|-------------------|
| Intel CPU | 最高 3.8× |
| ARM CPU | 最高 2.6× |
| NVIDIA GPU | 最高 1.7× |

关键：Ansor 找到了现有方法搜索空间之外的高性能程序。

## 四、个人评价

Ansor 的核心贡献是**自动化了 tensor program 的搜索**，不再需要人工写 TVM 模板。它的层次化搜索空间设计很巧妙——把"程序结构"和"具体参数"分开，既保证了覆盖率又保证了搜索效率。

影响：
1. **成为 TVM 的默认 auto-scheduler**：后来被集成进 Apache TVM
2. **推动了 AI 编译器的自动化**：从手工模板到自动搜索
3. **进化搜索 + 代价模型**的范式被后续很多工作采用

局限：
- 搜索时间仍然较长（小时级别）
- 代价模型的精度影响最终性能
- 后来 MetaSchedule（TVM 的下一代）改进了搜索策略
